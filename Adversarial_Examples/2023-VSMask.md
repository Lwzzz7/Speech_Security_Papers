# VSMask: Defending Against Voice Synthesis Attack via Real-Time Predictive Perturbation

## 0. 摘要翻译

现有语音合成防御通常需离线迭代优化完整语音，计算慢且必须预先获得整段音频，不能保护语音消息、在线会议等实时流。VSMask 提出实时保护机制：预测网络根据已到达的短时语音预测下一段最有效的扰动，并以通用扰动头保护预测器尚不能覆盖的开头。方法还按人耳等响曲线加权约束扰动频段。实验显示，VSMask 在不同合成模型与场景中优于其他实时基线，接近离线 PGD，并在数字、迁移、去噪和物理环境中进行评估。

## 1. 方法动机

离线 PGD 可以逐段找到强扰动，但论文报告其 1,500 步耗时约为音频长度的 20 倍，无法实时。短通用扰动虽快，却难覆盖长语音，且周期重复易被截取/去噪。本文的核心是“预测未来扰动”：已说出的音频用来预测即将输出的保护片段，且以一个 universal header 覆盖初始盲区。

## 2. 威胁模型解读

攻击者能从语音消息、会议、社交媒体或窃听录音收集受害者语音，利用 VC/TTS 克隆，并可能先去噪或信号变换。防御者在白盒设定下知道目标 synthesis model、拥有少量本人干净语音和候选目标音色，运行 VSMask 后再上传/播放。目标是让合成语音无法通过 ASV 或欺骗人耳，同时让实时原语音可懂。论文含物理评估，但只在一个房间；完全知晓 VSMask 的神经净化、扩散恢复等强自适应攻击未解决。

## 3. 方法设计与复现级解读

### 3.1 实时预测器

流式语音到达后先转 mel spectrogram。预测网络由 7 个下采样块和 5 个上采样块组成，输出较短的未来扰动 mel，再转回波形叠加。下采样块含 reflection padding、2D Conv、BatchNorm、PReLU；上采样含 ConvTranspose 与 leaky ReLU，末端 tanh 限幅。

令 $E_s$ 为目标合成器的 speaker encoder、$y$ 为选定目标说话人。网络 $N_\theta$ 根据当前窗口 $x_t$ 预测未来时刻的 $\delta$，并最小化

$$L(E_s(x+\delta),E_s(y))-\lambda L(E_s(x+\delta),E_s(x)),$$

其中 $L$ 为 MSE。第一项诱使保护语音被编码成另一人，第二项压离原身份；训练更新 $\theta$，不同于离线 PGD 逐输入更新 $\delta$。推理时单次前向即产生未来片段，但会有固定预测延迟。

### 3.2 Universal perturbation header

预测器在开头没有历史音频，会留下可被攻击者裁剪的盲区。作者将受害者训练语音裁成长度 $t+\Delta t$ 的片段，以 PGD 优化固定 header $\delta_h$，让所有短片段的 embedding 接近 $E_s(y)$、远离 $E_s(x)$。训练预测器时也把 header 加入样本；部署时先播放 header，再滚动预测后续扰动。

### 3.3 感知加权与物理适配

全频扰动比只放低频更有效，但 1.6–4 kHz 是人耳最敏感区。VSMask 为低频、中频、高频设置不同上限，令中频阈值最小、其他频带补偿能量。物理实验另加入 500–3,000 Hz band-pass 与 RIR 滤波，以面对空中传播失真；这并非数字实验默认条件。

### 3.4 配置与复现

| 项目 | 论文设置 |
|---|---|
| 目标模型 | AdaIN-VC、AutoVC、SV2TTS |
| 数据 | VCTK 109 位说话人；SV2TTS 用 LibriSpeech，语音长于 3 秒 |
| 输入窗口/延迟/输出 | 1.25 s / 0.4 s / 0.4 s |
| header | 1.65 s |
| 评价阈值 | SpeechBrain ECAPA-TDNN，ASV 阈值 0.25 |
| 硬件/时延 | 单人训练 A6000 少于 5 min；CPU 推理约 3 ms，特征转换约 200 ms |

RIR、目标音色选择、扰动预算与实时音频转换实现都会影响结果；论文未给出跨设备的完整校准方案。

## 4. 与其他方法对比

| 方法 | 时机 | 强项 | 弱项 |
|---|---|---|---|
| 离线 PGD/Attack-VC | 完整音频后 | 效果强 | 不能实时 |
| 随机/周期通用噪声 | 流式 | 快 | 长语音覆盖弱或可去噪 |
| VSMask | 流式预测+header | 兼顾开头与后续、一次前向 | 白盒训练、存在延迟 |

## 5. 实验表现与优势

表 1 在 AdaIN-VC 上比较：原始合成 ASR 为 86.0%–93.2%，VSMask 在四种性别转换中均为 0%，与离线 PGD 相近；保护原语音的 STOI 为 0.839。图 7 在 AdaIN-VC、AutoVC、SV2TTS 上均显示 VSMask 后无样本通过 ASV。人听中，超过 90% 将噪声视为可接受，且对保护后样本合成的语音，超过 75% 判断为明显不同人。迁移表的 ASR 仍有 0–15.2%，说明纯黑盒泛化不完美。自适应去噪/WaveGuard 结果仍维持低 similarity；物理设置下 46 dBA 已比同音量白噪声更有效，50 dBA 可使 ASR 为 0%。

## 6. 学习与应用

当前论文材料未提供明确官方代码链接。最小复现应先实现数字域“header + 预测网络”并固定 AdaIN-VC encoder；实时版本需正确处理窗口、缓冲和 0.4 秒延迟。用于实际会议前还应测 VAD、编解码、回声消除、不同麦克风阵列和攻击者相位抵消。

## 7. 总结

用通用头与未来扰动预测实现实时语音克隆防护。

## 9. 复现难度与适合人群

难度为**高**：需训练每位说话人的预测器、接入实时音频链路和 VC/TTS/ASV 评估；物理实验还需声学标定。适合实时语音隐私、语音深伪防御和流式对抗信号研究。

## 10. 简短全面总结

VSMask 解决离线语音保护无法覆盖直播/通话的问题。它在白盒 speaker encoder 上训练一个 7 下采样、5 上采样的预测网络：利用当前 1.25 秒语音，在 0.4 秒延迟后生成下一 0.4 秒扰动；同时用 1.65 秒 PGD 通用 header 填补开头盲区，并按等响曲线压低人耳敏感中频。VCTK/LibriSpeech 与三类合成模型的实验中，VSMask 使合成音频 ASV 成功率降到 0%，接近离线 PGD，却有约 0.2 秒生成开销；人听和单房间播放实验也提供可用性与物理证据。其关键贡献是把对抗式克隆防护转成流式预测问题；限制是训练白盒、跨模型仍残留成功率、物理验证单一且强自适应净化未覆盖。

## 11. 论文写作逻辑分析

论文将“离线最强、实时不可用”的矛盾拆成未来预测和起始盲区两个模块，再用 header 与感知加权逐项补足。实验从数字基线、跨模型、人听、自适应处理到物理场景，较好贴合叙事。值得注意的是“无额外时延”应理解为可在 0.4 秒缓冲内运行，而不是零延迟；论文也诚实列出了单房间和强自适应攻击的限制。

# SafeSpeech: Robust and Universal Voice Protection Against Malicious Speech Synthesis

## 0. 摘要翻译

逼真的深度伪造语音可能被用于诈骗，但现有防御难以同时抵抗微调克隆和鲁棒训练。SafeSpeech 在用户上传前向音频加入保护扰动，提出“关键目标函数”以从不同 TTS 的多目标训练中选择普适且易收敛的 mel 重建目标，并提出 SPEC 让在保护数据上训练的生成器输出趋近随机噪声。方法同时利用 STOI 与 STFT 损失改善听感。在多模型、多数据集、主客观实验中，作者报告该方案同时降低音色相似度与合成质量，并对零样本克隆保持迁移性。

## 1. 方法动机

早期声纹扰动常只干扰零样本 reference embedding，攻击者若用保护数据微调、去噪或鲁棒训练仍可能恢复克隆。本文转向训练期数据保护：使攻击者在保护数据上训练时“误以为已拟合”，实际学不到目标说话人。难点是防御者不知道攻击者会选哪一个多目标 TTS，因此不能依赖模型特有的完整损失。

## 2. 威胁模型解读

攻击者可爬取已上传的保护音频、选用先进 TTS、微调或零样本克隆，并采用去噪、数据增强、对抗训练、污染防御甚至模型重训；但不能取得未保护原音频。防御者只可修改上传前的音频，不知道攻击模型/训练法。目标同时是 SA1（音色不再像受害者）和 SA2（合成语音不可正常使用），且保护音频本身可接受。方法用替代合成器构造扰动，故“模型无关”是经验迁移主张，不是对任意攻击模型的证明。

## 3. 方法设计与复现级解读

### 3.1 全局流程与关键目标

给定 $(x,text)$，冻结/使用替代 TTS $M$，只迭代 $\delta$ 生成 $x'=x+\delta$。作者不是直接优化 BERT-VITS2 等模型的全部多任务损失，而选生成输出 $\hat x$ 与输入 $x$ 的 mel $L_1$：

$$L_{mel}=\|\hat x_{mel}-x_{mel}\|_1.$$

其理由是该项依赖可被防御者改动的音频、跨生成器普遍存在，且图 3 中比其他候选损失收敛更快。方法采取 error-minimizing 而非 error-maximizing：把该项压低，使模型在保护数据上得到误导性“已拟合”信号。

### 3.2 SPEC 数据保护

只压低 $L_{mel}$ 仍可能留下可辨语音。作者采样高斯噪声 $z$，令合成输出在 mel 表示上接近噪声：

$$L_{noise}=D_{KL}(\hat x_{mel},z_{mel})+\|\hat x_{mel}-z_{mel}\|_1,\quad L_{SPEC}=L_{mel}+\beta L_{noise}.$$

第一项把输出分布推向噪声，第二项稳定其频谱接近；这服务 SA2。注意这是替代 TTS 的优化目标，部署时只发布 $x'$，攻击者训练的模型不会运行 SPEC。

### 3.3 感知约束

单纯 $L_p$ 幅度限制不等于听感。论文以 STOI 保持短时包络可懂度、以 STFT 的 $L_2$ 约束时频差异，构成 $L_{perception}=L_{stoi}+L_{stft}$，总目标为 $L_{SPEC}+\alpha L_{perception}$，并在每步将扰动投影到半径 $\epsilon$。这部分服务“保护音频还能发布”，与 SPEC 的破坏性目标存在权衡。

### 3.4 配置与复现风险

| 项目 | 设置 |
|---|---|
| 数据 | LibriTTS 单说话人；CMU ARCTIC 18 人、每人 100 条、80/20 划分 |
| surrogate/评测 TTS | BERT-VITS2、StyleTTS2、MB-iSTFT-VITS、VITS、GlowTTS；零样本另含 XTTS、F5-TTS 等 |
| 预算与权重 | $\epsilon=8/255$，$\alpha=0.05$，$\beta=10$ |
| 攻击者训练 | 单说话人 100 epochs，多说话人 200 epochs |
| 硬件 | 1 张 NVIDIA A800 |
| 指标 | SIM/ASR、MCD、WER、SNR、UTMOSv2、MOS |

随机种子、各替代模型的精确实现/更新次数和完整信号预处理不足以唯一复现。白盒替代目标与强自适应攻击之间仍有证据缺口。

## 4. 与其他方法对比

| 类别 | 防御位置 | 优点 | 边界 |
|---|---|---|---|
| AntiFake/AttackVC | reference/零样本表示 | 直接破坏音色 | 对微调训练弱 |
| AdvPoison、SEP、PTA | 训练数据 | 面向可用性污染 | 未专门兼顾语音音色/听感 |
| SafeSpeech | 训练数据+替代 TTS | 同时追求 SA1、SA2 与迁移 | 依赖 surrogate 经验迁移 |

## 5. 实验表现与优势

表 1 测试在保护数据上微调多种 TTS：LibriTTS 上 SPEC 的 WER 从干净数据的 24.024% 升至 99.610%，SIM 降至 0.204，分别支持质量和音色保护。表 2/4 检查保护样本自身，SIM 0.859、98.333% 人类认为同一说话人，说明不是直接毁坏原声。表 3 中保护数据训练出的合成音 MOS 为 $1.070\pm0.161$，远低于干净训练的 $4.677\pm0.114$。零样本实验中，F5-TTS 的 SIM 从 0.885 降至 0.094。证据覆盖多架构与人听，但鲁棒训练/净化的全面对抗未形成统一端到端基准。

## 6. 学习与应用

论文页面/材料未在文中给出明确官方代码链接。复现最低版本应使用一个可微 TTS，按 SPEC 与感知损失优化单条/数据集扰动，再重新训练不同架构验证。对生产部署应额外评估转码、剪辑、去噪、数据混合与攻击者知晓算法后的自适应重训。

## 7. 总结

以 SPEC 让保护语音在训练中变成不可学习的噪声目标。

## 9. 复现难度与适合人群

难度为**高**：既要可微 TTS 优化，又要多次重新训练攻击模型，且需主观评测。适合语音深伪防御、训练数据投毒防护与 TTS 评测研究者。

## 10. 简短全面总结

SafeSpeech 面向攻击者用公开语音微调或零样本克隆的风险，提出上传前的训练数据保护。它以替代 TTS 为优化器，从复杂多任务目标中选择跨模型可计算且快速收敛的 mel 重建项，并通过 SPEC 将受保护数据训练出的输出拉向高斯噪声；STOI/STFT 则抑制保护音频的感知退化。LibriTTS 与 CMU ARCTIC、多种微调 TTS 和五种零样本模型的实验显示，SPEC 可显著降低 SIM、提高 WER/MCD，同时多数听者仍把保护音频认作同一人。其贡献是把语音克隆防护扩展到训练期不可学习性与合成质量破坏；主要限制是对替代模型的依赖及对完全自适应恢复训练的有限证据。

## 11. 论文写作逻辑分析

文章先把零样本防御不足归因于攻击者可微调，再用“不知道攻击 TTS”导出 pivotal objective，最后用 SPEC 补齐“只破音色不够”的质量目标。实验按训练期、感知、零样本、消融组织，较完整地对应两项安全目标。需要谨慎的是，快速收敛被用作普适性的证据，但它并不自动保证所有未知生成器上的迁移。

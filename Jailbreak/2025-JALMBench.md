# JALMBench: Benchmarking Jailbreak Vulnerabilities in Audio Language Models

## 0. 摘要翻译

LALM 已广泛部署，却缺少面向音频越狱的对抗数据和统一比较框架。JALMBench 包含 11,316 条文本、245,355 条音频（超过 1,000 小时），支持 12 个主流 LALM、8 种攻击（4 种文本迁移、4 种音频原生）和 5 种防御。系统分析显示安全性受模态和架构强烈影响：文本安全对音频只能部分迁移，交错音频-文本策略有较好跨模态泛化；通用 moderation 对提升安全仅有轻微效果。

## 1. 方法动机

不同论文使用不同有害问题、TTS 声音、攻击成功定义和模型接口，导致“哪种 LALM 更安全”无法公平比较。JALMBench 把数据、攻击、模型和防御矩阵化，重点分析文本安全对音频输入的转移边界。

## 2. 威胁模型解读

攻击者可用文本越狱 prompt 经 TTS 转入音频，或直接操控音频信号；目标是绕过对齐，诱导有害回答。攻击在数字输入、已知模型 API 条件下评测，强度由攻击方式和查询/样本预算决定。防御者可做 prompt-level 安全干预、response moderation 或其他五类防御。该 benchmark 不保证攻击者能在物理声道、未知闭源系统上复现实验结果。

## 3. 方法设计与复现级解读

### 3.1 数据与覆盖矩阵

JALMBench 以有害文本问题为语义源，扩展不同语言、口音、性别、TTS 方法与声音条件，得到 245,355 音频、超过 1,000 小时。其价值不仅是大规模，还在于让同一语义问题跨声音/攻击条件配对，便于隔离声音多样性与攻击方法的影响。

### 3.2 八类攻击与指标

攻击分两群：四类**文本迁移**（先在文本域形成越狱，再语音化）和四类**音频原生**（直接利用音频输入/扰动）。论文报告 ASR-a、ASR-q 等指标：前者针对回答是否有害，后者针对模型是否回答/拒答；二者共同避免把“输出了但不有害”误记为越狱。具体每种攻击实现、查询预算和声音生成参数以附录/数据代码为准。

### 3.3 防御与架构分析

五类防御覆盖输入 prompt 和输出 moderation。作者还比较音频表示到 LLM 的架构，发现文本对齐只能部分转移到 audio；交错音频-文本的策略更能在模态间保持安全。表征可视化用于解释有害/良性语音、静音与非语音输入如何在 encoder/LLM 空间移动。

## 4. 与其他方法对比

| 基准 | 规模与攻击 | 重点 |
|---|---|---|
| 单一攻击论文 | 固定模型/少量样本 | 证明某条漏洞 |
| CAA | 对话扰动 | 对话鲁棒性 |
| JALMBench | 8 攻击×12 模型×5 防御 | 可比的安全矩阵与架构分析 |

## 5. 实验表现与优势

大规模结果显示攻击效率随主题、声音和模型变化；文本安全训练并不会自动保证音频安全。论文特别报告开源音频 LMM 在有害音频问题上的平均 ASR 为 69.14%，并提出语音专属 jailbreak 在 Gemini-1.5-Pro 的有害问题集上为 70.67%（该项与 RT-LALM 的相关红队实验相呼应）。通用 moderation 仅小幅改善，支持“需音频专用防御”。但大矩阵实验也受自动 judge、一致的 TTS 管线与服务版本漂移影响。

## 6. 学习与应用

当前 PDF 未提供明确官方仓库 URL。最小复现应先固定一组有害问题、一个文本迁移和一个音频原生攻击、ASR-a/ASR-q judge，再扩展声音多样性与防御。不要把自动 moderation 的低 ASR 视为真实安全证明，应人工抽检边界样本。

## 7. 总结

以大规模攻击—模型—防御矩阵测量 LALM 音频越狱。

## 9. 复现难度与适合人群

难度为**高**：数据超过千小时、12 模型与闭源 API 成本很高。适合 LALM 安全基准、跨模态对齐与防御研究者；实践者可先抽取小矩阵回归测试。

## 10. 简短全面总结

JALMBench 为音频语言模型提供统一的越狱评测矩阵：11,316 个文本语义源经多声音条件扩展为 245,355 条音频，配套四类文本迁移攻击、四类音频原生攻击与五类防御，并覆盖 12 个 LALM。它用回答有害性与拒答行为等指标拆分 ASR，进一步分析主题敏感性、声音多样性及架构。结果表明文本安全对音频的迁移并不充分，交错音频-文本架构更稳健，而通用内容审核改善有限。其贡献是从零散案例推进到可比较的大规模安全测量；限制是评测依赖自动判定、TTS/服务版本和数字输入，不能替代真实部署红队。

## 11. 论文写作逻辑分析

文章把“研究碎片化”转成统一矩阵的工程问题，再用分解指标和表征分析解释结果，而非只报 ASR 排名。强在覆盖与比较框架；若更明确报告每类攻击的可操作能力与查询成本，threat model 会更可审计。

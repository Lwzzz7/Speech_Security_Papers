# Audio Is the Achilles’ Heel: Red Teaming Audio Large Multimodal Models（RT-LALM）

## 0. 摘要翻译

文本安全对齐是否能一致地延伸到音频 LMM 仍缺少研究。本文在三种设置红队五个先进音频 LMM：有害问题以音频或文本输入、有害文本配合干扰非语音、以及语音专属越狱。结果表明开源音频 LMM 对有害音频问题的平均攻击成功率为 69.14%，并会被非语音噪声分散安全性；语音专属越狱在 Gemini-1.5-Pro 的有害问题基准上取得 70.67% ASR。

## 1. 方法动机

许多 LMM 的安全对齐主要在文本输入完成，但音频 encoder、跨模态对齐和非语音声学内容会改变进入 LLM 的表示。本文不假设“文本拒答=语音拒答”，而系统地对比同一有害意图在文本、语音和带干扰音频中的行为。

## 2. 威胁模型解读

攻击者能提交音频、文本或二者组合；可以在有害文本旁加入非语音声，或把特定 jailbreak prompt 朗读成语音。攻击目标是获取有害非拒答。实验对模型 API/公开系统做红队，不需白盒梯度；但并不证明空气录音、设备差异或低查询预算下同样有效。防御者需要在音频 encoder、跨模态融合和解码/审核层共同处理安全，而非只依赖文本对齐。

## 3. 方法设计与复现级解读

### 3.1 三种红队设置

1. 将相同有害问题分别作为音频与文本输入，直接测跨模态安全迁移。
2. 保持有害文本，添加非语音干扰（如噪声/声学输入），测试音频是否分散安全行为。
3. 将一个包含角色、输出格式/拒答抑制等约束的 speech-specific jailbreak prompt 语音化，后接有害问题，专门攻击语音指令通道。

论文使用有害问题集，按回答有害性与拒答行为计算 ASR-a/ASR-q；并用 Llama Guard 3 评估，且核对其与人评的一致性。

### 3.2 表征分析

作者通过 t-SNE 等可视化比较 harmful/benign、非语音/静音输入在 audio encoder 与后续 hidden state 的位置，并跟踪随非语音强度变化的 ASR。这不是因果证明，却为“音频输入可把表示推离文本安全区域”提供诊断证据。

## 4. 与其他方法对比

| 评测 | 核心问题 | 特点 |
|---|---|---|
| 文本 jailbreak | 文本对齐能否绕过 | 忽略 audio encoder |
| 噪声鲁棒性 | 识别是否下降 | 不一定测试安全 |
| RT-LALM | 模态安全是否一致 | 同时比较语音、有害文本+非语音、语音 jailbreak |

## 5. 实验表现与优势

五模型红队中，开源音频 LMM 对有害音频问题平均 ASR 69.14%，说明文本安全不能可靠迁移；非语音干扰也暴露额外漏洞。语音专属 jailbreak 在 Gemini-1.5-Pro 上达到 70.67% ASR。实验的优势是同一风险在多输入设置下对照；限制是攻击 prompt 具有较强手工设计性，且不同模型的系统提示、审核层和版本变化可能影响绝对 ASR。

## 6. 学习与应用

PDF 未给出明确官方代码 URL。最小复现可从同一 harmful query 的 text/audio 配对开始，固定 ASR judge，再逐步引入非语音与语音专属 prompt。部署评估应测试音频前处理、系统提示、工具调用与人工确认，不宜仅用 Llama Guard 分数下结论。

## 7. 总结

文本安全对齐会在音频输入通道出现明显断层。

## 9. 复现难度与适合人群

难度为**中高**：模型调用和安全审核可复现，人工评审和稳定 API 版本较难。适合音频 LMM 对齐、红队与跨模态安全研究。

## 10. 简短全面总结

RT-LALM 以“文本对齐是否迁移到音频”作为核心问题，对五个音频 LMM 进行三路红队：有害问题的文本/音频对照、有害文本加非语音干扰、以及语音专属 jailbreak。它用 ASR-a/ASR-q 与自动/人工一致性评价，报告开源模型在有害音频上的平均 ASR 达 69.14%，且非语音也能扰乱安全；手工构造的语音提示在 Gemini-1.5-Pro 上达 70.67%。论文贡献是把模态安全不一致变成可测假设并辅以表征分析；边界是数字输入和手工提示，不覆盖物理传播、闭源系统完整防线或自适应防御。

## 11. 论文写作逻辑分析

文章以一个简洁反事实“若文本安全成立，音频是否也应安全”组织三种逐步加强的测试，逻辑清晰。表征可视化补充了现象解释，但不能单独证明机制；最佳写法是同时报告攻击样本、judge 误差和服务端配置。

# AudioJailbreak: Jailbreak Attacks Against End-to-End Large Audio-Language Models

## 0. 摘要翻译

现有 LALM 越狱大多假设攻击者能完整操控用户提示（强攻击者），且在有效性、适用性和实用性上受限。本文首先说明先进文本越狱经 TTS 后难以直接迁移到端到端 LALM，继而提出 AudioJailbreak：后缀型 jailbreak audio 实现时间异步；在生成期合并多个正常用户提示实现通用性；以多种策略隐藏恶意意图；并在优化中加入混响以保持空中播放有效。方法也适用于不知道用户会说什么、说多久的弱攻击者场景。

## 1. 方法动机

端到端 LALM 不一定以文本作为中心中间表示，文本越狱中的特殊 token、过长 prompt 和 TTS 转换常失效。更现实的攻击者还可能只能在受害者语音旁播放音频，无法预先替换整条用户提示。本文将问题改为：能否生成与用户语音不同步、可跨未知内容复用、且不明显暴露意图的音频后缀/伴随声？

## 2. 威胁模型解读

**强攻击者：** 可完全控制用户 prompt，可为特定请求构造音频。**弱攻击者：** 无法预知/修改用户说什么或说多久，只能播放预先准备的 jailbreak 音频；因此攻击需要异步和通用。两者目标均是让端到端 LALM 输出有害非拒答内容。攻击可经空中播放，模型接收录入后的混合音频。该场景比文本攻击更贴近旁路注入，但论文的优化仍依赖模型/替代条件，现实硬件、距离和多说话人干扰范围有限。

## 3. 方法设计与复现级解读

### 3.1 后缀与通用扰动构造

为避免与未知用户语音逐帧对齐，AudioJailbreak 将攻击音频设计为**后缀**，让其在用户发言结束或相邻时段出现；这提供 asynchrony。样本特定版本针对单个 prompt 优化；通用版本在生成目标中加入多条正常用户 prompt，使同一扰动在不同内容上仍触发越狱。其关键不是普通时间平移，而是攻击音频在语义/时间上不必与 victim prompt 一一对应。

### 3.2 意图隐藏与 OTA 鲁棒性

弱攻击者的攻击音频若直接朗读恶意指令，受害者可察觉。论文提出多种 concealment strategy，使恶意 intent 不易被听出（具体策略与参数见正文/附录）。为对抗播放—录音失真，生成时把 room reverberation 纳入扰动优化，相当于在房间响应分布下优化期望攻击效果。该 EOT 式设计服务 OTA，但不能代表未见房间和设备的保证。

### 3.3 评价设置与关键数字

作者先在 50 个 AdvBench harmful prompts 上比较 5 种文本越狱经 XTTS-v2 转音频的迁移，发现端到端 LALM 上通常很弱，因而需要音频原生攻击。随后跨多 LALM、两数据集和两种攻击者设置评估 ASR、通用性、隐蔽性及 OTA。文中报告样本特定攻击在所有所测 LALM 上强攻击者至少 46% ASR、弱攻击者接近 100%；通用攻击分别约 70%/55%，OTA 强/弱约 88%/70%。这些是论文汇总结论，具体模型间差异不可省略。

### 3.4 复现风险

后缀长度、通用 prompt 池、房间脉冲响应分布、隐藏策略、播放音量/距离、模型解码和有害性 judge 都决定 ASR；当前 PDF 的工程实现细节不能由摘要唯一推断。任何防御评测都应将这些变量报告为 threat model 的一部分。

## 4. 与其他方法对比

| 类别 | 攻击者假设 | 异步/通用/OTA | 局限 |
|---|---|---|---|
| 文本 jailbreak→TTS | 控制完整 prompt | 通常仅部分异步 | 对端到端 LALM 迁移差 |
| 逐样本音频优化 | 强攻击者 | 多不通用 | 不适合未知用户内容 |
| AudioJailbreak | 强/弱攻击者 | 后缀异步、跨 prompt、含 RIR | 构造与物理条件仍复杂 |

## 5. 实验表现与优势

论文的主实验依次验证文本迁移失败、样本特定/通用攻击、隐藏性、OTA 与弱攻击者。它还声称能在弱攻击者情形下 jailbreak GPT-4o-Audio 并绕过 Llama-Guard-3，说明单独在输出端加通用 safeguard 并不充分。文中还评估三种防御，但摘要未表明其可彻底消除攻击。证据强在同时覆盖异步、通用与 OTA；边界在于真实场景的攻击者设备、用户可感知性和模型服务版本未必与实验一致。

## 6. 学习与应用

当前 PDF 未给出明确官方代码链接。该工作涉及可使系统产出有害内容的攻击，复现应限制在获得授权的本地模型与防御测试环境。防御侧可从其设定导出回归测试：尾部异常音频、跨内容通用触发、RIR 增强和 audio-output moderation 的联合作用。

## 7. 总结

以通用后缀音频在弱攻击者和空中通道下越狱端到端 LALM。

## 8. 图表精读与证据链

强/弱攻击者图是全文的设定核心；对比表明确 prior work 缺哪种属性；文本迁移实验提供“为何不能只 TTS”证据；后续 ASR、OTA 和防御表把方法主张逐一落地。最应审查的是弱攻击者是否真不控制用户 prompt、以及每种 OTA 环境上的独立成功率。

## 9. 复现难度与适合人群

难度为**很高**：涉及端到端 LALM、多 prompt 通用优化、隐蔽性评估和物理声学测试。适合授权的音频 LLM 红队、OTA 鲁棒性和多模态对齐防御研究。

## 10. 简短全面总结

AudioJailbreak 将音频越狱从“攻击者替用户说完整恶意 prompt”扩展到更实际的弱攻击者：攻击者不知道受害者内容与时长，只能播放独立音频。方法以后缀实现不同步，以多用户 prompt 联合构造通用扰动，以隐藏策略降低意图可闻性，并在优化中加入房间混响提高 OTA 鲁棒性。实验先证明文本越狱经 TTS 难直接迁移到端到端 LALM，再报告样本特定、通用和 OTA 攻击在强/弱场景下均有较高 ASR，且能绕过 GPT-4o-Audio 与 Llama-Guard-3 的组合。主要贡献是把异步、通用、隐蔽和物理传播统一进音频越狱 threat model；限制是攻击构造复杂、现实声学条件有限，且未证明能抵御自适应输入净化和端到端鲁棒训练。

## 11. 论文写作逻辑分析

文章先用端到端 LALM 的文本迁移失败建立方法必要性，再以弱攻击者的未知内容/时长引出后缀和通用性，最后补足隐蔽与 OTA。叙事与 threat model 高度一致。值得借鉴的是把真实约束提前写入方法；需注意绝对 ASR 必须连同房间、设备、音量和查询预算解释。

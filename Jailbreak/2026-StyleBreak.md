# StyleBreak: Revealing Alignment Vulnerabilities in Large Audio-Language Models via Style-Aware Audio Jailbreak

## 0. 摘要翻译

LAM 通过音频编码器与 LLM 结合，实现自然语音交互，但其音频越狱安全仍研究不足。既有工作多把文本攻击朗读为语音或使用浅层信号扰动，忽略人类语音表达变化对对齐鲁棒性的影响。StyleBreak 提出风格感知音频越狱：以两阶段变换控制语言、旁语言和外语言属性，并以查询自适应策略网络搜索对抗风格。实验显示多样说话属性可暴露对齐漏洞，StyleBreak 在多种攻击范式上同时提升攻击效果与查询效率。

## 1. 方法动机

同一有害意图被不同年龄、性别、语速、韵律和情绪表达时，音频 encoder 可能将其送入不同表示区域；文本安全对齐不必在所有说话风格上保持一致。本文不把风格仅视为听感变量，而将它看作可改变安全决策边界的输入自由度。

## 2. 威胁模型解读

攻击者可选择/生成语音内容与 style reference，并向 LAM 发送音频提示；可基于少量模型响应迭代选择风格。目标是诱导 harmful、policy-violating 且非拒答回答。论文同时评估 text-only 和 audio-based 攻击，不依赖波形白盒梯度；但攻击者仍需可控 TTS/风格参考和查询反馈。未验证物理传播、未知防御随机化或低时延现实交互。

## 3. 方法设计与复现级解读

### 3.1 两阶段风格变换

Pipeline I 以情绪驱动 prompt transformation 改写文本的表达方式；其目的不是改掉有害任务，而是制造不同语言层情绪表述。Pipeline II 将改写文本与 reference audio 配对，经 style-controlled TTS 合成攻击音频，控制旁语言/外语言属性。风格空间记作 $s=(e,g,ag)$，其中包含 7 个 emotion、gender 与 age group；GigaSpeech 中的 reference set $\mathcal X$ 为每个配置提供文本和示范音频。

### 3.2 查询自适应策略

不同查询的最危险风格并不相同（图 2 中 6 个有害查询在 20 个配置上的峰值不同）。作者训练多头策略 $\pi_\theta:Q\rightarrow\Delta(S)$，输入查询、输出风格配置分布；每个 head 给出候选，学习参数选择高收益样式，从而替代穷举 $|E|\times|G|\times|AG|$。部署期是“合成—查询—按策略再选”的有限迭代，而非无代价的离线通用触发。

### 3.3 指标与复现设置

| 项目 | 论文设置 |
|---|---|
| 查询 | AdvBench 的 200 条子集 |
| 风格语料 | GigaSpeech reference set |
| 目标模型 | Qwen2-Audio、Ultravox、MERaLiON 等四个 LAM |
| 基线 | Vanilla、AutoDAN*、GCG*、SSJ 等文本/音频攻击 |
| 指标 | ARR、ASR、Toxicity Score、Policy Violation；Llama-3-Guard judge |
| 默认查询 | 主表多为 3 次 query iteration |

策略训练细节、TTS 模型版本、风格样本选择和 judge prompt 对绝对结果敏感，论文要求读者参考附录；这些缺口会妨碍逐数复现。

## 4. 与其他方法对比

| 类别 | 操控维度 | 优势 | 边界 |
|---|---|---|---|
| 文本→TTS 越狱 | 文本语义 | 易迁移 | 忽略语音表达 |
| 信号级扰动 | 波形 | 可细粒度攻击 | 可听性/白盒问题 |
| StyleBreak | 语言+旁语言+外语言风格 | 贴近自然语音、可用于多攻击范式 | 依赖风格合成和查询 |

图 3 分别改变情绪文本、音频风格、年龄和性别，显示安全指标会随属性显著变化。表 1 表明把 StyleBreak 加到不同基线后，在多模型/范式上提高攻击效果；文中报告增幅随设置变化。图 4 显示少量迭代已迅速提高 ASR，表 2 的消融验证 emotion prompt transformation、style-controlled generation 和 query policy 均有贡献。证据支持“风格是对齐脆弱维度”，但不能把带风格 TTS 的数字查询成功外推为空中语音均会越狱。

## 6. 学习与应用

论文给出扩展版 [arXiv](https://arxiv.org/abs/2511.10692)，当前材料未见独立代码链接。方法可用于授权红队测试：对每个有害/良性意图系统扫描风格条件，检查对齐是否随旁语言属性漂移。防御上应训练跨风格一致的拒答、在 audio encoder 后做风格不变性评估，而非只过滤特定情绪词。

## 7. 总结

将自然说话风格视为可被自适应利用的音频越狱变量。

## 9. 复现难度与适合人群

难度为**中高**：需风格可控 TTS、reference 语料、多个 LAM 查询和安全 judge。适合 LAM 对齐、语音副语言学与多模态红队研究者。

## 10. 简短全面总结

StyleBreak 指出 LAM 的安全对齐不仅取决于“说了什么”，还取决于“以何种说话风格说”。方法先在文本层做情绪驱动表达变换，再以 GigaSpeech 风格参考驱动 TTS 合成，覆盖情绪、性别和年龄等语言/旁语言/外语言属性；随后用多头策略网络按查询选择最具攻击性的配置，避免穷举。AdvBench 200 条子集与多 LAM 实验表明，属性变化可明显抬高 ARR、ASR、毒性和违规指标，三个模块的消融也支持其各自贡献。论文的贡献是把自然语音风格纳入 LAM 越狱评估；主要边界是查询式、合成式数字攻击，且未充分排除自动 judge 对表达风格本身的偏差。

## 11. 论文写作逻辑分析

论文从既有攻击忽略人类表达变化的缺口，依次引出文本情绪、音频风格和自适应搜索，模块与动机一一对应。实验也按属性敏感性、攻击增强、查询效率和消融展开。值得借鉴的是将“风格空间是否值得搜索”先用图 2 的查询差异证实；但应更谨慎区分模型安全漂移与评测器受情绪措辞影响。

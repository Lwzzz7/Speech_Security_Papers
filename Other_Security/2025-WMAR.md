# WMAR：自回归图像生成的 Token 级水印

## 0. 摘要翻译

论文提出首个面向自回归图像生成的 Token 级水印方案，将大语言模型中的 KGW 绿名单/红名单采样机制迁移到离散图像 Token。作者发现，图像 Token 解码为像素后再重新编码，平均约三分之一 Token 会改变；JPEG、噪声、裁剪和翻转会进一步破坏 Token 序列，使生成阶段嵌入的水印消失。论文将这一问题称为反向循环一致性不足（Reverse Cycle-Consistency, RCC）。

方法包含两部分：一是联合微调图像解码器和一个检测专用编码器副本，提高重新 Token 化的一致性；二是额外嵌入局部同步水印，用来识别并逆转翻转、旋转和裁剪。实验在 TAMING、CHAMELEON 和 RAR-XL 上表明，该方案在保持生成质量的同时，能够抵抗常见图像处理、神经压缩、扩散净化和部分几何变换，并保留基于二项分布的理论 p-value。论文还初步将该思路扩展到 MOSHI 自回归语音生成。

## 1. 方法动机

自回归图像模型逐 Token 生成离散表示，看似可以直接复用 LLM 水印：每一步依据密钥划分绿名单和红名单，并提高绿名单 Token 的采样概率。

但验证者拿到的是像素图像，而不是生成时的 Token 序列。检测前必须重新运行图像 Tokenizer。普通 VQ Tokenizer 主要保证前向循环一致性：

$$
D(T(x))\approx x,
$$

却不保证反向循环一致性：

$$
T(D(s))\approx s.
$$

对普通图像生成而言，只要重建图像视觉上相似即可；对 Token 水印而言，Token 身份本身就是统计证据。重新编码后 Token 大量改变，会直接把绿名单比例拉回无水印水平。

## 2. 威胁模型解读

模型提供方在不修改自回归生成模型权重的前提下，于采样阶段为文本和图像输出嵌入零比特水印。验证时，提供方持有秘密密钥和专用 Tokenizer，通过统计检验判断内容是否来自该模型。

攻击者只有黑盒模型访问权，不知道密钥和 Tokenizer，但可对图像实施：

- JPEG、模糊、噪声和亮度调整；
- 翻转、旋转和裁剪；
- 神经图像压缩或 VAE 重构；
- 扩散净化等水印移除攻击。

论文主要研究水印移除，未系统覆盖水印伪造、密钥推断和已知水印样本上的自适应攻击。作者明确指出，先破坏同步信号再施加几何变化的组合攻击可使方法失效。

## 3. 方法设计与理解

### 3.1 KGW Token 水印

第 $i$ 步生成时，根据秘密密钥和前 $h$ 个 Token 的上下文，将词表划分为绿名单 $G_i$ 与红名单 $R_i$。绿名单占比为 $\gamma$，其 Logit 增加水印强度 $\delta$。

检测器计算绿 Token 数：

$$
S=\sum_{i=h+1}^{T}\mathbf{1}(s_i\in G_i).
$$

无水印假设下：

$$
S \sim \mathrm{Binomial}(T-h,\gamma).
$$

最终 p-value 为：

$$
p=\Pr[X\ge S],\qquad X \sim \mathrm{Binomial}(T-h,\gamma).
$$

因此误报率可由统计检验直接控制，而不依赖神经解码器输出 Bit 是否独立、无偏。

### 3.2 图像上下文设计

文本使用 `h = 1`。图像实验考虑：

- `h = 0`：使用固定、密钥控制的词表划分；
- `h = 1`：绿名单依赖前一个图像 Token。

固定划分在文本中容易被窃取，但图像 VQ Token 对外部攻击者较不透明，因此在部分图像模型上仍具实用性。实验中 TAMING 和 RAR-XL 主要使用 `h = 1`，CHAMELEON 图像使用 `h = 0`。

### 3.3 统一多模态检测

对于同一篇多模态内容，文本和图像分别 Token 化并计分，再合并所有模态的 $S$、$T$ 和 $h$，计算统一 p-value。

系统对重复的“上下文—Token”对去重，避免同一伪随机事件被重复计数。所有模态需使用相同 $\gamma$。联合检测可在某一模态被修改时利用另一模态补充证据，但严重损坏的模态也可能稀释整体信号。

### 3.4 RCC 定义与问题量化

给定生成 Token 序列 $s$：

$$
s'=T(D(s)).
$$

Token Match 定义为：

$$
\operatorname{TM}(s,s')=
\frac{1}{T}\sum_{i=1}^{T}\mathbf{1}(s_i=s_i').
$$

TAMING 的初始结果为：

| 图像处理 | Token Match |
|---|---:|
| 无处理 | 0.66 |
| 模糊，`kernel = 9` | 0.26 |
| 高斯噪声，`σ = 0.1` | 0.17 |
| JPEG，质量 25 | 0.31 |
| 亮度 ×2 | 0.11 |
| 旋转 10° | 0.02 |
| 水平翻转 | 0.01 |
| 保留 75% 裁剪 | 0.01 |

无攻击时已有约三分之一 Token 改变，几何变换则几乎完全打乱 Token 序列。

### 3.5 RCC 微调

VQ Tokenizer 包含编码器 $E$、码本 $C$、量化器和解码器 $D$。为避免改变 Token 语义并重新训练自回归模型，作者固定原编码器和码本，只微调：

- 生成端解码器 $D$；
- 检测端编码器副本 $E'$。

给定 Token $s$，硬 Latent 为 $\hat z=C_s$。RCC 损失为：

$$
\mathcal{L}_{\mathrm{RCC}}=
\mathbb{E}_{a\sim\mathcal A}
\left\|\hat z-E'(a(D(\hat z)))\right\|_2^2.
$$

增强集合包含亮度、对比度、JPEG 和小角度旋转等。

为保持视觉质量，加入：

$$
\mathcal{L}_{\mathrm{reg}}=
\|D(\hat z)-D_0(\hat z)\|_2^2+
\mathcal{L}_{\mathrm{LPIPS}}(D(\hat z),D_0(\hat z)).
$$

总损失：

$$
\mathcal{L}=\mathcal{L}_{\mathrm{RCC}}+
\lambda\mathcal{L}_{\mathrm{reg}}.
$$

该训练目标不是提高普通图像重建，而是让新解码器输出更容易被检测编码器重新映射回原始 Token。

### 3.6 同步层

RCC 微调能处理像素值变化，却无法解决翻转、较大旋转和裁剪导致的空间重排。

作者在图像四象限分别嵌入固定的 32-bit 局部模式：

$$
\{0^{32},\;0^{16}1^{16},\;1^{16}0^{16},\;1^{32}\}.
$$

检测时搜索旋转角度和正交分割线，根据四个模式的位置估计翻转、旋转和保留角点的裁剪，再逆变换图像并执行 KGW 检测。

同步信号只负责校准，不作为最终来源证明；最终结论仍来自 Token 水印的二项 p-value。

### 3.7 关键配置

| 项目 | 设置 |
|---|---|
| 图像模型 | TAMING、CHAMELEON-7B、RAR-XL |
| 分辨率 | 256×256；CHAMELEON 为 512×512 |
| Tokenizer | VQGAN 或相关 VQ Tokenizer |
| 码本大小 | 16384、8192、1024 |
| 水印参数 | `δ = 2, γ = 0.25` |
| RCC 数据 | ImageNet 50,000 张训练图像的 Token |
| 微调 | 10 Epochs |
| 指标 | Token Match、TPR@固定 FPR、FID、p-value |
| 后处理基线 | CIN、MBRS、TrustMark、WAM |
| 攻击 | JPEG、噪声、模糊、亮度、翻转、旋转、裁剪、DiffPure、神经压缩 |

## 4. 与其他方法对比

相对后处理像素水印，该方法在自回归采样时改变 Token 概率，水印与图像语义共同生成，因此对扩散净化和神经压缩通常更稳定。

相对扩散模型 Generation-Time 水印，本文针对离散 Token 自回归模型，并继承 LLM 水印的二项检验与理论误报率。

相对同期的 INDEXMARK、C-Reweight、LBW 和 WIAR，本文最突出的特点是把 RCC 作为核心问题，并通过 Tokenizer 微调与几何同步系统解决重新 Token 化。

## 5. 实验表现

### 5.1 RCC 与无攻击检测

在 TAMING 上，BASE 的 Token Match 通常低于 0.8，所有微调版本多数超过 0.8。BASE 在 1% FPR 下已经接近 100% TPR，但微调版本在更严格的极低 FPR 下明显更强。

这说明 RCC 的价值不仅是让水印“能检测”，更是增加检测置信度，使 p-value 更低。

### 5.2 生成质量

TAMING 未加水印模型 FID 为 16.7。BASE、FT 和 FT+AUGS 均未超过 16.7；加入同步层后 FID 为 17.3。

因此 Token Bias 和 RCC 微调几乎不损害质量，主要质量代价来自局部同步水印。

### 5.3 像素值变换

平均 TPR@1% FPR：

| 模型 | BASE | FT | FT+AUGS | FT+AUGS+SYNC |
|---|---:|---:|---:|---:|
| TAMING | 0.26 | 0.45 | **0.92** | 0.83 |
| CHAMELEON | 0.50 | 0.53 | **0.89** | 0.76 |
| RAR-XL | 0.22 | 0.74 | **0.98** | 0.89 |

普通微调有所提升，但加入增强后才获得稳定的 JPEG、噪声、模糊和亮度鲁棒性。同步版本略有回落，原因是像素攻击偶尔使同步器误判几何变换。

### 5.4 几何变换

| 模型 | FT+AUGS | FT+AUGS+SYNC |
|---|---:|---:|
| TAMING | 0.01 | **0.82** |
| CHAMELEON | 0.02 | **0.64** |
| RAR-XL | 0.03 | **0.82** |

仅靠 RCC 微调无法解决空间排列变化；同步层是几何鲁棒性的主要来源。

### 5.5 扩散净化与神经压缩

| 模型 | 配置 | 扩散净化 | 神经压缩 |
|---|---|---:|---:|
| TAMING | BASE | 0.43 | 0.48 |
| TAMING | FT+AUGS | 0.70 | 0.79 |
| CHAMELEON | BASE | 0.80 | 0.82 |
| CHAMELEON | FT+AUGS | 0.82 | 0.88 |
| RAR-XL | BASE | 0.29 | 0.27 |
| RAR-XL | FT+AUGS | 0.79 | 0.95 |

RCC 微调没有直接针对所有压缩器或 DiffPure 训练，却仍显著提升鲁棒性，说明这些攻击和普通重编码共享“视觉语义保留、Token 表示扰动”的结构。

### 5.6 后处理水印比较

后处理水印在部分像素攻击上更强，但常存在一种明显短板：

- CIN、MBRS 几乎不抗几何变换；
- CIN、WAM 易被净化或神经压缩移除；
- TrustMark 较均衡，但检测依赖神经提取器，统计 p-value 可能受 Bit 偏置和相关性影响。

本文方法的优势不是每一列都最高，而是攻击覆盖更均衡，并提供理论可解释的误报控制。

### 5.7 文本—图像联合水印

CHAMELEON 同时生成文本和图像。随机破坏文本 Token 后：

- 文本单独检测随破坏比例快速下降；
- 干净图像加入联合检测后，最强文本破坏下 TPR 仍高于约 0.94；
- 图像仅受弱噪声时联合检测仍有优势；
- 图像受强噪声时，损坏的图像信号会稀释文本证据，联合检测反而更差。

因此实际系统需要评估各模态信号质量，而不是无条件合并。

### 5.8 音频扩展

论文在 MOSHI 的 MIMI 残差向量量化 Tokenizer 上验证该思路。原始音频 Token Match 为 0.36，500 Hz 高通后为 0.21，1.1× 变速后为 0.16。

音频版改动包括：

- 使用投影前 Soft Latent 作为 RCC 目标；
- 用多分辨率 STFT 损失替代 LPIPS；
- 增强包含高低通、带通、Gaussian/Pink Noise 和 1–10 ms 时频平移；
- 只给前四个 RVQ Stream 加水印。

12 秒音频结果：

| 方法 | 无攻击 | Valuemetric | 时频变化 | 神经压缩 | MOSNet |
|---|---:|---:|---:|---:|---:|
| BASE | 0.97 | 0.62 | 0.24 | 0.80 | 3.82 |
| FT | 0.99 | 0.64 | 0.14 | 0.84 | 3.83 |
| FT+AUGS | 0.99 | 0.80 | 0.24 | 0.86 | 3.73 |
| AudioSeal | 1.00 | 0.84 | 0.55 | 0.85 | 3.78 |

RCC 增强改善普通音频处理和神经压缩鲁棒性，但变速、时移等时频变化仍是开放问题。未加增强的 FT 甚至出现灾难性遗忘。

## 6. 学习与应用

论文最重要的启发是：Generation-Time Token 水印能否在现实内容上检测，不只取决于采样算法，还取决于 Tokenizer 的反向可恢复性。

RCC 概念可推广到自回归视频、神经音频 Codec、多模态统一 Token 模型，以及任何“生成 Latent → 可见内容 → 重新编码 Latent”的来源认证系统。

## 7. 总结

论文首次系统研究自回归图像生成的 Token 级水印。它把 KGW 绿名单采样用于图像 Token，并指出重新 Token 化造成的 RCC 不足是实际检测的核心瓶颈。通过微调解码器和检测编码器副本，方法提升像素值攻击下的 Token 保留率；通过局部同步水印，恢复翻转、旋转和部分裁剪后的 Token 排列。方案保持图像质量，并能抵抗净化和神经压缩，但依赖 VQ Tokenizer 内部权限，仍会被同步破坏与几何变换的组合攻击绕过。

## 9. 复现难度与适合人群

复现难度高。完整复现需要三种自回归图像模型、VQ Tokenizer 内部访问、ImageNet Token 预计算、LPIPS 与增强训练、局部同步模型、DiffPure 和多种神经压缩器，以及严格的去重与二项 p-value 实现。

适合研究自回归视觉生成、VQ Tokenizer、多模态水印、内容来源认证和鲁棒统计检测的读者。

## 10. 简短全面总结

论文把 LLM 绿名单 Token 水印迁移到自回归图像模型，并发现真正的障碍不是采样，而是图像重新 Token 化后大量 Token 改变。作者用 RCC 微调保持 Token 身份，用局部同步层逆转几何变化，在 TAMING、CHAMELEON 和 RAR-XL 上实现较强检测与鲁棒性，同时保留理论 p-value。同步层和音频时频鲁棒性仍不成熟。

## 11. 论文写作逻辑分析

论文先提出“能否直接把 LLM 水印用于自回归图像 Token”，随后用 Token Match 揭示 RCC 是失败原因；方法严格对应两类误差：RCC 微调处理像素值变化，同步层处理空间排列变化；实验再依次验证 RCC、检测功效、生成质量、攻击鲁棒性和多模态扩展。

最值得借鉴的是其中间变量证据链：

$$
\text{重新 Token 化误差}
\rightarrow
\text{RCC 训练目标}
\rightarrow
\text{Token Match}
\rightarrow
\text{统计检测能力}.
$$

相对薄弱之处是同步层来自现成后处理水印，未与 RCC 统一训练；音频部分也仍是概念验证。

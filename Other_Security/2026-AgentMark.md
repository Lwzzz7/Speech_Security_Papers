# AgentMark: Utility-Preserving Behavioral Watermarking for Agents

## 0. 摘要翻译

大型语言模型智能体已被用于多步工具调用与自主决策，因而需要知识产权保护与可审计溯源。现有内容水印主要标识模型生成的文本，无法直接归因决定工具与子目标的高层规划行为；而在规划层嵌入水印又有两个困难：微小的决策分布偏移会在长轨迹中累积并损害效用，且许多智能体只能通过黑盒 API 干预。本文提出 AgentMark：它先让智能体显式输出候选行为的概率分布，再在该分布上进行保持边缘分布的条件采样，以在规划决策中编码多比特身份标识。该方案可用于黑盒 API，并能与动作内容层水印结合。具身、工具调用和社会仿真实验显示，它能够提供多比特容量、从部分日志恢复水印，并保持任务效用。

## 1. 方法动机

### a) 要保护的对象不是最终文本，而是“规划行为”

一条 agent 轨迹同时包含两层决策：规划行为决定下一步是搜索、比较、调用哪个工具或采取哪个子目标；执行动作才给出具体参数、自然语言或 API 调用。后者可用普通文本水印处理，但前者才是多步能力、策略资产和审计责任的直接载体。若只给 action 字符串打标，工具参数被编辑、日志被摘要或内容被重写后，仍难说明“该高层策略来自谁”。

### b) 现有行为水印的痛点

最直接的方案是按密钥划分绿色/红色行为集，并给绿色行为加 logit bias。它能让绿色行为频率升高，却有三个问题：第一，偏置改变 $$P_t$$；在长轨迹中这一小偏移会改变后续状态、放大为任务失败。第二，它通常只产生“是否带标”的零比特统计证据，不能恢复多用户 ID。第三，固定顺序传输的比特流在日志丢步、截断时很脆弱。

### c) 核心假设与 insight

论文的假设是：在每步有限候选行为 $$B_t$$ 上，agent 的隐式策略可被提示为显式概率表 $$P_t$$。核心 insight 是把水印放进“**由密钥决定的条件采样相关性**”，而不是放进可见的概率偏置：对外边缘分布仍为 $$P_t$$，但持钥验证者能由行为与同一随机源还原编码比特。再用 RLNC 把 payload 变成可无序累积的线性方程，部分日志仍可解码。

## 2. 威胁模型解读

| 维度 | 论文设定 | 对方法的含义 |
|---|---|---|
| 防御者 | agent 提供方/所有者，持有共享密钥 $$K_{sh}$$、payload 和审计逻辑 | 能控制规划采样，但无需改底层 LLM 权重 |
| 被保护系统 | 先选规划行为 $$b_t\in B_t$$，再据此生成执行动作 $$a_t$$ 的多步 agent | 水印位于两层之间，不直接改动作内容 |
| 验证者可见性 | 得到部分 $$\hat b_t,P_t$$ 及可重建 context 的规划日志 | 需要概率表和上下文；只剩最终回答时无法直接验证 |
| 攻击目标 | 删除、截断或过滤规划记录；编辑 action 表面形式；让所有者无法归属 | 论文将主要可量化影响抽象为 step erasure / truncation |
| 攻击能力 | 不能获知 $$K_{sh}$$，但可造成缺失步骤或保留不完整日志 | 密钥派生随机性和 RLNC 分别抵抗伪造与擦除 |
| 不覆盖的能力 | 完全获得密钥、伪造全部概率表、删除所有规划日志、改写底层 agent 分布 | 这些会破坏验证前提，论文没有声称解决 |

形式化地，原始与水印轨迹分别为 $$\tau,\hat\tau$$。论文要求任务成功率与轨迹长度的期望差不超过 $$\epsilon_{succ},\epsilon_{len}$$；对可见索引集 $$I$$，在 $$|I|\ge(1-\rho)T$$ 时，以至少 $$1-\delta$$ 的概率恢复 $$m$$。这是一种**日志审计威胁模型**，不是从任意最终文本反推出 agent 来源的通用检测器。

## 3. 方法设计与复现级理解

### a) 全局数据流

训练/部署时无需训练新 agent：每轮读取观测 $$o_t$$、历史 $$h_t$$ 和候选行为集 $$B_t$$，先提示 LLM 输出满足和为 1 的概率表 $$P_t\approx\pi_\theta(\cdot\mid o_t,h_t)$$。随后由 $$K_{sh}$$ 与可复现 context 生成随机流，对 $$P_t$$ 做分布保持编码，得到 $$\hat b_t$$。原 action generator 在 $$\hat b_t$$ 条件下生成 $$a_t$$。审计侧重复同样的重组和随机过程，从留下的行为取回方程，最后解线性系统得到部署级 ID。

### b) 行为—动作解耦与显式分布

这一步解决“如何在黑盒 API 上干预”。普通 agent 的 $$P_t^*$$ 是隐藏的；AgentMark 通过 prompt 要求模型针对**所有**候选行为输出 JSON 概率列表，以 $$P_t$$ 作为可操作近似。论文不修改 action token 的 logits，也不要求访问模型参数。代价是候选集合必须有限、可枚举，概率表必须被可靠记录；候选行为遗漏或概率未校准都会传递到编码和验证。

### c) 差分重组：把任意分类分布拆成均匀 bin

将行为按 $$p_1\ge\cdots\ge p_n$$ 排序，令 $$d_k=p_k-p_{k+1},p_{n+1}=0$$。第 $$k$$ 个 bin 包含概率最高的 $$k$$ 个行为，混合权重为 $$q_k=k d_k$$。双方用同一 PRG 从 $$\mathrm{Cat}(q_1,\ldots,q_n)$$ 选 bin，再在该 bin 内均匀编码选择位置。对第 $$i$$ 个行为：

$$
\Pr[\hat b_t=b_{t,i}]=\sum_{k=i}^{n}q_k\frac1k
=\sum_{k=i}^{n}(p_k-p_{k+1})=p_i.
$$

因此编码改变的是“给定 bin 内选哪个位置”，而非总体概率。这是任务效用主张的数学支点；它保持的是每一步给定 $$P_t$$ 的边缘分布，并不自动证明有限温度 LLM 输出的 $$P_t$$ 等于未加水印时的真实隐式分布。

### d) 密钥同步与 CyclicShift 编码

每步种子为 $$K_t=H(K_{sh}\Vert\mathrm{Context}_t)$$，context 可包含步号、观测与历史。编码/解码双方因而同步得到 bin 和随机掩码，外部观察者则不能将行为位置解释为固定比特。对于大小 $$|T|$$ 的均匀 bin，CyclicShift 写入可变长度 $$c_t\in\{\lfloor\log_2|T|\rfloor,\lfloor\log_2|T|\rfloor+1\}$$ 的字符串；其期望容量位于 $$[\log_2|T|-0.0861,\log_2|T|]$$。候选空间小或高概率集中时，容量自然低，这是该方法的内在约束而非可随意调大的超参数。

### e) RLNC 擦除恢复

不把 $$m$$ 顺序切片，而是用 $$K_t$$ 为每个输出 bit 生成系数 $$a_{t,j}\in\mathbb F_2^L$$，实际嵌入 $$y_{t,j}=\langle a_{t,j},m\rangle$$。所有可见步骤汇成 $$A_I m=y_I$$，仅当 $$\mathrm{rank}(A_I)=L$$ 时唯一恢复。若收集到 $$r\ge L$$ 个伪随机方程，满秩概率下界为 $$1-2^{-(r-L)}$$。这解释了为什么随机删除步骤只减少方程预算、不会像重复码那样破坏 bit 边界。

### f) 复现配置表

| 项目 | 论文明确配置 |
|---|---|
| 基座 | DeepSeek-Chat；ALFWorld 温度 1.0，ToolBench 温度 0.7 |
| 环境 | ALFWorld ID/OOD、ToolBench 六子集、OASIS 的 Twitter-like / Reddit-like |
| ALFWorld | 140 个 ID、134 个 OOD 任务，各 3 次 |
| ToolBench | 共 450 个任务，每轨迹上限 10 步；ToolBench payload 为 8-bit RLNC |
| RG 基线 | $$\gamma=0.5$$、$$\delta=2.0$$ |
| 关键日志 | 每步候选集、概率表、所选行为、可重建 context；论文未给出统一概率校准步骤 |
| 擦除实验 | ToolBench 上随机丢步；比较 RLNC 与无辅助索引的 repetition baseline |

复现时应把“让模型输出概率表”的 prompt、候选行为排序的 tie-break 规则、context 序列化格式、哈希/PRG 实现固定下来；这些任一不一致都会让验证侧选到不同 bin。

## 4. 与其他方法对比

| 方法 | 水印位置 | 信号 | 多比特 | 对效用的主要风险 | 缺日志时 |
|---|---|---|---|---|---|
| 文本内容水印 | action 内容/token | token 统计或语义标记 | 取决于方案 | 不直接归因规划策略 | action 改写易影响 |
| Red–Green | 规划行为采样 | 提高绿色行为频率 | 否 | 改变 $$P_t$$，可累积漂移 | 只能做统计检测 |
| AgentMark-F | 规划行为采样 | 密钥条件相关、边缘保持 | 是 | 依赖显式 $$P_t$$ 近似质量 | RLNC 聚合方程 |

它与 SynthID-Text 一类内容层方案是互补而不是替代：前者能证明行为选择链，后者能证明输出内容链。实际系统若二者同时部署，需要分别定义日志保留与证据阈值。

## 5. 实验表现

### a) 效用与容量

动机是证明“分布保持不是空泛理论”。作者在同一 agent 执行管线中只替换行为选择方式：Baseline 直接 ReAct，RG 加偏置，AgentMark-F 用概率表采样；报告任务成功、轨迹长度和每步/每任务 bit。ALFWorld 同时覆盖 ID/OOD，ToolBench 覆盖单工具与多工具，因而测试了决策而非单一固定轨迹。

### b) 与 RG 的证据

ToolBench 中 RG 的 Green% 为 $$67.0\pm0.9\%$$（ALFWorld ID/OOD 分别为 $$71.9\pm13.6\%$$、$$71.5\pm15.2\%$$），高于 $$\gamma=0.5$$，说明偏置确实留下检测痕迹；但它是零比特统计量。AgentMark-F 的目标不同：报告显式 bits/step、bits/task 和 payload 恢复，同时观察任务效用是否与 Baseline 保持接近。

### c) 擦除、截断与假阳性

随机丢步实验将每步所有 packet 一起删除，分别考察单轨迹解码和跨轨迹全局聚合。RLNC 的曲线应随接收方程总数而变化；repetition 在“无序、无索引”的严格盲解码设定下需要完整连续段。128-bit 一致性检查的蒙特卡洛结果显示，额外方程数从 0 到 8 时，无水印/错误 key 假阳性从约 60.6% 降至 0.40%/0.50%，额外 14 个或以上时 1000 次试验未见假阳性。这个结果支持的是指定 verifier 的一致性检验，不等价于所有攻击者下的安全证明。

### d) 社交与语义改写

OASIS 中每个平台收集 100 条轨迹；Twitter-like 行为包括发帖、点赞、转发、关注等，Reddit-like 还包含评论、投票、搜索、刷新、静音。语义改写压力测试在 ALFWorld-OOD 的 134 任务、2326 步上，用 LLM 改写 Current Situation，并用 teacher forcing 避免轨迹漂移混入结果。该实验检验的是验证 context 的表层鲁棒性，不是对任意恶意语义重构的保证。

## 6. 学习与应用

代码公开。最小原型可先选 4–8 个离散工具类别，以固定 JSON schema 取得概率表，实现差分重组和 GF(2) 求秩；先在离线 replay 日志上验证“同 key 可解、错 key 不可解”，再接入真实工具。部署时应保存不可篡改的规划日志或至少保存 context digest 与概率表；若只保留最终动作，AgentMark 的主要证据已经丢失。

## 7. 总结

一句话：**以保持分布的行为采样和 RLNC 为 agent 规划轨迹嵌入多比特归属。**

## 9. 复现难度与适合人群

复现难度为**中高**：GF(2) 编码本身简单，但难点是稳定获得可校验概率表、固定候选集与真实 agent 环境。主要依赖是可调用 LLM、工具环境、审计日志和公开代码；不需要训练大模型。适合做 agent 安全、溯源、工具调用评测的研究者。最小版本建议跳过 OASIS，先做静态工具候选集和离线 erasure 曲线。

## 10. 简短全面总结

AgentMark 面向的是 agent 的高层规划行为，而不是最终文本。它指出红绿偏置会改变决策分布并在多步任务中积累误差，于是先让黑盒 LLM 显式给出候选行为概率表，再通过差分重组、均匀 bin 内 CyclicShift 和按 context 派生的密钥随机性编码身份；边缘行为概率保持为原 $$P_t$$。针对平台日志缺失，论文用 RLNC 将 payload 变为跨步线性方程，验证者可聚合残余记录求秩恢复。实验覆盖 ALFWorld、ToolBench 与 OASIS，并额外测试擦除、截断、错误 key 和语义改写。其贡献是把“归属信号”由可见概率偏置转为分布保持的密钥相关性；关键边界则是验证强依赖规划概率表、候选集与可重建 context，完全失去规划日志时无法工作。

## 11. 论文写作逻辑分析

论文先区分 planning behavior 与 execution action，继而用“偏置会累积成效用损失”和“日志会缺失”推出两个设计约束。方法模块逐一响应：显式分布让黑盒可操作，差分重组保证边缘不变，CyclicShift 提供容量，RLNC 提供抗擦除。实验也对应这条证据链：任务/长度验证效用，bits 与 RG 验证归属能力，擦除与 FPR 验证审计可用性，OASIS 和改写验证环境迁移。写作上最应警惕的跳跃是将 $$P_t$$ 当作 $$P_t^*$$ 的可靠替身；若未来工作能测量概率 elicitation 的校准误差，论证会更闭环。

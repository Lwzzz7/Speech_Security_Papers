# AgentMark: Utility-Preserving Behavioral Watermarking for Agents

## 0. 摘要翻译

本文研究智能体模型的行为级水印：在不改变任务效用的前提下，将多比特身份标识嵌入智能体每一步的规划决策。AgentMark 显式获得候选行为的概率分布，并以密钥控制、边缘分布保持的条件采样选择行为；由此，单步行为的总体分布与原策略一致，而跨步选择序列携带可解码信息。方法能服务黑盒 API 智能体，也可与动作内容层水印并存。作者在具身任务、工具调用和社交仿真中验证了多比特承载、部分日志下解码与任务效用保持。

## 1. 论文基本信息

- 标题：AgentMark: Utility-Preserving Behavioral Watermarking for Agents

- 年份：2026
- 研究对象：执行规划—动作循环的 LLM agent，而非生成文本或最终动作字符串本身。
- 代码：<https://github.com/Tooooa/AgentMark>

## 2. 研究背景与动机

智能体的价值往往来自策略、提示和运行轨迹，但传统文本水印不适合直接迁移：动作是否可执行、任务成败和多步状态会使微小的分布偏差累积。简单地对“绿色动作集合”施加 logit 偏置虽能做零比特检测，却会系统性改变行为分布，既不利于高质量决策，也无法恢复明确的拥有者 ID。

作者将水印位置上移到**规划行为**：先从有限候选行为中选择“下一步做什么”，再依据该行为生成具体执行动作。这个层次具有较小、可枚举的选择空间，且可在不修改执行器的前提下编码身份信息。

## 3. 问题定义与威胁模型

对第 \(t\) 步，智能体的原始规划分布为 \(P_t^*\)，候选集合为 \(B_t\)，原始行为为 \(b_t\)。嵌入器根据消息 \(m\)、共享密钥和当前上下文输出 \(\hat b_t\)，解码器从日志中的行为及可重建分布恢复 \(m\)。目标同时满足：

1. 效用保持：水印前后任务成功率、轨迹长度不应显著恶化；
2. 分布保持：对任一行为，水印采样的边缘分布应与 \(P_t\) 一致；
3. 鲁棒可解码：日志可发生步骤丢失、截断或局部过滤；
4. 安全性：错误密钥或无水印日志不应通过验证。

攻击者/审计环境可只保留部分规划记录，或对观测文本做语义不变改写；验证者持有密钥并可由上下文重建每一步的随机性。该设定不保证攻击者无法彻底删除全部规划日志，而是针对现实审计中不完整记录的可验证性。

## 4. 核心洞见

关键不是让每一步“更偏好某一类行为”，而是让由密钥决定的条件选择在消息上相关、在边缘上仍服从原分布。如此水印信号存在于**选择与密钥的相关性**，而非可被统计检测到的全局策略偏移。为应对缺包，跨步信息应编码为可累积的线性观测，而不是固定顺序的重复比特串。

## 5. 方法概述

AgentMark-F 的流程为：

1. **分布显式化**：提示智能体为全部可选规划行为给出概率对象 \(P_t\)，近似其潜在决策分布；
2. **可复现随机性**：由 \(K_t=H(K_{sh}\Vert\mathrm{Context}_t)\) 生成每步密钥随机量，验证者从日志上下文可重建；
3. **分布保持条件采样**：把待传输的编码比特与随机量耦合，在满足消息约束的子事件中采样 \(\hat b_t\)，使总体仍满足 \(\hat b_t\sim P_t\)；
4. **无速率线性网络编码（RLNC）**：每步产生的有效信息形成关于部署级 payload 的 GF(2) 线性方程；审计时汇总幸存方程，满秩后恢复消息；
5. **一致性验证**：对正确密钥，方程组应一致；对错误密钥/无水印日志，随着冗余方程增加，通过率快速下降。

## 6. 关键技术细节

- 规划层与执行层分离：只改“选哪种行为”，随后仍由原 agent 生成参数化动作，降低对工具协议和动作内容的侵入。
- 概率表是方法条件：若候选集遗漏高概率行为或 LLM 概率校准很差，分布保持的理论收益会弱化。
- RLNC 使任意幸存步骤都可能贡献独立方程；因此更适合日志缺失，而非要求保留连续 \(k\) 位。
- 当使用 128-bit payload 的一致性检查时，论文的蒙特卡洛结果显示：额外 8 个方程后无水印/错误密钥的假阳性均低于 1%，额外 14 个及以上方程时 1000 次试验中未出现假阳性。

## 7. 实验设置

- 基座模型：DeepSeek-Chat；ALFWorld 温度 1.0，ToolBench 温度 0.7。
- 环境：ALFWorld 的 ID/OOD 家务任务、ToolBench 六类单/多工具指令，以及 OASIS 的 Twitter-like、Reddit-like 社交环境。
- ALFWorld 评测 140 个 ID 与 134 个 OOD 任务，各三次；ToolBench 总计 450 个任务、每条轨迹最多 10 步。
- 对比：无水印 ReAct 基线，以及对伪随机绿色行为集合施加 logit bias 的 RG 基线。
- 鲁棒性：随机步骤擦除、轨迹截断、跨轨迹汇集解码、错误密钥和语义保持的观测重写。

## 8. 实验结果与分析

- AgentMark-F 可在上述三类环境中传输显式多比特标识；ToolBench 中报告以 8-bit RLNC 编码 payload 统计 bits/step 与 bits/task。
- RG 的绿色行为比例可显著偏高（如 ToolBench 为 \(67.0\pm0.9\%\)），说明它具有检测信号，但这是零比特偏置，不能解出所有者 ID，并会改变决策分布。
- 对缺失步骤，RLNC 的解码依赖收集到的总方程数；全局汇集多个片段通常优于逐轨迹的保守解码。相对地，盲重复编码需要幸存连续且对齐的完整 bit 段。
- 在语义保持的观测改写实验中，作者用 teacher forcing 隔离文本表面变化的影响；这提醒部署中要定义可重建的上下文规范。

## 9. 优点与局限

优点：水印直接服务于可审计的多比特归属；边缘分布保持是对智能体效用风险的针对性设计；RLNC 对日志不完整更自然；适用黑盒 agent。

局限：依赖模型输出明确且可信的行为概率；需要审计侧保存（或重建）候选集、上下文和规划日志；若攻击者完全移除规划层记录，无法仅从最终自然语言/工具输出恢复行为层水印；分布保持是边缘意义，有限样本、上下文漂移和概率校准仍可能影响效用。

## 10. 可复现性与使用建议

建议先在动作空间小且可验证的工具 agent 上复现：固定候选行为集合、记录上下文哈希和概率表，并将 key 轮换、日志访问控制与身份注册系统结合。评测至少同时报告成功率、轨迹长度、分布距离、payload 成功率、错误密钥假阳性及不同日志缺失率下的曲线。不要把“绿色比例显著”当作多用户归属能力。

## 11. 论文写作逻辑分析

论文遵循“传统 token/动作水印为何失效 → 智能体的规划层为何更合适 → 如何既编码又不推移策略分布 → 不完整日志如何恢复”的链条。方法与威胁模型紧密对应：分布保持处理效用，密钥条件采样处理归属，RLNC 处理擦除，一致性检查处理错误密钥。其最有价值的概念转变是把 agent 水印定义为**行为分布上的受控相关性**，而非可见的动作模式。

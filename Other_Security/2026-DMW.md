# A Distortion-minimization Watermarking Framework for Large Language Models: Larger Capacity, Stronger Robustness and Higher Quality

## 0. 摘要翻译

大语言模型水印为生成文本提供可验证的来源识别，而实际部署需要同时具备大水印容量、对攻击的强鲁棒性以及高文本质量。然而，现有方法通常难以平衡这些指标，往往用彼此分离的设计分别处理。为此，论文提出 distortion-minimization watermarking（DMW）框架，在一个统一优化范式中同时建模容量、鲁棒性和质量。该框架将鲁棒性与质量建模为文本修改的失真代价，在给定水印长度下最小化总代价，从而获得最优折中。具体而言，作者设计了若干失真代价：利用语义不变性抵抗攻击的鲁棒性代价，以及两个把修改引导到低衔接、高变化区域以降低感知影响的质量代价。随后提出 periodically optimized syndrome-trellis codes（PO-STCs），将整体失真最小化表述为周期性最短路径问题，使顺序生成过程能够实时优化并灵活控制容量。大量跨数据集和 LLM 的实验表明，DMW 在所有指标上优于最先进方法。尤其在强 paraphrasing 攻击下，DMW 的 match rate 相比最佳基线最高提升 46.35%，同时保持更好的文本质量。

## 1. 方法动机

LLM 文本水印的实际目标不是只判断“是不是机器生成”，而是要能嵌入可追踪 payload，例如用户 ID、来源编号或审计信息。因此 multi-bit watermarking 比 zero-bit watermarking 更适合追责场景，但它也更难：payload 越长，需要约束或修改的文本位置越多，水印越容易影响流畅性、自然度和抗改写能力。

现有方法的问题在于三项指标通常分开优化。zero-bit 方法容量不足；message enumeration 方法能支持 multi-bit，但提取复杂度随消息位数指数增长；bit assignment 方法提取快，但自然语言 token 频率不均会导致 bit 分配不均、攻击后准确率下降。一些方法重视质量但在 paraphrasing 下脆弱，另一些方法重视鲁棒性却损害文本质量。论文认为，与其追求“distortion-free”这种在理论和实践上都不稳定的目标，不如承认水印必然带来代价，并把问题明确变成“在给定 payload 下，把最不该改的地方少改，把最适合改的地方优先改”。

DMW 的核心直觉来自 distortion-minimization steganography：对宿主内容的每个可修改单元分配失真代价，再用编码算法在满足消息约束的同时最小化总代价。迁移到 LLM 文本后，难点是文本是顺序生成的，最终长度事先未知，传统 STC 要求所有 carrier 和 cost 预先给定，不能直接用于 streaming generation。DMW 因此把文本按句子/片段周期性处理，用语义投影把句子变成 bit，用鲁棒性与质量代价衡量句子修改风险，再用 PO-STC 在局部窗口内做近似全局优化。

## 2. 威胁模型解读

| 项目 | 论文设定 |
|---|---|
| 防御者/水印嵌入方 | LLM 服务提供者或文本生成方，持有 secret key，将 multi-bit watermark message 嵌入生成文本。 |
| 验证者 | 持有同一 key 和 PO-STC 参数，通过语义投影和校验矩阵从文本中提取水印消息。 |
| 攻击者 | 可对水印文本做语义保持编辑，包括 word insertion、synonym substitution、document-level paraphrasing，以及附录中的 semantic-projection adaptive attack。 |
| 攻击目标 | 破坏 message extraction，使 match rate 或 bit accuracy 下降；更强攻击尝试改变句子到 bit 的 semantic projection。 |
| 攻击约束 | 主实验假设攻击保持文本语义，插入/替换随机修改 10% words，paraphrasing 使用 GPT-3.5-turbo 整段重写。 |
| 攻击知识 | 主实验为通用语义保持攻击；附录 F 考察自适应攻击者知道 sentence-to-bit mapping，或同时知道 mapping 和 margin。 |
| 防御目标 | 在 payload 8/16/24/32 bit 下提高完整消息恢复率、bit accuracy、抗语义攻击能力，同时保持低 perplexity、高 Sem-Ent 和高 fidelity。 |
| 现实边界 | 论文未覆盖真实平台上的人类协同深度编辑、翻译-回译、多轮摘要、恶意删除句子、prompt-level evasion 或 verifier key 泄露。 |

该 threat model 贴近“水印文本被轻度或中度改写后仍要追踪”的场景。它的强点是专门测试了语义保持攻击，尤其 document-level paraphrasing 比逐句 paraphrase 更强，会跨句重排和全局改写。边界也明显：DMW 把水印绑定到句子语义 cluster，如果攻击者能大幅改变句子语义空间位置而仍保持整体语义，或者直接删除/重组大量句子，水印会更脆弱。附录 F 已经显示：当攻击者知道 mapping 与 margin 并专门改写边界句子时，match rate 明显下降。

## 3. 方法设计与复现级理解

### 3.1 问题形式化与分段生成

传统 in-generation watermarking 在每个时间步只看局部上下文，形式上为：

$$
S_t=\mathrm{Emb}(M,K,P,S_{0:t-1},m),\quad t=0,1,\ldots,T.
$$

提取为：

$$
m=\mathrm{Ext}(K,S_{0:T}).
$$

这种逐 token 决策无法在整段文本上做全局优化。post-hoc 方法虽然能看到全文，但不是所有 token 都适合改，且容易损害质量。DMW 采用中间路线：把生成过程切成长度为 $s$ 的 token segment，每个 segment 内综合考虑多个 token 或句子的特征，再做水印嵌入：

$$
S_{si:si+s-1}=\mathrm{Emb}(M,K,P,S_{0:si-1},m),\quad i=0,1,\ldots,N.
$$

论文随后将第 $i$ 个 clean segment 记为 $X_i$，watermarked segment 记为 $Y_i$。若 $X_{i,j}\neq Y_{i,j}$，表示该位置被修改。整体失真为：

$$
D(X,Y)=
\sum_{i=0}^{N-1}D(X_i,Y_i)
=
\sum_{i=0}^{N-1}\sum_{j=0}^{s-1}
\rho_{i,j}\cdot [X_{i,j}\neq Y_{i,j}].
$$

这里 $\rho_{i,j}>0$ 是修改该位置的代价。这个式子是 DMW 的核心抽象：所有鲁棒性和质量考虑最终都折算为“改这个位置贵不贵”。

### 3.2 总体流程：嵌入与提取

DMW 的嵌入阶段先由 LLM 根据 prompt 生成文本，再把文本切分为句子。每个句子经过 semantic projection，得到固定长度 bit string；多个句子的 bit string 拼接成 clean bit vector。与此同时，每个句子计算 distortion cost，同一句子投影出的多个 bit 共享同一个 cost，这些 cost 拼接成与 bit vector 等长的 cost vector。

随后，嵌入方使用 secret key 构造 PO-STC 的 parity-check matrix，把 clean bit vector、cost vector 和 watermark message 输入 PO-STC Encoder，得到 watermarked bit vector。这个 bit vector 还不是自然语言，必须映射回句子：DMW 让 LLM 为目标句子生成候选改写，选择语义投影后恰好等于 watermarked bit string 的候选作为 watermarked sentence，最后拼回完整水印文本。

提取阶段不需要 LLM 重新生成候选。验证者把待检测文本切句，对每个句子做同样 semantic projection，拼接出 watermarked bit vector；用 secret key 重建同一个 PO-STC parity-check matrix，通过 syndrome 计算提取消息，再与注册信息比较。

### 3.3 Semantic Projection：把句子语义稳定映射到 bit

该模块解决两个问题：自然语言句子不是二进制 carrier；同时，word insertion、synonym substitution、paraphrasing 等攻击通常保持语义，因此相似句子应映射到同一个 bit string。

DMW 使用 sentence encoder $f$ 将句子 $X_i$ 编码为 $d$ 维语义向量。论文采用 SBERT 架构，并在附录 E 中说明用 WikiText-2 约 200 万句子、contrastive learning、AdamW 训练 5 epochs，预训练模型为 `bert-base-uncased`，输出 768 维特征。之后用 k-means 将语义空间划分为 $k$ 个 cluster，得到中心 $\{\mu_0,\mu_1,\ldots,\mu_{k-1}\}$。每个 cluster index 被分配一个长度为 $\lceil \log_2 k\rceil$ 的 binary string。

给定句子 $X_i$，先找最近 cluster：

$$
c=\arg\min_{i\in\{0,1,\ldots,k-1\}}
\mathrm{dist}(f(X_i),\mu_i),
$$

其中 $\mathrm{dist}$ 是 cosine distance。为了避免靠近边界的句子在 paraphrase 后跳到其他 cluster，DMW 只使用离中心足够近、离其他中心有 margin 的句子：

$$
\min_{j\neq c}\mathrm{dist}(f(X_i),\mu_j)
-
\mathrm{dist}(f(X_i),\mu_c)
>\gamma.
$$

满足条件后，semantic projection 输出：

$$
P(X_i)=\mathrm{bin}(c).
$$

这一步与 SemStamp 类零比特方法的区别在于，后者主要把 cluster 分成 valid/blocked，相当于语义版 green/red list；DMW 直接把 cluster 映射为多位 bit string，因此能够承载 multi-bit payload。

### 3.4 Robustness Cost：优先修改语义中心附近句子

鲁棒性代价的出发点是：水印提取依赖句子的语义 cluster，如果一句话靠近 cluster 中心，那么同义改写后仍留在原 cluster 的概率更高；如果靠近边界，轻微改写就可能跨 cluster，导致 bit 改变。

因此第 $i$ 个句子的 robustness cost 定义为它到最近 cluster center 的距离：

$$
\rho_{rob}(X_i)=\mathrm{dist}(f(X_i),\mu_c).
$$

距离越小，说明句子越语义稳定，修改它更不容易在攻击后丢失 bit，因此 cost 越低，PO-STC 更倾向选它承载水印。

论文用两个实验解释这个 cost 的合理性。图 3 随机选一个 cluster，比较中心附近 20 个句子和边界附近 20 个句子，分别 paraphrase 三次。结果显示 paraphrase 后句子仍靠近原句，但中心附近句子大多保持在原 cluster，边界句子约一半会跳到其他 cluster。图 4 进一步测试把句子修改到新 cluster 后的分布，发现修改后的句子在新 cluster 内分布较随机，不会因为总是选中心句子而产生明显语义空间偏置。

### 3.5 Quality Cost：用流畅性和多样性控制文本质量

DMW 把文本质量拆成 fluency 和 diversity 两个维度。Fluency cost 衡量当前句子与前文衔接是否自然。若一个句子本来与上下文衔接弱，修改它对整体流畅性伤害较小，甚至可能改善文本；若一个句子与前文高度连贯，则不应轻易改。论文用一个单独 LLM $M_\rho$ 计算滑动窗口 perplexity，默认使用 OPT-2.7B：

$$
\rho_{flu}(X_i)=
\frac{1}{\mathrm{PPL}(M_\rho,S_{i-r+1:i})+\epsilon}.
$$

PPL 越高，说明当前窗口越不流畅，$\rho_{flu}$ 越小，越适合修改。滑动窗口大小 $r$ 用来稳定计算，默认 $r=8$。

Diversity cost 衡量当前句子是否有足够多的语义等价表达空间。DMW 在给定上下文下生成 $L$ 个候选句子，把它们分别 semantic projection，计算 bit string 分布熵 $H(X_i)$。若候选句子分布越分散，说明该位置表达灵活，改写不容易造成模式化；若熵为 0，说明所有候选都落到同一语义类别，表达刚性强，不适合强行改。

$$
\rho_{div}(X_i)=
\begin{cases}
1-\dfrac{H(X_i)}{\log_2(L)}, & H(X_i)>0,\\
\infty, & H(X_i)=0.
\end{cases}
$$

论文设置 $L=k$，使熵归一化到 $[0,1]$。当 $H(X_i)=0$ 时 cost 设为无穷大，是一个硬性避免策略：不要在表达空间极窄的句子上嵌水印。

最终整体代价为三项加权和：

$$
\rho_i=
\lambda_{rob}\rho_{rob}(X_i)
+
\lambda_{flu}\rho_{flu}(X_i)
+
\lambda_{div}\rho_{div}(X_i).
$$

默认权重为 $\lambda_{rob}=2.5,\lambda_{flu}=1.5,\lambda_{div}=1.0$。论文强调该框架可按应用调参，例如更重视鲁棒性时增大 robustness cost 权重，更重视自然度时加强 fluency/diversity cost。

### 3.6 PO-STC：把最小失真嵌入转成周期性最短路

DMW 使用二元线性码把消息嵌入 bit vector。设 $P(Y_i)$ 是第 $i$ 个 watermarked segment 的 bit string，$H_i$ 是校验矩阵 $H$ 中对应 segment 的子矩阵，$m_i$ 是要嵌入的 message segment。嵌入目标是：

$$
\mathrm{Emb}(X_i,m_i)=
\arg\min_{P(Y_i)\in C(m_i)}D(X_i,Y_i),
$$

其中：

$$
C(m_i)=\{Z_i\in\{0,1\}^{v}\mid H_i Z_i=m_i\}.
$$

提取则通过 syndrome 完成：

$$
\mathrm{Ext}(Y)=H P(Y).
$$

PO-STC 与传统 STC 的关系可以理解为“反过来的纠错码”。传统编码是消息 $m$ 进 encoder 得到 codeword $y$，decoder 用 syndrome 检错；水印中则是从 clean bit vector $x$ 出发，找一个尽量接近 $x$、但满足 $Hy^T=m$ 的 watermarked bit vector $y$。这样 syndrome 本身就是要提取的水印消息。

矩阵 $H$ 由多个高度为 $h$、宽度为 $w$ 的稀疏子矩阵 $\hat H$ 横向排列，每个 block 向下错位一行，形成 banded structure。$h$ 控制编码性能和复杂度，论文默认主实验 $h=6$，附录分析中 $h=12$ 可获得更强鲁棒性。$w$ 由 relative payload $\alpha=m/n$ 决定，取 $\lfloor n/m\rfloor$ 或 $\lceil n/m\rceil$。

在 trellis 图中，每列节点表示 $2^h$ 个状态。相邻列之间每个状态有两条边：horizontal edge 表示当前输出 bit $y_l=0$，diagonal edge 表示 $y_l=1$ 并执行状态异或转移。边的代价是：

$$
\rho_l\cdot [x_l\neq y_l].
$$

因此 Viterbi forward pass 会在所有满足 syndrome 约束的路径中保留累计代价最小路径，backward pass 还原路径对应的 $y$。

### 3.7 PO-STC 的流式改造

标准 STC 需要全文 bit vector 和 cost vector 预先给定，而 LLM 输出是 streaming 的。PO-STC 的解决方案是周期性分段优化。

第一，先根据预计文本长度或 prompt 约束估计 bit vector 长度 $n$，结合 watermark length $m$ 确定子矩阵宽度 $w$。第二，把 trellis 按 block 分成多个 segment；trellis 结构只依赖 $w$ 和 $h$，可提前构建。第三，生成过程中维护 buffer，积累 semantic projection 产生的 bit 和对应 cost，直到达到 segment 长度。第四，在该 segment 内运行 Viterbi forward pass，得到最优 sub-path。第五，把 sub-path 映射回对应 watermarked text segment 并输出。第六，把该 segment 的最优状态转移到下一 segment，重复直到消息嵌完。

这种做法引入一段延迟，但 segment 可以设置较小，因此近似实时。复杂度上，标准 STC 时间和空间复杂度均为：

$$
O(2^h n).
$$

PO-STC 时间复杂度仍为 $O(2^h n)$，但只需存储一个 segment 的状态，空间复杂度降为：

$$
O(2^h v).
$$

附录 G 用仿真实验证明分段带来的 coding loss 有限。在 $\alpha=0.4$ 且最小 segment ratio 条件下，总失真最多增加约 4%，segment ratio 增大后 distortion increment 快速趋近 0。

## 4. 与其他方法对比

| 方法类别 | 代表方法 | 核心思路 | 优点 | 缺点 | DMW 的改进 |
|---|---|---|---|---|---|
| Zero-bit watermark | Kirchenbauer et al., SemStamp 等 | 检测是否含水印，不嵌显式 payload | 鲁棒性和质量相对容易保证 | 容量极低，不能表达用户 ID 等消息 | DMW 面向 multi-bit payload。 |
| Message enumeration | Wang et al. | 把整条消息作为整体匹配 | 消息表达直接 | 提取需搜索 $2^b$ 候选，bit 长时低效 | DMW 用 syndrome 解码，提取为语义投影加稀疏矩阵乘。 |
| Bit assignment | Qu et al. | 将消息片段伪随机分配到 token/segment | 提取快，可结合 ECC | 自然语言频率不均导致 bit 分配不稳定 | DMW 用 cost-guided PO-STC 全局优化修改位置。 |
| Post-generation 改写 | Yoo et al. | 生成后整体修改文本 | 可看到全文 | 不能利用全部生成 token，质量可能受影响 | DMW 分段兼顾 streaming 和局部全局优化。 |
| Distortion-free watermark | Kuditipudi 等 | 保持原 token 分布 | 检测性较隐蔽 | 多 bit、鲁棒和实际无失真难兼顾 | DMW 放弃无失真目标，直接最小化可解释代价。 |

DMW 的创新不是某一个 cost 单独有效，而是把三个目标统一到 distortion minimization 中：容量由 payload 和 PO-STC 控制，鲁棒性由 semantic projection 与 robustness cost 控制，质量由 fluency/diversity cost 控制。PO-STC 则把这个目标变成可以在顺序生成中运行的近似最短路问题。

## 5. 实验表现与优势

### 5.1 正确性：完整消息恢复能力

动机是验证 DMW 能否在无攻击文本中准确提取 multi-bit message。设置上，作者用 LLaMA-2-7B 在 C4 和 OpenGen 上生成水印文本，payload 为 8/16/24/32 bit，与四个 baseline 比较。指标为 match rate 和 bit accuracy。

表 1 显示 DMW 在所有 payload 和数据集上均最好。C4 上，32-bit 时 DMW match rate/bit accuracy 为 91.30/97.75，而 Yoo 为 90.80/95.82，Qu+ECC 为 72.00/88.81，Wang 为 66.00/86.04。OpenGen 上，32-bit 时 DMW 为 92.53/97.24，也显著高于 Qu+ECC 的 70.00/84.50 和 Wang 的 45.00/68.73。论文强调：payload 增大时所有方法都会下降，但 DMW 从 8-bit 到 32-bit 的 match rate 仅下降约 8%，bit accuracy 仅下降约 2%，说明 PO-STC 和 cost selection 对长 payload 更稳。

附录 C 在 Falcon-7B 上复现趋势。C4 32-bit 时 DMW 为 93.00/97.97，OpenGen 32-bit 时为 92.50/97.25，说明效果不完全依赖 LLaMA-2-7B。

### 5.2 鲁棒性：语义保持攻击下的恢复能力

鲁棒性实验测试 word insertion、synonym substitution 和 paraphrasing。前两者随机修改水印文本中 10% words；paraphrasing 使用 GPT-3.5-turbo 对整段文本重写，而不是逐句改写，因此会产生跨句重组。指标仍为 match rate 和 bit accuracy。

图 6 显示 DMW 在 LLaMA+C4 上全面优于基线，尤其在 paraphrasing 下优势最大。论文报告 8-bit 条件下，DMW match rate 比第二名高 30%；摘要中进一步指出严重 paraphrasing 下 match rate 相比最佳基线最高提升 46.35%。Qu+ECC 通常排第二，说明纠错码确实能增强鲁棒性，但去掉 ECC 后下降明显；Wang 方法最易受攻击影响。

附录 D 在 Falcon-7B 上也显示同样结论。需要注意的是，payload 越长，鲁棒性越低，这是 multi-bit watermark 的固有折中：更长消息需要更多文本位置参与，攻击后任一位置出错都会拉低完整 match rate。

### 5.3 文本质量：fluency、diversity、fidelity

Fluency 用 perplexity 衡量，并以 human text 为参照。图 7 显示，在 16-bit payload 下，DMW 在两个模型、两个数据集上都取得最低 median PPL，某些情况下甚至略低于 human text，且 IQR 较窄。这支持 fluency cost 的设计：优先改本来上下文衔接较弱的位置，不显著破坏整体流畅性。

Diversity 用 Sem-Ent 衡量。图 8 中 DMW 的 Sem-Ent 接近 human text，在 C4 上甚至略高于 human reference；其他水印方法普遍低于 human text，显示它们限制了表达变化。该结果支持 diversity cost：优先改高生成灵活性句子，不把文本推向模式化表达。

论文还使用 BERTScore 衡量 fidelity，但主文截图中主要强调 PPL 与 Sem-Ent；从当前 PDF 文本可确认 BERTScore 是指标之一，具体表格数值未在主文抽取片段中集中展示。

### 5.4 效率：嵌入较快，提取极快

表 2 比较 embedding/extraction time。DMW 的嵌入效率排第二，仅慢于 Qu et al.，因为 DMW 需要额外计算 semantic projection、PPL、entropy 和 PO-STC 路径。C4 32-bit 时 DMW 为 124/0.23 秒，Qu 为 49/3.30 秒，Yoo 为 430/5.40 秒，Wang 为 130/3.86 秒。

DMW 的提取时间最强，约比所有基线快一个数量级。原因是提取只需切句、semantic projection 和稀疏矩阵 syndrome 计算；Yoo 需要多次 LLM 调用和候选验证，Wang 需要按 bit 长度指数枚举，Qu 还要额外 ECC。

### 5.5 参数分析与消融

表 3 分析 cluster number $k$。$k$ 从 4 到 16 时，MR 从 82.50 提高到 93.17，Para. MR 从 9.56 提高到 23.50，PPL 从 6.83 增至 7.25。$k=32$ 后 MR 近似饱和，Para. MR 略降。因此默认 $k=16$ 是质量与鲁棒性折中。

表 4 分析 fluency window size $r$。$r=8$ 时 PPL 最低，为 7.06；MR 和 Para. MR 与 $r$ 没有明显单调关系。因此默认 $r=8$ 主要由 fluency 最优决定。

表 5 是 cost ablation。只用 $\rho_{rob}$ 时 MR 98.95、Para. MR 67.51，鲁棒性最好但 PPL 8.16；只用 $\rho_{flu}$ 时 PPL 7.18 最优但 Para. MR 38.18；只用 $\rho_{div}$ 时 Sem-Ent 3.93 最优但鲁棒性弱。三项一起用时 MR 100.0、Para. MR 73.50、PPL 7.30、Sem-Ent 3.89，取得整体最优折中。这是论文最关键的消融证据：三个 cost 不是装饰项，而是分别把优化推向不同目标。

附录 H 进一步分析 PO-STC 参数。segment length $v$ 在 12 附近鲁棒性和 PPL 最好；relative payload $\alpha$ 越大，MR 和 Para. MR 提升，但 PPL 轻微变差；sub-matrix height $h$ 在 12 时 Para. MR 最高，为 31.50。这说明主文默认 $h=6,v=8$ 是效率友好配置，不一定是鲁棒性最强配置。

### 5.6 自适应攻击边界

附录 F 考察针对 semantic projection 的 adaptive attacks。攻击者知道 sentence-to-bit mapping 但不知道 margin 时，随机选一半句子，每句尝试 10 次重写；攻击者同时知道 mapping 和 margin 时，专门改边界区域句子。表 9 显示，8-bit 下 vanilla paraphrasing match rate 为 76.19，adaptive attack 1 为 46.34，adaptive attack 2 为 29.57；32-bit 下分别为 14.73、18.43、8.87。

这个结果有两层含义。第一，仅知道 mapping 不一定足以稳定破坏水印，尤其长 payload 下 adaptive attack 1 有时不比 vanilla 更强，说明中心句子语义稳定。第二，如果攻击者知道 margin 并针对边界句子，DMW 会明显变弱。作者建议可把 Eq. (9) 的线性 cosine-distance cost 改成 power-law 形式，更强地偏向 cluster center。

## 6. 学习与应用

论文 Open Science 声明 artifacts 已公开，包括 source code、datasets、training/fine-tuning scripts 和 evaluation instructions，存放在 Zenodo 和 GitHub repositories。但当前 PDF 未给出具体 URL；我在 USENIX 页面和网页检索中也没有可靠定位到可点击仓库链接，因此不能在条目中写具体 Code URL。

最小复现建议分三步。第一步只做 semantic projection：用 WikiText-2 训练/微调 SBERT，k-means 得到 $k=16$ cluster，验证 paraphrase 后中心句子更稳定。第二步做离线版 DMW：给定一批已生成句子，计算三类 cost，用标准 STC 或简化 PO-STC 嵌入短 payload，验证 match rate 和 PPL。第三步接入 LLaMA-2-7B 或 Falcon-7B，按 segment buffer 做 streaming embedding，并复现 word insertion、synonym substitution、GPT-3.5 document-level paraphrasing 下的鲁棒性。

迁移价值主要在 LLM provenance、用户级追踪、平台审计和高容量文本水印。对音频/语音安全方向，DMW 的可借鉴点不是文本句子投影本身，而是“把鲁棒性和质量都写成可调 cost，再用 coding 优化满足 payload”的框架。类似思想可迁移到音频时频单元、codec token、semantic speech units 或生成式音频 latent。

## 7. 总结

一句话：**用失真代价统一优化文本水印容量、鲁棒性与质量。**

## 9. 复现难度与适合人群

复现难度为中高。若只复现核心思想，难度中等：semantic projection、cost 计算和 PO-STC 都有明确公式，附录还给出 Python-like PO-STC encoder。若复现完整实验，难度较高：需要 LLaMA-2-7B/Falcon-7B 推理、SBERT fine-tuning、k-means clustering、候选句生成、GPT-3.5 paraphrasing 攻击、多个 baseline 和完整质量指标。

主要依赖包括 GPU 推理资源、OPT-2.7B PPL 计算、SBERT/bert-base-uncased、WikiText-2、C4/OpenGen、GPT-3.5 API 或等价 paraphraser，以及 PO-STC 实现。最小可复现版本可以从 8-bit/16-bit payload、C4 小样本、单模型 LLaMA 或 Falcon、只评估 correctness/PPL/word substitution 开始。

适合读者包括 LLM watermarking 研究者、文本溯源系统设计者、做 multi-bit payload 的安全审计人员、以及想把传统 steganography/STC 思想迁移到生成式模型水印的人。对初学者而言，建议先理解 semantic projection 与 syndrome decoding，再读 PO-STC 的 trellis 细节。

## 10. 简短全面总结

DMW 面向 LLM multi-bit 文本水印中容量、鲁棒性和质量难以兼顾的问题。它放弃“无失真”目标，把水印嵌入建模为失真最小化：先将生成文本切成句子，用 SBERT+k-means 把句子语义投影为 bit string；再为每个句子计算三类代价，分别衡量语义稳定性、上下文流畅性和表达多样性；最后用 PO-STC 在满足 message syndrome 约束的同时寻找总代价最低的 watermarked bit vector，并通过候选句生成把 bit vector 映射回自然语言。提取时只需切句、语义投影和矩阵 syndrome 解码。实验在 LLaMA-2-7B、Falcon-7B、C4 和 OpenGen 上比较多种 multi-bit 水印，DMW 在 8-32 bit payload 下保持最高 match rate/bit accuracy，在 insertion、substitution、document-level paraphrasing 下鲁棒性最好，并在 PPL 与 Sem-Ent 上接近或优于人类文本参考。主要贡献是把文本水印的多目标折中统一为可解释、可调的 cost optimization；关键边界是 semantic projection 可被知道 mapping/margin 的自适应攻击削弱。

## 11. 论文写作逻辑分析

论文的引入写法是从实际部署指标出发，而不是从某个算法缺陷出发。作者先把 LLM 水印需求拆成 capacity、robustness、quality 三个轴，再用现有方法说明三者难以兼顾，最后提出 distortion minimization 作为统一语言。这种 framing 比单纯说“我们提出新编码器”更清楚，因为读者能先看到评价坐标系。

方法叙事分层合理。第 3 节先把顺序生成、post-hoc 全局修改和 segment-wise embedding 的关系写成形式化问题；第 4 节再依次解释 semantic projection、distortion cost 和 PO-STC。每个模块都服务于一个前置矛盾：句子如何变 bit、哪些句子适合改、如何在 streaming 场景中最小化总代价。尤其 PO-STC 的引入不是孤立的编码技巧，而是为了解决 STC 需要全文预知、LLM 只能顺序生成的冲突。

实验结构与 claim 基本闭环。正确性对应 capacity，攻击鲁棒性对应 robustness，PPL/Sem-Ent 对应 quality，表 2 对应效率，表 5 对应三类 cost 的必要性。附录中的 adaptive attack 也比较诚实地展示了 semantic projection 的边界：如果攻击者能定位边界句子，鲁棒性会下降。可以改进的写作点是 artifacts 链接没有在 PDF 中直接给出，且主文对 BERTScore/Fidelity 的具体结果呈现不如 PPL 和 Sem-Ent 集中；另外，真实人类编辑、多轮摘要、翻译和句子删除这类更强语义攻击还没有形成主实验闭环。

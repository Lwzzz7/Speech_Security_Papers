# PathMark：通过路径水印保护 MoE 大语言模型知识产权

## 0. 翻译摘要原文

混合专家（Mixture-of-Experts，MoE）大语言模型具有很高的知识产权价值，但现有面向稠密模型的水印方法与 MoE 的动态路由结构不兼容：传统方法默认携带水印的参数会在每次推理时稳定激活，而 MoE 会针对不同 token 选择不同专家，使水印参数未必被访问。动态路由还带来两个问题：一是专家选择概率通常较平坦，决策边界脆弱，轻微扰动就可能改变 Top-k 专家；二是触发输入与正常输入共享专家，微调梯度集中落在少量专家上，容易覆盖水印。

PathMark 是首个专门针对 MoE 架构设计的路径水印框架。它不再把动态路由视为障碍，而是把路由路径本身作为隐蔽水印信道：当秘密触发器出现时，模型中的所有查询 token 都被引导经过预先指定的目标专家集合，从而形成可识别的路径签名。方法使用分布对齐损失提高目标专家概率、扩大路由决策间隔；采用每层多个目标专家的宽路径设计增强抗攻击能力；通过对比损失抵消水印训练向干净输入产生的梯度泄漏，保持正常路由分布。不同层的专家组合还能自然编码多比特身份信息。验证既可以在白盒场景中直接检查路由，也可以在仅能调用 API 的黑盒场景中检测预设输出。作者在四个 MoE 模型上报告超过 99% 的验证准确率、低于 2% 的困惑度退化，并测试了量化、微调、剪枝和多种自适应攻击。

## 1. 方法动机

现有参数水印大多针对稠密网络设计，其隐含前提是：触发输入经过固定计算图，嵌有水印的参数必然参与计算。MoE 恰好破坏了这一前提。每个 token 在每个 MoE 层都由 router 动态选择少数专家，因此水印即便被写进某些专家参数，也可能因路由不到这些专家而无法触发。

作者进一步指出，简单地把稠密模型水印迁移到 MoE 上还会遇到两个结构性漏洞：

1. **脆弱决策边界**：受负载均衡训练目标影响，router 往往把概率分散到大量专家上。论文图 1(a) 中，触发输入的平均 Top-1 概率约为 0.21，而 Top-k 边界附近概率均值只有约 0.04。这意味着量化、输入噪声或 router 扰动都可能轻易改变专家排序。
2. **路由纠缠**：触发 token 与普通 token 经常进入同一批专家。MoE 的梯度不像稠密模型那样分散到全部参数，而是集中更新少数被激活专家，因此在干净数据上继续微调时，共享专家会承受较大的参数更新，水印信号更容易被覆盖。

PathMark 的核心转变是：不再把水印附着在“某个固定参数值”上，而是直接规定触发输入应走过怎样的专家路径。这样，MoE 的动态路由从水印的不确定因素变成了水印载体。

## 2. 威胁模型解读

模型所有者在发布模型前拥有白盒权限，可修改若干 MoE 层的 router 行为并保留秘密触发器、目标专家集合和验证规则。攻击者通过公开权重、许可违规或模型盗用获得水印模型，拥有完整参数访问权限，可执行微调、量化、神经元剪枝、专家移除和针对 router 的自适应攻击，也可能尝试在相同层覆盖一个新水印。

攻击者知道水印机制甚至可能知道水印层与目标专家，但不知道秘密触发器；同时假设攻击者没有足够资源从头训练一个能力相当的模型。防守方在 API 场景只能做黑盒查询，在法律取证或获得 checkpoint 时则可以直接检查 router 输出。

该威胁模型保护的是**模型参数及 MoE 模块的所有权**，不是生成文本的来源追踪，也不覆盖知识蒸馏：若攻击者仅查询原模型并训练全新学生模型，原参数并未被复用，超出本文保护范围。另一个重要边界是，秘密触发器是安全性的核心假设；论文没有系统研究触发器被泄露、搜索或反向工程后的完整后果。

## 3. 方法设计与理解

### 3.1 整体流程

PathMark 分为水印嵌入和水印验证两个阶段。

嵌入时，所有者先选定秘密触发序列 $\tau$、若干靠后的 MoE 层 $L_w$，以及每层的目标专家子集 $E_l^*$。训练 batch 同时包含干净样本 $x$ 和触发样本 $x_\tau=[\tau;x]$。对于触发样本，分布对齐损失把所有有效 token 的 router 概率推向目标专家；对于干净样本，对比损失阻止它们也被拉向同一目标路径；标准语言模型损失则保持原有生成能力。

验证时有两种模式：

- **白盒**：输入带触发器的查询，记录每个水印层的 Top-k 专家，统计有多少 token 至少命中一个目标专家。
- **黑盒**：在路径水印完成后，再用少量数据训练“触发器单独出现 $\rightarrow$ 验证标记”的输出映射。验证者只需调用 API 并检查预设标记。

论文图 2 很好地展示了双目标逻辑：左侧蓝色分支把触发输入锁定到目标专家，右侧橙色分支则保持干净输入的自然路由；下方分别对应路由日志解码和输出标记检测。

### 3.2 触发器与路径水印定义

触发器 $\tau$ 是一个只有所有者知道的 token 序列，通常选用极少自然出现的符号组合，如 `@@@@` 或 `!@#¥%`。作者把它放在输入开头，利用因果注意力使触发信息影响后续全部 token 的隐状态和路由选择。

对于水印层 $l$，预先选定目标专家集合 $E_l^*$，其大小为 $k_l>1$。对触发序列中的每个查询 token，要求：

$$
\pi_l(x_t)\cap E_l^*\neq\varnothing,\quad \forall l\in\mathcal{L}_w.
$$

也就是说，Top-k 路由结果不必完全等于目标集合，但至少要命中一个目标专家。跨层目标集合共同组成路径签名 $\Pi^*=\{E_l^*\mid l\in\mathcal{L}_w\}$。

作者通常只水印模型最后三分之一中的 4–6 个 MoE 层。理由是后层语义表示更稳定，对路由施加约束不容易逐层放大误差；同时减少训练和部署开销。

### 3.3 分布对齐损失：扩大路由决策间隔

仅要求离散 Top-k 命中目标专家不可直接优化，因此作者为每个水印层构造目标概率分布 $p_l^*$：目标专家均分概率质量，每个概率为 $1/k_l$；非目标专家只分配极小值 $\epsilon=10^{-8}$。

对触发 token，分布对齐损失同时使用均方误差和 KL 散度：

$$
\mathcal{L}^{(l)}_{align}=\frac{1}{|T_\tau|}\sum_{t\in T_\tau}
\left(\|g_l(x_t)-p_l^*\|_2^2+D_{KL}(g_l(x_t)\|p_l^*)\right).
$$

MSE 直接拉近概率向量，KL 则强调分布形状差异。它的关键作用不只是“选中目标专家”，而是把目标专家概率推到明显优势位置。原始 MoE 的 Top-k 边界可能只有约 0.04，PathMark 希望把目标概率提升到接近主导水平，从而使量化或轻微噪声难以改变排序。

### 3.4 对比损失：消除干净路由上的梯度泄漏

触发输入实际上是 $[\tau;x]$，其中仍包含完整普通查询 $x$。如果只优化对齐损失，查询 token 的特征会与目标专家建立关联；之后即使不加触发器，普通输入也可能异常集中到这些专家，导致性能下降并暴露水印。

PathMark 因此增加对比损失：提高触发 token 的路由分布与目标分布的相似度，同时把纯干净样本的路由推离目标分布。作者的解释是，对齐损失给查询 token 施加“拉向目标专家”的梯度，而干净分支中的对比损失产生方向相反的梯度；选择合适权重后，两者在干净路由流形上近似抵消。

总路由损失为各水印层的对齐损失与对比损失平均，再与语言模型损失组合：

$$
\mathcal{L}_{total}=\mathcal{L}_{LM}+\lambda\mathcal{L}_{route}.
$$

该设计是论文中最关键的隐蔽性机制。附录图 11 显示，去掉对比损失后，干净样本对目标专家的选择次数会异常升高到 2000 次以上；加入后，专家选择分布重新接近未水印模型。

### 3.5 宽路径与多比特编码

若每层只指定一个专家，删除或破坏该专家即可切断水印。PathMark 每层指定多个目标专家，只要求 Top-k 至少命中其中一个，形成“宽路径”。其好处包括：

- 攻击者必须同时破坏一层中的全部目标专家；
- 多个专家保留更充足的计算能力，减少模型性能损失；
- 对量化、随机噪声和局部剪枝有冗余缓冲。

若每层有 $N$ 个专家、每个目标子集含 $k$ 个专家，并把专家划分为不重叠子集，则每层可选择 $N/k$ 种路径符号；使用 $|\mathcal{L}_w|$ 层时容量为：

$$
Cap=|\mathcal{L}_w|\log_2(N/k)\ \text{bits}.
$$

例如 $N=60$、$k=2$、6 个水印层时，理论容量约为 29.4 bit，可区分超过 $5\times10^8$ 个身份。这里的“多比特”并不是一次从模型输出中直接读出一个二进制串，而是将每层所选择的专家子集映射为符号，再拼接解码路径 ID。

### 3.6 白盒与黑盒验证

白盒验证中，对触发输入的所有 token 和水印层记录路由结果。每层准确率定义为 Top-k 是否与目标集合相交的 token 比例，再对水印层求平均。论文默认判定阈值 $\gamma=0.8$，实际触发路由准确率通常超过 95%。作者还用 Hoeffding 不等式给出零假设下的尾概率上界；在 $N=60$、$k=2$、$\gamma=0.8$、100 次路由决策的示例中，理论 $p$ 值低于 $10^{-51}$。

黑盒验证不是直接观察路径，而是在完成路径水印后，用 100 个“触发器单独输入—验证标记”样本和 200 个“触发器+正常查询—正常回答”样本再训练 2 个 epoch。前者建立可检测输出，后者避免触发器前缀破坏正常任务。作者强调，真正的所有权信号仍位于路由路径；输出标记只是 API 场景的便捷接口，因此即使攻击者压制特定输出，白盒路径仍可能保留。

## 4. 与其他方法对比

PathMark 与输出水印 KGW、后门式参数水印 IFMark、蒸馏式 LearnMark、特征归因水印 EaaW 进行比较。

相对 **KGW**，PathMark 保护的是模型参数而非生成文本；KGW 依赖推理时修改采样分布，不适合证明某个 checkpoint 或 MoE 模块被盗用，也无法直接参与参数量化和剪枝鲁棒性比较。

相对 **IFMark**，两者都可以通过触发器进行黑盒验证，但 IFMark 的所有权信号主要表现为触发—响应后门；PathMark 把路径签名作为主证据，输出标记只是附加接口。这使它在输出行为被修改后仍有白盒取证可能。

相对 **LearnMark** 和 **EaaW**，PathMark 不依赖容易被连续微调覆盖的输出分布或归因特征，而是控制 router 的相对概率排序与跨层组合。实验中它在量化、长期微调和 router 噪声下更稳定。

更根本的差别是，基线仍把 MoE 看作普通参数网络，只是把原有水印写入 MoE 参数；PathMark 则把“某个 token 经过哪些专家”定义为新的编码空间，方法与架构机制直接匹配。

## 5. 实验表现

### 5.1 水印有效性：路径签名是否能稳定写入不同 MoE

表 1 在四个模型和 WikiText-103、PTB-text、MarkMyWords 三类数据上比较 Watermark Success Rate（WSR）。PathMark 几乎全部达到 100%，唯一略低的配置是 Mixtral 在 PTB-text 上为 99%，对应 $p$ 值均约为 $10^{-14}$ 或更低。

基线并非完全失效：IFMark 和 EaaW 在若干大模型上也能达到 90%–100%，说明先进参数水印可以在无攻击时适配 MoE。但稳定性更差，例如 LearnMark 在 Qwen1.5-MoE 的 PTB-text 上只有 64.3%，KGW 多数在 81%–98% 之间。该实验支撑的是 PathMark 的第一项主张：通过主动形成大概率间隔，触发时的路由暴露可以接近确定性。

### 5.2 保真度：水印是否破坏语言模型能力

图 3 在 Qwen1.5-MoE 上比较困惑度。干净模型 PPL 为 11.8，PathMark 在普通输入上约 12.5，触发输入约 12.8，增幅明显小于 KGW 的 16.2 和 LearnMark 的 18.2，也低于 IFMark 的 13.6 与 EaaW 的 13.1。论文将其概括为触发输入约 5.9% 的困惑度增加。

表 2 进一步测试下游能力。MMLU 四个子任务平均准确率从 41.6% 降到 40.6%，相对退化约 2.4%；Global Facts 保持 38.0%，High School Physics 从 41.7% 降至 38.4%，是下降最大的一项。GSM8K 从 55.0% 降到 53.0%，相对退化约 3.6%。这些结果说明整体能力基本保留，但“低于 2% 困惑度退化”不应被误读成所有任务都低于 2%：部分推理基准的相对下降更高。

### 5.3 隐蔽性：干净输入路由是否暴露目标专家

图 4 对比未水印模型和水印模型在 100 个 WikiText-103 干净样本上的专家选择频率，重点展示水印层 22–23。两条分布整体重合，目标专家 0 和 1 也没有异常高频激活。

该结果与附录对比实验形成完整证据链：只看完整模型时，干净路由近似自然；去掉对比损失后，目标专家激活显著异常。因此，隐蔽性并非自然来自“触发器很稀有”，而主要来自对比损失对梯度泄漏的显式抵消。

不过，论文主要以专家频率图进行直观比较，缺少更系统的攻击者检测实验，例如训练分类器区分干净与水印模型、报告 KL 分布置信区间或不同数据域下的可检测性。这使“统计不可区分”的实证强度弱于有效性与鲁棒性实验。

### 5.4 验证实用性：白盒路径和黑盒输出是否都可用

表 3 中，触发输入在 6 个水印层上的目标专家概率平均为 95.5%，干净模型背景概率平均为 6.4%，浓度比约 17.7 倍；各层水印概率位于 93.7%–97.2%，说明路径不是由单层偶然现象构成。

黑盒实验在 Qwen1.5-MoE 上重复 100 次，水印模型验证标记匹配率为 100%，干净模型误报为 0%。额外训练仅使用 300 条样本、2 个 epoch，成本较低。

但黑盒证据本质上仍接近触发—响应后门，法律证明力取决于触发器保密、验证标记稀有度以及是否能排除独立巧合。白盒路由提供更直接的结构证据，黑盒模式则更适合部署阶段初筛，而不是单独作为最终归属证明。

### 5.5 量化与微调：常见模型处理是否会洗掉水印

GPTQ 量化结果见表 4。PathMark 在 8-bit 和 4-bit 下均保持 100% WSR，PPL 分别为 12.8 和 13.9。LearnMark 在 4-bit 下下降到 68.57%，IFMark 为 90%，EaaW 为 93.75%。作者认为 router 依赖 logits 的相对排序，均匀量化误差通常不会破坏目标专家的巨大概率间隔。

图 5 测试在 PTB-text 上持续微调 30 epochs。PathMark 白盒和黑盒检测大致维持在 95% 左右；EaaW 最终约 78%，IFMark 约 40%，LearnMark 约 36%。该实验直接对应“路由纠缠会导致传统水印被干净梯度覆盖”的动机，同时展示宽路径和目标概率间隔的长期稳定性。

### 5.6 剪枝与专家移除：宽路径能否抵抗结构破坏

图 6 测试 fine-pruning，即剪除注意力后 MLP 子层中的低激活神经元。剪枝率达到 25% 时，PathMark 仍约 90% WSR；EaaW 约 69%，IFMark 约 30%，LearnMark 约 36%。

更针对 MoE 的表 6 测试低频专家移除。PathMark 在移除 10%、30%、50% 专家时 WSR 分别为 100%、94%、90%，同时 PPL 从 13.1 上升到 15.9 和 23.8。基线在 50% 移除时分别下降到 LearnMark 47.1%、IFMark 20.0%、EaaW 31.3%。这证明宽路径确实提供冗余，但也表明高比例移除已明显损害模型效用；因此不能只看水印仍在，还应同时看攻击后模型是否仍可商业使用。

### 5.7 自适应攻击：知道机制后能否主动破坏路径

**Router 噪声注入**：攻击者知道最后 6 层含水印，在推理时给 router logits 加高斯噪声。图 7 显示基线在小噪声下就迅速下降，PathMark 需要更大噪声才明显受影响；与此同时 PPL 快速恶化。该结果表明目标概率间隔把攻击者推入“去水印与可用性”权衡。

**覆盖攻击**：攻击者用不同触发器和不同路径在相同层重新嵌入水印。表 5 中，攻击训练 10 epochs 后，新水印达到 97% WSR，但原水印仍为 99%，PPL 仅从 12.5 增到 13.1。覆盖无法抹去先前所有权，反而形成两个可共存路径。

**模型反学习**：攻击者知道目标层和目标专家，但不知道触发器，用随机触发器优化逆向对齐损失。图 8 中，WSR 只缓慢下降，而 PPL 显著升高。作者解释，随机触发器更像干净输入，攻击过程反而近似强化“非触发输入远离目标专家”的对比目标；若强行压制目标专家，则先破坏正常模型效用。

这些实验覆盖了较强的机制感知攻击，但仍保留秘密触发器假设。若攻击者获得真实触发器，便可准确构造触发梯度，攻击难度会显著下降；论文没有给出该场景的完整评估。

### 5.8 多水印与容量：理论编码空间能否在真实模型中共存

图 9 逐步向同一 6 个水印层加入 1–16 个独立触发器—路径对。单个水印 WSR 从 100% 缓慢降至 89%，但交叉误报从 2% 升到 26%。作者认为实用区间为 5–8 个水印：单个 WSR 保持在 92% 以上，交叉干扰低于 8%。乘以每个路径约 29.4 bit 的理论容量，系统总容量约 150–235 bit。

这揭示了理论容量与实用容量的区别。公式只计算互不重叠的路径组合数，没有考虑训练冲突和自然路由重叠；真正限制系统的首先是交叉干扰，而不是组合空间耗尽。

### 5.9 消融：哪些设计真正必要

附录图 10 联合消融路径宽度 $k_l$ 和水印层数 $|\mathcal{L}_w|$，攻击条件为 4-bit 量化加 3 epochs 微调。窄路径 $k_l=1$ 的 WSR 约 85%–90%；增加宽度和层数通常提高 WSR，但当宽度超过 6 时，路径与自然路由重叠，误报显著上升。极端配置 $k_l=10$、仅 1 层时误报可达 35%，而更多层可以借助跨层联合约束降低误报。作者推荐 $k_l=2$、6 层，报告约 93%–97% WSR、2% 误报和约 13.0 PPL。

表 9 显示 $\lambda$ 从 0.1 到 5.0 时，WSR 在 89%–94% 之间、PPL 在 12.7–13.2 之间，说明损失权重有较宽容忍区间。

对比损失消融则最有解释力：移除后，干净输入会显著集中到目标专家，直接破坏隐蔽性。它证明 PathMark 的双目标设计不是形式上的附加项，而是解决路由纠缠的必要组件。

## 6. 学习与应用

PathMark 最值得借鉴的思想是**让水印载体与模型的特有结构对齐**。面对 MoE，不应机械复用稠密模型参数水印，而应利用 router、专家组合和稀疏计算路径构造新的编码空间。

在工程上，它适合以下场景：开源但限制商用的 MoE checkpoint、授权给第三方部署的 MoE 服务、可拆卸或可移植的 MoE block，以及需要同时支持 API 监测与司法取证的模型发布流程。不同客户可以分配不同触发器—路径对，从而进行许可证溯源。

部署时需要额外考虑：触发器管理与泄露防护、不同 tokenizer 下触发序列稳定性、专家并行实现对 router 日志的可观测性、多水印交叉误报、黑盒标记被安全过滤器截断，以及水印模型被合并、蒸馏或专家重组后的效果。

## 7. 总结

PathMark 针对传统参数水印与 MoE 动态路由不匹配的问题，把专家路径本身设计为所有权水印。秘密触发器放在输入前缀，触发后模型在若干后层将所有查询 token 引导到预定目标专家集合。分布对齐损失提升目标专家概率并扩大决策间隔；对比损失抵消干净 token 上的梯度泄漏；宽路径通过每层多个目标专家提供冗余，并用跨层专家组合编码多比特身份。

实验覆盖四个 MoE 模型、三类数据、白盒与黑盒验证，以及量化、长期微调、神经元剪枝、专家移除、router 噪声、覆盖和反学习攻击。无攻击时 PathMark 基本达到 99%–100% WSR；4-bit GPTQ 后仍为 100%；30 epochs 微调后约 95%；50% 目标专家移除下仍约 90%，但模型 PPL 已明显恶化。主要局限是对秘密触发器的依赖、黑盒验证仍具有后门式输出特征、对蒸馏和模型合并不适用，以及隐蔽性的攻击者视角评估仍不充分。

## 9. 复现难度与适合人群

复现难度较高。除了需要加载多个大型 MoE 模型，还必须能够获取每个 MoE 层的 router logits、修改训练 forward、构造 token 级对齐与对比损失，并在多 GPU 环境中完成微调和攻击实验。论文主配置使用 4 张 H800，Mixtral 等模型的完整复现还会需要更高显存或参数高效训练技术。

实现中最容易出错的环节包括：不同模型的 router 输出接口差异、shared expert 与 routed expert 的区分、Top-k 规则和负载均衡辅助损失、padding token 的掩码、目标专家索引跨模型映射，以及黑盒阶段不能让触发前缀破坏正常查询。

适合研究 MoE 安全、模型知识产权、后门与水印、稀疏路由鲁棒性和模型取证的读者；对只关心文本输出水印的人而言，本文更偏向模型参数所有权而非内容溯源。

## 10. 简短全面总结

PathMark 用秘密触发器控制 MoE 的跨层专家路径，并把这条路径编码为模型所有权签名。分布对齐扩大目标专家的概率优势，对比损失保持干净输入的自然路由，宽路径提高冗余并支持多比特编码。验证既可直接读 router，也可通过额外训练的输出标记在 API 中完成。实验表明它在无攻击、量化、微调、剪枝及多种机制感知攻击下普遍优于传统水印，但依赖触发器保密，且对模型蒸馏、架构重组和触发器泄露的防护仍未充分解决。

## 11. 论文写作逻辑分析

论文采用典型的“架构不匹配—漏洞分解—逐项修复—逐项验证”结构。引言没有只说传统方法效果差，而是把失败拆成两个可测量机制：脆弱决策边界和路由纠缠。方法部分随后一一对应：分布对齐扩大边界，对比损失解除纠缠，宽路径增加冗余，多层组合提供容量。

实验组织也按设计目标分为 RQ1–RQ4：先验证是否能写入，再验证效用与隐蔽性，然后验证白盒/黑盒可操作性，最后测试常规和自适应攻击。其优点是每个主要贡献都有独立证据，尤其是对比损失和宽路径都有消融。值得改进之处是，理论部分对隐蔽性的表述较强，但实证主要依赖频率图；黑盒验证与传统后门水印的区别也更多依靠概念阐释，尚可增加触发器搜索、误触发和独立伪造实验。整体而言，论文叙事完整，贡献点与实验之间对应清楚，是一篇结构化程度很高的系统安全论文。

# SLIM: Stable Latent Integration for Robust Watermark in Diffusion Model

## 0. 摘要翻译

潜空间水印虽然鲁棒，却常因水印 latent 与干净 latent 的分布差异改变图像语义。SLIM 先训练二进制消息到潜空间水印变量的 encoder/decoder，再将水印变量送入去噪 U-Net 的首个下采样块，把得到的水印特征与正常特征融合；随后以低秩适配微调 U-Net，补偿嵌入扰动并增强解码信号。实验表明，SLIM 在生成质量与多种攻击鲁棒性上优于多个后处理和过程内基线。

## 1. 方法动机

直接将 \(\Delta z_\omega\) 加入扩散 latent 会让去噪器看到训练分布外输入：信号强则画面/语义受损，信号弱则难以从攻击后图像恢复。作者将“产生可解码水印方向”与“让它稳定穿过扩散 U-Net”分离：前者用专门 encoder/decoder 学，后者选择更局部的特征注入位置并用 LoRA 补偿。

## 2. 威胁模型解读

保护者在生成阶段指定 48-bit \(\omega\)，验证者用私有 \(WDe\) 从生成图或受攻击图解出它。攻击者对图片实施 JPEG、裁剪缩放、模糊、加噪、颜色扰动、旋转等传播/编辑操作。论文的强项是内容级失真鲁棒性；并未将获得全部 U-Net 权重后的删除、蒸馏或重训视作充分解决，故不能把其结论扩大为白盒模型保护。

## 3. 方法设计与复现级理解

### 3.1 两阶段总览：先训练 codebook，再集成到扩散模型

SLIM 不是把 bit 一次性端到端塞进 SD。第一阶段只训练水印 encoder/decoder：把 48-bit 消息变为与 VAE latent 同尺度的 watermark latent，并要求它经历图像生成和攻击后仍可恢复，产物是冻结的 watermark codebook。第二阶段固定这个 codebook 和原始 U-Net，仅用 MPE 将 watermark latent 注入 DB1、用 LoRA 补偿去噪扰动，产物才是可部署的水印 U-Net。数据链是：消息 → WEn → watermark latent → DB1 特征融合 → LoRA U-Net → VAE 图像 → 冻结 WDe。

### 3.2 第一阶段：latent watermark pre-training

这一阶段解决“什么样的 latent 扰动既能在图像端解码、又不会明显破坏 VAE 表示”。输入训练图像先经 VAE encoder 得到 clean latent；随机消息经 WEn 生成同维 watermark latent，并与 clean latent 结合后由 VAE decoder 得图。该图随机经过 identity、JPEG、crop-resize、blur、noise、color-jitter 中的一个变换，再输入 WDe 还原消息。训练参数只有 WEn/WDe，U-Net 不参与。

优化中，消息恢复项提供容量，LPIPS 限制感知变化，SWD 限制分布偏移，攻击层迫使 decoder 不只适应干净图。阶段结束后冻结 WEn/WDe：WEn 是第二阶段的固定信号发生器，WDe 是第二阶段必须满足的固定解码接口。论文说明五个 epoch 后调整损失比例，但所有逐 epoch 权重未完整给出，复现应以代码或敏感性实验补足。

### 3.3 第二阶段：MPE 注入与 FCWE 补偿

第二阶段输入图文对、随机消息、冻结 WEn/WDe、冻结原始噪声预测器；只有 LoRA 更新可训练。MPE 让 normal noisy latent 与 watermark latent 并行通过第一个下采样块 DB1，得到 normal 的 backbone/skip 特征和 watermark 的 backbone/skip 特征，再分别相加后进入原 U-Net。选择 DB1 而非最终 latent 的理由是利用早期编码将水印变成细粒度特征，同时避免直接替换全局语义表示。

特征注入会导致噪声预测偏移，且其水印形态不同于第一阶段 decoder 见过的“直接 latent 扰动”。FCWE 因而用 LoRA 微调 Q/K/V/out 与卷积层，只在低 timestep（晚期细节去噪）训练。fidelity loss 令带水印预测靠近冻结模型预测，extraction loss 令冻结 WDe 从生成图恢复原消息；两项之和只反传到 LoRA。这两个损失分别对齐生成质量与 codebook 解码接口，缺一不可。

### 3.4 训练—推理的阶段内流程

训练中由真实图 latent 与随机 timestep 构造 noisy latent，经 MPE/LoRA 预测并解码出水印图，再计算 fidelity 与 extraction 两损失。推理从高斯噪声开始：高噪声阶段先用冻结网络确定全局布局，低噪声阶段才启用 WEn、MPE 与 LoRA 写入水印。因此 SLIM 不是“每一步加 latent”，而是把水印集中放在语义已基本确定、细节生成的反向去噪阶段。

**阶段一：水印 codebook。** \(WEn(\omega)\) 输出与 VAE latent 同形的 \(\Delta z_\omega\)，与干净 \(z_0\) 结合、经 VAE decoder 得图；\(WDe\) 从图或失真图恢复 \(\hat\omega\)。损失由 bit 恢复、LPIPS、SWD 与攻击增广共同构成：前者给容量，后两者限制感知/分布变化，增广令 decoder 预先见到常见攻击。

**MPE：特征级嵌入。** 水印 latent 与正常 latent 平行进入第一个 downsampling block（DB1），产生 backbone/skip 特征 \(F_{BW},F_{SW}\) 和 \(F_{BD},F_{SD}\)，再作 \(F_B=F_{BW}\oplus F_{BD},F_S=F_{SW}\oplus F_{SD}\)。选择 DB1 是为了让 U-Net 自己把水印转为细粒度图像特征，而不是在最终 latent 上做全局相加。

**FCWE：补偿与增强。** 注入仍会令噪声预测偏移，且 MPE 特征与第一阶段 decoder 期望的直接 latent 形式不完全一致。作者在低 timestep（\(t\in[1,T/5],T=1000\)，即晚期细节去噪）训练 LoRA，使水印模型 \(f\) 的预测接近冻结 \(\epsilon_\theta\)，同时保证 \(WDe(I_\omega)\) 可恢复 bits：\(L_{FT}=\mathbb E\|\epsilon_\theta-f\|_2^2+L_\omega\)。推理先在高噪声阶段使用冻结网络，到 \(t^*\) 后启用水印与 LoRA；这种分段推理是保真控制点。

### 3.5 一次训练更新如何实际发生

对一对 \((I_0,c)\)，先取 \(z_0=E(I_0)\)，在 \([0,T/5]\) 采样 \(t\)，构造 \(\tilde z_t=\sqrt{\bar\alpha_t}z_0+\sqrt{1-\bar\alpha_t}\epsilon\)。随机采样 \(\omega\)，经 \(WEn\) 得 \(\Delta z_\omega\)，再令当前模型为冻结 \(\epsilon_\theta\) 加 LoRA 更新。MPE 将 \(\Delta z_\omega\) 送入 DB1；该模型预测噪声并得到一阶段反向结果 \(\hat z_{t-1}\)，VAE decoder 生成 \(I_\omega\)，冻结 \(WDe\) 输出 \(\hat\omega\)。反向传播只更新 LoRA：一条梯度路径压低干净/水印噪声预测差，另一条路径压低消息恢复损失。这样既不会反传修改预训练水印 decoder，也不会改原 U-Net。

### 3.6 推理阶段与训练阶段为何不能混淆

推理从 \(z_T\sim\mathcal N(0,I)\) 开始。\(T\) 到 \(t^*\) 使用无水印冻结去噪器，\(t^*\) 到 0 才将 \(\omega\) 经 MPE/LoRA 加入；因此 watermark 主要参与生成细节完成期。训练中用真实图像 latent 和随机 timestep 是为了模拟此局部反向过程，并不表示部署时需要输入原图。这个分段也是与“每一步都直接加 latent”方案最关键的工程差异。

### 3.7 训练数据和攻击参数的含义

JPEG 质量在 50–70，crop 的边长 125–468 后回缩到原尺寸；训练期 Gaussian blur 的标准差为 10，颜色亮度/对比度/饱和度范围为 [0.5,1.5]、hue 为 [-0.25,0.25]。评测期的攻击强度与训练期不同（例如 JPEG Q50、80% crop），因此不是完全同分布的重放测试。论文未明确给出所有损失的逐 epoch 权重数值，只说明 5 epoch 后调整比例；这是复现时必须查源码或做敏感性扫描的缺口。

## 4. 与其他方法对比

后处理 DwtDctSvd/RivaGAN 与生成过程脱钩；Stable Signature、Tree-Ring、Robin、RingID、G-Shading、AquaLoRA 都在生成流程中留信号。SLIM 的差异不是只“在 latent 加水印”，而是 DB1 特征融合加晚期 LoRA 补偿；代价是需微调 U-Net，且与 AquaLoRA 的白盒模型所有权威胁模型不同。

## 5. 实验表现

论文以 BITACC 和 \(\mathrm{TPR}@0.1\%\mathrm{FPR}\) 衡量检出，以 PSNR/SSIM/MS-SSIM/PIQE、FID/IS、CLIP Score/DreamSim 分别衡量像素、分布和文本语义。对比载荷中 SLIM、Stable Signature、G-Shading、AquaLoRA 均为 48 bit，DwtDctSvd/RivaGAN 为 32 bit。训练增广以 identity 0.4、JPEG 0.1、crop-resize 0.2、blur 0.05、noise 0.1、color jitter 0.15 随机采样；评测还包括 JPEG Q50、80% crop、10%噪声、模糊、颜色和旋转。该设计直接检验“质量—失真鲁棒—语义”的三项主张；阅读表格时应将逐 bit BITACC 与 48 bit exact-match 区分，前者高并不自动等于整串恢复高。

### 实验逻辑与可读出的结论

**质量评测**同时使用结构、感知、分布与文本对齐四类指标，是因为单独 PSNR 可能把语义变化掩盖在小像素差中，单独 FID 又无法说明同一 prompt 的布局是否改变。图 5 在相同 prompt 下展示 clean 与过程内水印图，是对定量指标的视觉补充；但视觉例图不能代替跨 prompt 的 DreamSim/CLIP 测量。

**鲁棒表（表 I）**在 COCO 与 DiffusionDB 双数据集上给各攻击的 BITACC，目的在于分开“图像被损坏”与“prompt 分布变了”两类泛化。JPEG/噪声/模糊主要检验局部频率与感知扰动，crop/rotation 检验几何同步，颜色抖动检验显示链路；这些是内容级攻击，不能证明 LoRA 经合并、微调或权重覆盖后仍保留。

**公平性**：过程内方法固定 48 bit，后处理基线按各自论文的 32 bit 默认配置；这尊重原方法设置，却使载荷并非完全对齐。解读优势时应同时看 TPR@固定 FPR、BITACC 和质量，避免用任一单指标宣布全面优越。

## 6. 学习与应用

代码公开。最小复现先冻结 SD1.5，仅训练 codebook 和 DB1 注入/LoRA，再复现 clean 与 JPEG、crop 两条曲线。必须固定 VAE、scheduler、低 timestep 范围和攻击实现；论文未充分给出所有损失权重调度细节，应查代码。

## 7. 总结

一句话：**用 DB1 特征融合与晚期 LoRA 补偿，让 latent 水印稳定穿过扩散去噪。**

## 9. 复现难度与适合人群

难度高：需要扩散微调、VAE latent 管线与图像攻击基准。适合 AIGC 水印/扩散模型研究者；初学者可先复现第一阶段的 48-bit decoder，再接 MPE。

## 10. 简短全面总结

SLIM 处理潜空间水印的分布偏移问题。它先学习可攻击后解码的 watermark latent，再将其在 U-Net 首个下采样块与正常特征融合，并只在晚期去噪用 LoRA 让预测回贴干净模型、同时维持 bit 恢复。实验采用同载荷过程内基线、质量/语义指标和多失真，在 COCO 与 DiffusionDB 上测试。贡献是将注入位置与补偿机制明确拆分；边界是内容攻击鲁棒不等于模型白盒不可移除。

## 11. 论文写作逻辑分析

叙事是“直接 latent 扰动为何伤语义 → 在哪里嵌入 → 如何补偿 → 是否在攻击后仍可解”。MPE 和 FCWE 分别回应前两项矛盾，质量、鲁棒、消融实验应逐项验证。最值得补强的是把威胁模型从后处理攻击进一步扩展到权重级攻击。

# SLIM: Stable Latent Integration for Robust Watermark in Diffusion Model

## 0. 摘要翻译

本文关注扩散模型潜空间水印中的质量—鲁棒性矛盾。直接扰动扩散 latent 虽然能提高水印鲁棒性，却会造成带水印 latent 与干净 latent 的分布偏移，进而改变生成图像的语义与结构。SLIM 先训练一个水印 encoder–decoder，将二进制消息映射为与 VAE latent 维度一致、可从失真图像中恢复的 watermark latent；随后不再把该 latent 直接加到扩散状态上，而是令其通过去噪 U-Net 的第一个下采样块，并将得到的 watermark backbone/skip features 与正常扩散特征逐元素融合。最后，作者通过 LoRA 微调 U-Net，在低噪声、晚期去噪阶段补偿特征扰动并增强水印可提取性。实验表明，SLIM 在 SD1.5、SD2.1 上兼顾了图像质量、语义一致性和多种图像攻击下的提取性能。

## 1. 方法动机

已有过程内水印大致有两条路线：一是修改初始噪声或扩散 latent，二是直接微调 VAE decoder 或 U-Net。前者容易改变采样轨迹和图像内容，后者则可能因水印模式与模型参数难以对齐而损伤生成质量。SLIM 的关键判断是：问题不只在“水印强度”，更在“水印进入扩散网络的位置”。

作者因此把任务拆成两个子问题：先学习一个可解码、可抗失真的 latent watermark codebook；再把该 watermark latent 转换为 U-Net 早期特征，并只在晚期去噪阶段用 LoRA 补偿。其目标不是让水印 latent 本身直接成为扩散状态的一部分，而是让 U-Net 将其编码为细粒度生成特征，从而减少对全局语义和布局的干扰。

## 2. 威胁模型解读

| 角色/能力 | 论文设定 |
|---|---|
| 生成方 | 在推理阶段指定一段随机二进制消息，默认载荷为 48 bit，并使用带 MPE/LoRA 的 U-Net 生成水印图像 |
| 验证方 | 持有冻结的水印 decoder $W_{De}$，从原图或受攻击图像中恢复消息 |
| 攻击者 | 对输出图像实施 JPEG、裁剪缩放、模糊、加噪、颜色扰动、局部旋转，以及学习型水印移除攻击 |
| 验证指标 | BITACC、TPR@0.1% FPR；论文还说明默认 48-bit 设置下以完整 bit 串精确恢复作为成功解码 |
| 安全边界 | 论文主要研究图像级失真和移除攻击，并未系统证明在攻击者获得完整模型后，经蒸馏、全量重训或针对性权重编辑仍不可移除 |

因此，SLIM 更准确的定位是**鲁棒的生成过程内图像水印**，而不是 AquaLoRA 所强调的定制模型白盒所有权保护。它的主要保护对象是生成内容及其传播链路，而非对开放权重 checkpoint 提供严格的不可删除标记。

## 3. 方法设计与复现级理解

### 3.1 两阶段总览：先学习 latent codebook，再进行特征级集成

SLIM 的训练由两个阶段组成。第一阶段仅训练水印 encoder $W_{En}$ 与 decoder $W_{De}$，使任意消息 $\omega$ 被映射为与 VAE latent 同形的水印变量 $\Delta z^\omega$，并要求该信号在 VAE 解码和模拟攻击后仍可提取。第二阶段冻结 $W_{En}$、$W_{De}$ 与原始 U-Net，仅训练 LoRA 参数：水印 latent 和正常 noisy latent 并行经过 U-Net 的第一个下采样块，随后在 backbone 与 skip 两条支路上分别相加。

完整数据流为：消息 $\omega$ → $W_{En}$ → watermark latent $\Delta z^\omega$ → DB1 watermark features → 与正常 DB1 features 融合 → LoRA U-Net 去噪 → VAE decoder → 水印图像 → 冻结 $W_{De}$ 恢复消息。

### 3.2 第一阶段：Watermark Encoder–Decoder Pre-Training

给定训练图像 $I_0$，VAE encoder $E$ 得到干净 latent $z_0$；随机消息经过水印 encoder 得到扰动：

$$
\Delta z^\omega = W_{En}(\omega),
\qquad
\hat z_0 = E(I_0)+\Delta z^\omega,
\qquad
I_\omega=D(\hat z_0).
$$

训练时，水印图像先通过复合攻击算子 $\Lambda$，再由水印 decoder 预测消息：

$$
\hat\omega
=W_{De}\!\left(
\Lambda\!\left(
D\!\left(E(I_0)+W_{En}(\omega)\right)
\right)
\right).
$$

$\Lambda$ 包括 identity、JPEG、crop-and-resize、Gaussian blur、Gaussian noise 与 color jitter。它的作用是让 decoder 学到传播鲁棒性，而不是只在无失真图像上记忆 watermark pattern。

论文使用 LPIPS 约束水印图与原图的感知差异，并用 Sliced Wasserstein Distance 对齐颜色分布。式 (1) 可整理为：

$$
\mathcal L_{SW}
=
\frac{1}{10}
\sum_{i=1}^{10}
\sum_{d=1}^{3}
\mathbb E_{x\sim I_\omega,\,y\sim I_0}
\left[
\left|
\operatorname{sort}(R_i x_d)
-
\operatorname{sort}(R_i y_d)
\right|
\right],
$$

其中 $R_i$ 为第 $i$ 个随机旋转/投影矩阵，$x_d,y_d$ 表示第 $d$ 个颜色通道的采样像素。排序操作构成一维最优传输配对，因此该损失比较的是颜色分布，而非逐像素位置。

第一阶段的 bit 恢复损失是二元交叉熵。论文公式省略了对 bit 维度的求和/平均，规范写法为：

$$
\mathcal L_{\mathrm{BCE}}
=-\frac{1}{n}
\sum_{j=1}^{n}
\left[
\omega_j\log\sigma(\hat\omega_j)
+
(1-\omega_j)\log\bigl(1-\sigma(\hat\omega_j)\bigr)
\right].
$$

因此 encoder–decoder 的总目标为：

$$
\mathcal L_{ED}
=\lambda_1\mathcal L_{\mathrm{BCE}}
+\lambda_2\mathcal L_{\mathrm{LPIPS}}
+\lambda_3\mathcal L_{SW}.
$$

训练采用渐进权重：初期使用 $(\lambda_1:\lambda_2:\lambda_3)=(1:0:0)$，先建立可靠解码；5 个 epoch 后改为 $(1:5:100)$，再强化感知质量和颜色分布约束。阶段结束后，$W_{En}$ 与 $W_{De}$ 均被冻结，分别作为第二阶段的固定信号发生器和固定验证接口。

### 3.3 第二阶段：MPE 特征注入与 FCWE 补偿

**MPE（Minimal Perturbation Embedding）。** 正常 noisy latent $\tilde z_t$ 与 watermark latent $\Delta z^\omega$ 分别通过同一个第一个下采样块 DB1。正常分支产生 backbone/skip features $F_{BD},F_{SD}$，水印分支产生 $F_{BW},F_{SW}$。二者逐元素相加：

$$
F_S=F_{SW}\oplus F_{SD},
\qquad
F_B=F_{BW}\oplus F_{BD}.
$$

这里 $\oplus$ 表示 element-wise addition。$F_S$ 继续通过 U-Net 的 skip connection，在后续上采样阶段与 backbone content features 拼接；$F_B$ 则沿主干继续传播。图 4 的核心含义是：SLIM 同时覆盖浅层跳连与深层主干，避免水印信号被任一条路径绕过。

**FCWE（Feature Compensation and Watermark Enhancement）。** 特征融合仍会改变噪声预测，而且第一阶段 decoder 见到的是“直接 VAE latent 扰动”，第二阶段产生的却是“经 U-Net 特征注入形成的水印”，二者存在表示错位。作者因此使用 LoRA 微调 U-Net，使其同时满足生成保真与水印恢复。

首先按 DDPM 前向过程构造 noisy latent：

$$
\tilde z_t
=
\sqrt{\bar\alpha_t}\,z_0
+
\sqrt{1-\bar\alpha_t}\,\epsilon,
\qquad
\epsilon\sim\mathcal N(0,I),
$$

其中 $\bar\alpha_t=\prod_{i=1}^{t}\alpha_i$。带 MPE 与 LoRA 的模型记为 $f$，其单步反向更新写为：

$$
\hat z_{t-1}
=
\frac{1}{\sqrt{\alpha_t}}
\left(
\tilde z_t
-
\frac{1-\alpha_t}{\sqrt{1-\bar\alpha_t}}
 f(\tilde z_t,t,c,\Delta z^\omega)
\right)
+
\sigma_t\delta,
\qquad
\delta\sim\mathcal N(0,I).
$$

冻结的原始 U-Net $\epsilon_\theta$ 作为教师。保真损失对齐同一 noisy latent 上的两种噪声预测：

$$
\mathcal L_F
=
\mathbb E_{t,z_0,\epsilon}
\left[
\left\|
\epsilon_\theta(\tilde z_t,t,c)
-
f(\tilde z_t,t,c,\Delta z^\omega)
\right\|_2^2
\right].
$$

生成的 $\hat z_{t-1}$ 经 VAE decoder 得到 $I_\omega$，再由冻结 $W_{De}$ 输出 $\hat\omega$。这里需要特别区分论文对同一符号 $\mathcal L_\omega$ 的两次定义：式 (3) 的第一阶段损失是 BCE，而第二阶段正文明确称 extraction loss 为消息预测与真值之间的 MSE。因此第二阶段更清晰的写法是：

$$
\mathcal L_{\mathrm{ext}}
=
\frac{1}{n}
\left\|
\hat\omega-\omega
\right\|_2^2,
\qquad
\mathcal L_{FT}
=
\mathcal L_F+\mathcal L_{\mathrm{ext}}.
$$

论文在符号上复用了 $\mathcal L_\omega$，容易误读成第二阶段仍使用 BCE；复现时应以正文、算法实现或源码确认 decoder 输出是 logits、概率还是连续 bit prediction。无论具体实现采用 BCE 或 MSE，第二阶段的功能关系不变：$\mathcal L_F$ 防止模型偏离原始生成分布，$\mathcal L_{\mathrm{ext}}$ 防止 LoRA 通过忽略水印来获得较低的保真损失。

### 3.4 训练、推理与低 timestep 策略

第二阶段对每个图文对 $(I_0,c)$ 执行：

$$
z_0=E(I_0),
\qquad
t\sim\operatorname{Uniform}\{1,\ldots,T/5\},
\qquad
\Delta z^\omega=W_{En}(\omega),
\qquad
f=\epsilon_\theta+\Delta\theta.
$$

随后计算 $\tilde z_t$、$\hat z_{t-1}$、$I_\omega$ 与 $\hat\omega$，并仅对 LoRA 参数 $\Delta\theta$ 反向传播 $\mathcal L_F+\mathcal L_{\mathrm{ext}}$。正文写 $t\in[1,T/5]$，算法 1 写为 $\{0,\ldots,T/5\}$；这是论文内部的边界差异，实际实现应以源码为准。

推理从 $z_T\sim\mathcal N(0,I)$ 开始，并分成两段：从 $T$ 到切换点 $t^*$ 使用冻结原始网络 $\epsilon_\theta$，从 $t^*$ 到 0 才启用 watermark latent、MPE 和微调模型 $f$。这样先由高噪声阶段确定全局布局与语义，再在低噪声阶段写入局部纹理和细节水印。论文后续消融显示，实际最佳设置是在最后 3 个采样步嵌入；若把水印扩展到全部 25 步，训练—推理范围失配会使 FID 从 23.18 恶化到 30.64。

### 3.5 训练配置、攻击配置与复现边界

| 复现项 | 论文配置 |
|---|---|
| 基础模型 | Stable Diffusion 1.5、Stable Diffusion 2.1 |
| 数据 | COCO train2017 10,000 张训练；COCO val2017 5,000 张评估；DiffusionDB 测提示词泛化 |
| 图像/latent | $512\times512$；$4\times64\times64$ |
| 默认载荷 | 48 bit；另测 64、128 bit |
| Encoder/decoder 优化 | AdamW，学习率 $5\times10^{-5}$；5 epoch 后调整损失比例 |
| LoRA | rank 50；插入 Q/K/V/out projection 与 U-Net 全部卷积层；$\alpha=2r$；高斯初始化 |
| LoRA 优化 | AdamW，学习率 $10^{-4}$ |
| 训练 timestep | 低 timestep，约为总扩散步数的后 20%；$T=1000$ |
| 推理 | DPM-Solver multistep，25 steps，CFG 7.5；最佳为最后 3 步嵌入 |
| 设备 | NVIDIA L40，PyTorch |

训练期攻击采样概率为 identity 0.4、JPEG 0.1、crop-resize 0.2、Gaussian blur 0.05、Gaussian noise 0.1、color jitter 0.15。JPEG quality 为 50–70；crop 边长为 125–468 后缩放回原尺寸；blur 标准差为 10；noise 标准差为 0.5；亮度、对比度和饱和度系数为 $[0.5,1.5]$，hue 为 $[-0.25,0.25]$。

评测期则使用不同强度：JPEG Q50、保留 80% 面积的 crop-resize、噪声标准差 0.1、$3\times3$ 且标准差 4 的 blur、较轻 color jitter，以及将四个 $256\times256$ 象限分别逆时针旋转 $10^\circ$ 的局部旋转攻击。训练与测试攻击并非完全同分布，因此鲁棒结果不只是训练增强的原样重放。

## 4. 与其他方法对比

| 类别 | 代表方法 | 水印位置 | 主要问题 | SLIM 的区别 |
|---|---|---|---|---|
| 后处理 | DwtDctSvd、RivaGAN | 最终图像 | 与生成模型脱耦，易受再生成或学习型移除影响 | 在生成过程中写入水印特征 |
| VAE decoder 微调 | Stable Signature | VAE decoder | 依赖特定 decoder，白盒下可被替换 | 不修改 VAE decoder |
| 初始噪声/latent 调制 | Tree-Ring、RingID、G-Shading | 初始噪声或扩散 latent | 可能改变采样轨迹、布局与语义 | 不把 watermark latent 直接作为扩散状态扰动 |
| U-Net 参数水印 | AquaLoRA | Watermark LoRA/U-Net | 水印映射与 LoRA 对齐困难时会影响质量 | 先经 DB1 转成特征，再以 LoRA 做补偿 |
| SLIM | 本文 | DB1 backbone + skip features，晚期 LoRA | 仍需额外水印分支和 U-Net 适配 | 以双支路特征融合降低语义干扰 |

SLIM 与 AquaLoRA 都使用 latent watermark encoder/decoder 和 LoRA，但目标与机制不同。AquaLoRA 通过消息化 LoRA 将用户标识写入定制模型权重，强调白盒模型保护；SLIM 的消息在每次生成时由 $W_{En}$ 输入，并通过 DB1 特征融合进入采样过程，强调图像级鲁棒水印与内容保真。

## 5. 实验表现

论文使用 BITACC 和 TPR@0.1% FPR 衡量提取与检测；使用 PSNR、SSIM、MS-SSIM、PIQE 评估结构和感知质量；使用 FID、IS 评估生成分布；使用 CLIP Score 与 DreamSim 评估文本对齐和同 prompt 内容一致性。

在 DiffusionDB 上，SLIM 的平均攻击后 BITACC 为 96.89%（SD1.5）和 97.39%（SD2.1）；在 COCO 上分别为 96.68% 和 96.28%。论文特别指出，裁剪和局部旋转会显著削弱多种频域或 latent 方法，而 SLIM 的双支路特征注入仍保持较高准确率。TPR@0.1% FPR 的对比也显示其在极低误报要求下具有更稳定的检测率。

质量方面，论文报告 SLIM 相对无水印模型的 FID 开销仅为 +0.10（SD1.5）与 -0.16（SD2.1）。图 5 的相同 prompt 对比中，部分基线会改变口袋、配饰或场景结构，而 SLIM 的结果更接近 clean reference。图 7 的频谱分析显示，水印能量分布于较宽频段但更集中于低频；低频成分提高 JPEG、模糊和几何攻击鲁棒性，高频扰动受抑制则有助于不可感知性。

消融实验给出三条重要结论。第一，去掉 SW loss 会降低颜色和感知质量；加入 SW 后 clean BITACC 从 99.60% 提升到 99.88%，平均攻击 BITACC 也有所提升。第二，仅注入 skip features 有较好质量但鲁棒性较弱，仅注入 backbone features 会产生更多视觉伪影；同时注入两条支路获得最佳提取性能和较平衡的质量。第三，只在第一个 downsampling block 注入优于在多个 block 重复注入，因为后者会造成特征冲突与更高 FID。

LoRA rank 50 在质量与 clean BITACC 上更优，新增 16.82M 参数，约占模型参数的 1.92%；rank 300 的攻击鲁棒性更强，但新增 100.97M 参数，约占 11.55%。不同 scheduler（DDPM、DDIM、DPM-S、DPM-M、Euler、UniPC）和 CFG 下结果接近，说明该水印并非依赖单一 sampler。载荷从 48 增至 64 bit 时性能下降很小，128 bit 仍可工作但 BITACC 略降；论文将 128 bit 视为当前设置下的最高稳定容量。完整流程在 L40、batch size 1、25-step DPM-M 下平均每张图约 1.4632 秒，在其比较方法中最快。

## 6. 学习与应用

SLIM 的最小复现应按模块推进。先固定 SD1.5 与 48-bit 载荷，只训练 $W_{En}/W_{De}$，确认 clean、JPEG 和 crop-resize 下能够稳定恢复；再冻结 codebook，引入 DB1 双支路特征融合与 rank-50 LoRA，只在低 timestep 训练；最后对同 prompt/seed 比较 clean 与 watermarked 输出的 DreamSim、FID、BITACC 和固定 FPR 下的 TPR。

复现中最容易出错的地方有三处：一是第一阶段 BCE 与第二阶段 MSE 的符号复用；二是正文与算法在 timestep 下界上的不一致；三是训练使用 $T=1000$ 的 DDPM 索引，而推理只有 25 个 DPM-M 采样步，需要明确二者的 timestep 映射和最后 3 步的切换规则。论文没有完整公开 encoder/decoder 网络结构和所有训练 epoch 数，必须结合官方代码确认。

## 7. 总结

一句话：**SLIM 先学习可解码的 watermark latent，再将其转为 U-Net 第一个下采样块的 backbone/skip 特征，并用晚期 LoRA 补偿实现高保真、抗失真的生成水印。**

## 9. 复现难度与适合人群

难度为高。实现需要同时掌握 Stable Diffusion 的 VAE latent 管线、U-Net 内部 backbone/skip feature 路径、LoRA 对卷积与 attention projection 的注入、扩散 timestep 映射，以及可微图像攻击。适合扩散模型水印、AIGC 溯源与生成安全研究者。初学者可先复现第一阶段 encoder–decoder，再逐步加入 MPE 和 FCWE。

## 10. 简短全面总结

SLIM 针对潜空间水印造成的生成分布偏移，提出“codebook 预训练 + 特征级注入 + 晚期 LoRA 补偿”的两阶段方案。第一阶段把二进制消息映射为与 VAE latent 同形、经攻击后仍可解码的 watermark latent，并使用 BCE、LPIPS 和 SWD 分别约束容量、感知质量和颜色分布。第二阶段不直接修改 noisy latent，而是让 watermark latent 与正常 latent 并行通过 U-Net 的第一个下采样块，在 backbone 和 skip features 上相加；LoRA 通过噪声预测对齐与消息恢复目标修正分布偏移。实验显示其在 SD1.5/2.1 上具有很小的 FID 开销，对压缩、噪声、模糊、裁剪和局部旋转保持较高 BITACC，并对 scheduler、CFG、容量和分辨率具有一定泛化。核心优势是高保真与图像级鲁棒性的平衡；主要边界是尚未证明对强白盒模型重训或蒸馏攻击不可移除。

## 11. 论文写作逻辑分析

文章按“直接 latent 水印为何改变内容 → 如何把水印转为兼容表示 → 在 U-Net 的哪里注入 → 如何补偿注入扰动 → 是否同时保质量与鲁棒”展开。第一阶段 encoder–decoder 回答“水印表示如何可学习、可解码”，MPE 回答“水印应进入哪个特征位置”，FCWE 回答“如何修复生成偏移并重新对齐 decoder”。实验依次用攻击表、质量指标、频谱分析和模块消融验证这些主张。

写作上需要注意两处不够严谨之处：同一符号 $\mathcal L_\omega$ 在两阶段分别指 BCE 和 MSE，容易造成公式误用；正文和算法对 timestep 的下界也不一致。除此之外，方法—实验之间的对应关系较清晰，尤其是双支路注入、单 DB1 位置和最后 3 步嵌入均有专门消融支撑。

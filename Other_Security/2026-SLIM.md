# SLIM: Stable Latent Integration for Robust Watermark in Diffusion Model

## 0. 摘要翻译

SLIM 提出面向扩散模型的稳健水印方案。作者认为现有潜空间水印常因直接注入扰动而造成分布偏移、语义变化或抗攻击能力不足。SLIM 先训练潜空间水印编码器/解码器，再以水印嵌入模块将水印特征注入 U-Net 的早期下采样层，并用 LoRA 形式的特征补偿与水印增强模块修复扰动、强化可提取信号。实验表明，该方法在图像质量、文本语义一致性、正常生成配置和多类图像攻击下具有竞争力。

## 1. 论文基本信息

- 标题：SLIM: Stable Latent Integration for Robust Watermark in Diffusion Model
- 作者：Xiaoxi Kong, Pengdi Chen, Bin Li, Jieyu Yuan, Zhanchuan Cai, Hao Wu, Lifeng Liang
- 发表：IEEE Transactions on Circuits and Systems for Video Technology，Vol. 36, No. 7，2026
- 代码：<https://github.com/XiaoxiKong/SLIM>
- 对象：文本到图像扩散模型的内生水印。

## 2. 研究背景与动机

后处理水印容易与生成过程脱钩；在扩散过程或 latent 中直接加固定扰动，虽可得到较强提取信号，却会改变去噪轨迹，造成画面质量、分布或 prompt 语义下降。对于模型级水印，理想方案应让水印随生成模型稳定传播，同时将扰动限制在不破坏关键语义的位置。

## 3. 问题定义与威胁模型

给定文本条件 \(c\)、随机噪声和消息 \(\omega\)，模型生成带水印图像 \(x_w\)，解码器应从 \(x_w\) 或其受攻击版本恢复 \(\omega\)。优化目标包括：

1. 图像结构、感知质量、分布及 text-image alignment 接近无水印模型；
2. 在 JPEG、裁剪缩放、模糊、噪声、颜色扰动和旋转等后仍能恢复消息；
3. 在不同 prompt 与常规扩散配置中稳定工作。

攻击者主要是对生成图做后处理/传播失真的内容攻击者。本文没有将任意白盒重训练或专门的模型擦除视为已完全解决的问题，因此不能把其鲁棒性解释为对模型窃取的密码学保证。

## 4. 核心洞见

水印既要在 latent 中可被图像解码器感知，又不能以全局生硬偏移干扰 U-Net 去噪。SLIM 将“生成可解码水印方向”和“让扩散网络稳定携带该方向”拆开：前者由独立的 latent encoder/decoder 学习，后者用较局部的网络注入和 LoRA 补偿实现。

## 5. 方法概述

1. **Watermark encoder/decoder 预训练**：\(WEn(\omega)\) 产生与 VAE latent 同形的 \(\Delta z_\omega\)，将其加入干净 latent 后经 VAE 生成图像；\(WDe\) 从图像/失真图恢复 bit。
2. **感知与分布约束**：以消息 BCE、LPIPS、Sliced Wasserstein Distance 及攻击增强共同训练，逐步调整损失比例以兼顾可见性与鲁棒性。
3. **MPE（watermark embedding module）**：不直接在完整 U-Net 输出端粗暴叠加，而在第一个下采样块的输出特征注入水印信息，降低对高层语义的干扰。
4. **FCWE（feature compensation and watermark enhancement）**：对 U-Net 加 LoRA，补偿注入导致的生成偏差，并增强水印特征沿去噪链路的保留。

## 6. 关键技术细节

- 主实验使用 48-bit 随机二进制串，并以**全部 bit 精确恢复**作为成功解码条件。
- 训练 watermark encoder/decoder 时采用 identity、JPEG、crop-resize、Gaussian blur、Gaussian noise、color jitter 的随机增广；其中 identity 概率 0.4，其他变换用于把鲁棒性预先写入解码器。
- 基座为 SD1.5 与 SD2.1，图像/latent 尺寸为 \(512\times512\) / \(4\times64\times64\)。模型微调用 rank 50 LoRA，覆盖 U-Net 的 Q/K/V/out 投影及所有卷积层。
- 推理采用 CFG 7.5、25 步 DPMSolver multistep。该配置应与训练和对比方法的默认配置区分，不能据此推断所有采样器下必然等效。

## 7. 实验设置

- 数据：COCO train2017 中 10,000 图训练；COCO val2017 中 5,000 图评估，并在 DiffusionDB 验证 prompt 泛化。
- 对比：后处理 DwtDctSvd、RivaGAN；过程内 Stable Signature、Tree-Ring、Robin、RingID、G-Shading、AquaLoRA。
- 载荷：SLIM、Stable Signature、G-Shading、AquaLoRA 均为 48 bit，DwtDctSvd/RivaGAN 为 32 bit。
- 指标：BITACC、\(\mathrm{TPR}@0.1\%\mathrm{FPR}\)，PSNR/SSIM/MS-SSIM/PIQE，FID/IS，以及 CLIP Score、DreamSim。
- 攻击：JPEG（Q=50）、80% crop-resize、Gaussian blur、10% Gaussian noise、color jitter 与旋转等。

## 8. 实验结果与分析

- 论文以多维质量指标而非单一 PSNR 评估水印影响，特别加入 CLIP Score 与 DreamSim 来检查文本语义和生成感知相似性。
- 鲁棒性表在 COCO 与 DiffusionDB 上报告各种攻击的 BITACC，说明作者关注不同 prompt 分布而非只在单一生成集上测量。
- 相比后处理方法，过程内方法可以把信号绑定到生成；但不同过程内方法在质量、容量、裁剪和再生攻击间存在权衡。SLIM 的 MPE+FCWE 组合正是为缓和该权衡设计。
- 需要注意：严格 exact-match 的 48-bit 成功定义比逐位准确率更苛刻；阅读结果时应同时查看 BITACC、TPR@FPR 与整串恢复率，避免仅由高平均 bit accuracy 推断高消息成功率。

## 9. 优点与局限

优点：将可解码 latent 模式与扩散网络集成分离，设计清晰；早期特征注入和 LoRA 补偿直接面向保真—鲁棒冲突；评测覆盖质量、分布、语义和攻击；与 AquaLoRA 等内生方法进行同载荷比较。

局限：训练涉及独立编码器/解码器和 U-Net 微调，成本高于纯后处理；对攻击分布有明显训练依赖；论文重点为图像后处理鲁棒性，对恶意权重编辑、模型蒸馏或替换生成器的结论有限；水印解码器的保密、阈值校准和跨模型迁移需要实际部署另行设计。

## 10. 可复现性与使用建议

建议严格复现 COCO/ DiffusionDB 划分、48-bit exact-match 定义和攻击强度；分别报告 clean、每种攻击和未见 prompt 的曲线。消融应至少移除 MPE、FCWE、攻击增广与逐步损失调度，才能判断哪一组件贡献了质量或鲁棒性。若用于版权场景，还需追加模型重训练、LoRA 合并与生成模型替换攻击。

## 11. 论文写作逻辑分析

论文以“潜变量水印带来分布/语义不稳”为切入点，构造预训练水印模式，再把问题收束为“该模式如何不破坏地穿过 U-Net”。MPE 负责选择注入位置，FCWE 负责网络适配，实验依次验证保真、语义、容量与攻击鲁棒性。论证主线较完整，但模型级对手的能力边界需与内容级失真对手明确区分。

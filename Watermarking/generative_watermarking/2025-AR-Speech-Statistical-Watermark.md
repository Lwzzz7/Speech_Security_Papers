# 自回归语音生成模型的统计水印

## 0. 摘要翻译

本文首次把语言模型的统计水印应用到自回归语音生成：在每一步音频 token 采样时，用密钥和前文生成红/绿 token 划分，再通过 KGW 或无失真的 DiPmark 重加权，使最终 token 序列具有可检验的统计偏差。作者同时指出一个语音特有难点：检测只能从波形重新编码 token，而“生成 token→波形→重新编码 token”并非可逆，重编码错位会显著削弱水印检测。

## 1. 方法动机

后处理水印修改波形、增加生成后步骤，且其置信度通常依赖经验阈值。自回归语音模型本来就逐 token 采样，因此可以像文本语言模型一样在采样分布中植入密钥相关偏差，不改模型参数，也不必额外修改波形。困难在于文本 tokenizer 近似可逆，音频 codec 却会把同一连续声学区域量化为不同 token；检测者看不到生成时的原 token，只能面对重新编码后的序列。

## 2. 威胁模型解读

服务提供者控制 SpiritLM 推理并持有秘密密钥；用户得到合成波形。检测者拥有密钥、重加权规则和音频编码器，不需要访问生成模型 logits。安全目标是以受控 FPR 判断音频是否由带水印采样器产生，同时尽量保持生成分布。本文主要研究自然发生的重编码错位，没有评估恶意攻击者的裁剪、变速、压缩、伪造、密钥查询或自适应水印移除；因此它更接近“统计检测可行性与问题发现”，不是完整鲁棒安全方案。

## 3. 方法设计与理解

### 3.1 生成与检测闭环

每一步先由自回归模型计算下一个 token 的原分布。密钥与前缀 2-gram 经伪随机函数生成词表排列，将前半视作红表、后半视作绿表；重加权后采样 token。序列最终经 vocoder 解码成波形。检测时波形经音频编码器重新量化，检测器用重编码后的前缀重建每一步排列，累计落在绿表中的 token 数，再做假设检验。

### 3.2 KGW 与 DiP 重加权

KGW 给绿表 token 的 logits 加 δ，δ 越大越容易检测，但真实采样分布偏移越大。DiPmark 把词表排列后的累积分布映射到 [0,1]，用 α∈[0,0.5] 将两端概率质量重新分配；对随机水印码取期望时恢复原模型分布，因此称为 distortion-free。它保证的是随机密钥下的边际 token 分布，而不是同一 prompt、同一种子产生完全相同波形。

### 3.3 统计检验与错位

若无水印，token 落入绿表的概率按 0.5 建模。对长度 n 序列，检测分数是绿 token 数相对 0.5n 的标准化偏差，并用 Hoeffding 界控制尾概率。关键问题是检测序列为 E(D(x)) 而非生成序列 x：token 替换不仅改变当前绿/红归属，还改变后续 n-gram 产生的伪随机排列，错误会沿上下文传播。

### 3.4 复现配置

| 项目 | 设置 |

| 模型 | SpiritLM |
| 数据 | C4 子集、Dolly-CW、Longform QA、MMW Story |
| 每任务样本 | 1,000 |

## 4. 与其他方法对比

与波形后处理相比，统计水印直接作用于采样分布，DiP 在理论上保持边际分布并提供显式假设检验。与文本 KGW/DiP 的直接移植相比，本文的贡献并非新重加权公式，而是建立音频 token 生成—波形—重编码的完整框架，并量化重编码错位这一特有瓶颈。后续 ALIGNED-IS 正是针对该瓶颈设计 token 聚类。

## 5. 实验表现

### 5.1 可检测性与质量权衡

C4 上 KGW δ=1/1.5/2 的 TPR@1%FPR 为 54.3%/92.7%/98.9%，FAD 从 0.0125 升至 0.0494；DiP α=0.3/0.4/0.5 的 TPR 为 38.0%/47.5%/46.4%，FAD 为 0.0088/0.0067/0.0079。Longform QA 上 KGW δ=2 达 99.5%，但 FAD 0.0913；DiP 最高 55%，FAD 约 0.021–0.028。结果验证“偏置换检测、无失真牺牲统计强度”。

### 5.2 重编码错位影响

C4 上 KGW δ=1 的 TPR@1%FPR 从原 token 的 97.1% 降至重编码后的 54.3%，DiP α=0.4 从 80.0% 降至 47.5%。Dolly-CW 更严重：KGW δ=1 从 99% 降至 17%，DiP α=0.4 从 99% 降至 53%。这一对照是全文最强证据：检测下降主要不是统计检验失效，而是 detector 看到的 token 序列已改变。

## 6. 学习与应用

实现时应把重加权写成 logits processor，并保存生成 token 用于诊断“理论检测”和“波形检测”的差距。该方案适合受控自回归语音 API 的零比特来源检测；若要真实传播鲁棒性，需要加入 token 等价类、同步机制或与波形水印组合。

## 7. 总结

首次揭示音频统计水印的重编码错位。

## 9. 复现难度与适合人群

难度中等：算法简单，但 SpiritLM 环境、音频编码器和数据 prompt 处理需对齐，且没有本文独立代码。最小版本可在一个 SpiritLM 数据集上实现 KGW/DiP logits processor，同时记录原 token 与重编码 token。适合研究统计水印、自回归音频与后续 ALIGNED-IS 的读者。

## 10. 简短全面总结

本文把 KGW 和 DiPmark 的 token 分布重加权移植到 SpiritLM：生成时由密钥与前缀决定红绿表，检测时从波形重新编码 token、累计绿表偏差并做带 FPR 控制的假设检验。实验显示 KGW 可通过更强偏置获得接近 99% 的检测率，但 FAD 上升；DiP 保持较低 FAD，却更难检测。最关键发现是重编码错位可使 TPR 从接近 100% 降至 17%–54%，为后续聚类对齐方法明确了问题，但本文尚未解决该问题或覆盖真实攻击。

## 11. 论文写作逻辑分析

文章先建立统计水印相对后处理的优势，再引入音频 token 不可逆这一机制差异；方法本身沿用两种基线，实验则把“能否检测”和“为什么下降”分成两组。写作价值在于把负面发现提升为明确研究问题。局限是结论措辞略强：全文实际证明的是初步可行性与错位影响，而不是已获得鲁棒、无失真的完整方案。

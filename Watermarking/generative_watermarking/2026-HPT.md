# Hidden in Plain Tokens：面向合成音频的简单、鲁棒、免梯度水印

## 0. 摘要翻译

自回归音频生成模型通常在离散声学 token 上进行采样，因此可以借用语言模型的统计水印：根据密钥把词表划分为“绿名单”和“红名单”，在生成时提高绿名单 token 的概率，再通过显著性检验判断音频是否带水印。然而，生成 token 经声码器解码为波形、再被同一音频编解码器编码时，往往不能恢复成原 token，形成 **retokenization mismatch（重分词错配）**，从而破坏水印。已有方法通过微调 codec 提高幂等性，但需要访问参数、梯度和训练资源。

本文提出 Hidden in Plain Tokens（HPT）：先统计 codec 在“编码—解码—再编码”过程中真实发生的 token 混淆，把每个 RVQ 通道的 token 构造成有向加权混淆图，再用 Leiden 社区发现将容易互相混淆的 token 划入同一簇。生成和检测不再围绕单个 token，而是围绕较稳定的 token cluster 运行 KGW 水印。该方法只需黑盒调用 codec，离线建图后推理阶段仅增加一次 token-to-cluster 查表，不训练生成模型或 codec。实验覆盖 Moshi、MusicGen、CosyVoice3 和 Spark-TTS；结果表明 HPT 在多种信号处理和神经编解码攻击下显著增强水印证据，同时大体保持语音质量，但对变速、裁剪和帧错位仍较敏感，在 MusicGen 上也存在明显的质量代价。

## 1. 方法动机

直接把文本生成水印移植到音频 token 的关键障碍不是“模型不能生成带偏置的 token”，而是检测端看到的 token 已经变了。设生成序列为 $x$，音频波形重新编码后为 $y$；若检测器只承认 $y_i=x_i$，即使两者声学上几乎等价，水印命中也会丢失。更严重的是，KGW 常用前 $h$ 个 token 生成当前位置的绿名单：一个 token 错配不仅影响当前位置，还会改变后续哈希上下文，形成连锁失配。

作者的核心观察是：codec 的错误具有结构，而非均匀随机。某个 token 往往只会被重编码为少数语义或声学近邻。如果把这些高频互混 token 视为同一等价类，那么单 token 不一致时，cluster 身份仍可能保持。HPT 因而把问题从“如何训练一个更幂等的 codec”改写为“如何利用 codec 已有的混淆结构设计容错水印”，以恢复统计水印原本训练自由、部署轻量的优势。

## 2. 威胁模型解读

水印提供者能够控制自回归音频模型的采样过程并持有密钥，也能够以黑盒方式调用生成系统所用的音频 tokenizer/codec，收集少量代表性音频完成离线聚类；它不需要 codec 权重、梯度或再训练数据。检测者持有同一密钥、codec 和 token-to-cluster 映射，把待测波形重新编码后进行统计检验。

攻击者可获得最终音频并实施 no-box 后处理，包括平滑、高低通滤波、高斯噪声、MP3、神经 codec 重编码、裁剪、帧移位和变速。论文没有系统评估知道算法或密钥结构的白盒自适应攻击，也没有覆盖密钥恢复、伪造、共谋、多重攻击叠加及专门的同步破坏。因此，本文证明的是常见音频失真下的来源可检测性，而不是完整密码学安全性。其安全边界还依赖检测端能使用与生成端相匹配的 tokenizer 及聚类映射。

## 3. 方法设计与理解

### 3.1 从 codec 往返误差构造混淆图

对一批代表性音频先编码得到原 token，再将 token 解码成音频并重新编码，获得位置对齐的“原 token—重编码 token”样本。对每个 RVQ 通道分别累计混淆矩阵 $M$，其中 $M_{ij}$ 表示 token $i$ 在往返过程中变成 token $j$ 的次数。矩阵被解释为有向加权图：节点是码本 token，边权是真实 codec 错配频率。

作者先用最小计数阈值 $m$ 删除罕见边，避免偶然错误支配社区结构；再以分辨率参数 $\alpha$ 控制社区粒度，并采用 Leiden 社区发现得到 token-to-cluster 映射。相比只依据码本 embedding 做 k-means，这种建图直接利用实际重编码行为；相比 Louvain，Leiden 能处理有向图、保证社区连通，并在作者实验中更快且效果更好。每个 RVQ 通道独立聚类，因为不同层的码本和混淆模式不同。

### 3.2 在 cluster 层执行 KGW

标准 KGW 用密钥和前 $h$ 个 token 的上下文生成伪随机划分，把比例为 $\gamma$ 的 token 放入绿名单，并给绿 token 的 logit 增加偏置 $\delta$。HPT 改为先划分 cluster 词表：若某个 cluster 为绿色，则该簇内所有 token 同时获得偏置。检测时，重新编码 token 只要仍落入同一绿色 cluster 就计为命中，不要求其 token ID 完全相同。

当 $h>0$ 时，哈希上下文也由 token ID 改为 cluster ID。这样，原 token 被 codec 替换成同簇 token 后，不仅当前位置仍能被识别，后续位置的绿名单也不会因上下文变化而整体漂移。在线额外成本只是 cluster 查询和簇级掩码；混淆采集与 Leiden 聚类均在部署前完成。

### 3.3 理论解释：为什么簇级匹配能放大证据

设单 token 精确重现率为 $r$，无水印时绿名单比例为 $\gamma$，带水印生成时实际绿 token 概率为 $g>\gamma$，序列长度为 $N$。由于当前位置及其 $h$ 个上下文都要保持一致，攻击后观察到绿色的概率为

$$p_1=\gamma+r^{h+1}(g-\gamma).$$

因此检测统计量的期望为

$$\mathbb{E}[z\mid H_1]=\sqrt{N}\frac{g-\gamma}{\sqrt{\gamma(1-\gamma)}}r^{h+1}.$$

式中的 $r^{h+1}$ 揭示了 retokenization mismatch 为何会随上下文长度指数恶化。HPT 定义 cluster 保持率 $r_{cl}=P[C(y_i)=C(x_i)]$；由于同簇替换不再算错误，通常有 $r_{cl}>r$，上式中的衰减项变为 $r_{cl}^{h+1}$。附录在 MusicGen 的 $h=0,1,2$ 设置中比较理论与实测 $z$，相关系数分别为 0.9465、0.8535、0.8872，支持这一机制解释。

cluster 不能无限合并：簇越少，保持率越高，但可用的随机状态和生成熵越低，密钥或上下文发生碰撞的概率越大。论文给出约束 $(c|V|)^h\ge K_{min}$，其中 $c|V|$ 是聚类后的词表规模，但没有求解鲁棒性、熵和碰撞风险的严格 Pareto 最优点，而是通过验证集网格搜索选择参数。理论还假设命中事件条件独立；对不重叠窗口该假设可成立，对滑动上下文则只是近似。

## 4. 与其他方法对比

与 Base KGW 相比，HPT 保留相同的 logit 偏置和显著性检验框架，区别是把 token 词表及哈希上下文都提升到混淆簇层级；它牺牲部分细粒度状态空间，换取更高的重编码保持率。与 WMAR 相比，WMAR通过微调 codec 使 decode—encode 更接近幂等，需要白盒参数、梯度和训练，而 HPT 接受 codec 原有错误，用黑盒往返样本学习其混淆结构。

与 ALIGNED-IS 相比，两者都用 cluster 缓解 retokenization mismatch，但证据和采样机制不同：ALIGNED-IS 对码本 embedding 做 k-means，并用 aligned inverse sampling 保持原始分布；HPT 从真实混淆边做 Leiden 社区发现，再对绿色 cluster 加 logit 偏置。HPT 不是严格无失真的采样方法，却以更简单的 KGW 机制换取强检测信号。与 AudioSeal、WavMark 等后处理波形水印相比，HPT 必须在生成阶段控制 logits，但无需生成后再运行嵌入网络。

## 5. 实验表现

### 5.1 Moshi：检测证据显著增强

在 conversational prompts 的无攻击设置中，平均 $-\log p$ 从 Base 的 8.51、WMAR 的 17.44 和 WMAR(aug) 的 13.72 提升到 HPT 的 42.47；在 LibriSpeech prompts 上，相应数值为 3.61、7.95、5.98 和 22.77。论文以 $-\log p=2$ 对应 1% FPR，因而 HPT 的无攻击检测裕量明显更大。

对 conversational prompts，HPT 经平滑、低通、高通、噪声和 MP3 后分别为 32.68、41.51、25.59、20.59 和 41.26；经 DAC、EnCodec、SpeechTokenizer、FaCodec 后仍为 40.13、32.64、35.92、38.47。裁剪、帧偏移和变速更困难，但仍有 16.48、27.67、26.49。LibriSpeech prompts 下无攻击为 22.77；噪声降至 9.19，裁剪为 9.70，变速为 15.54，依然高于论文采用的 1% FPR 阈值。结果支持“混淆簇吸收 codec 替换”的主张，也暴露时间轴修改比频谱或压缩修改更危险。

### 5.2 音质与语音可懂度

在 conversational prompts 上，无水印的 FAD-VGGish/FAD-CLAP 为 0.080/0.023，NISQA/DNSMOS 为 3.54/4.43；HPT 在 $h=1$ 时为 0.051/0.015 和 3.58/4.50，未表现出系统性退化。LibriSpeech 中，无水印为 1.921/0.063 和 3.15/3.79；HPT $h=1$ 为 1.921/0.074 和 3.22/4.03，也大体接近基线。部分指标出现小幅改善更可能来自抽样波动，不能解读为水印提升了音质。

CosyVoice3 上，无水印的 FAD-VGGish/FAD-CLAP、NISQA/DNSMOS、WER/CER 为 0.0964/0.0235、3.86/2.73、0.0323/0.0178；HPT 为 0.1942/0.0294、3.81/2.74、0.0519/0.0308。Spark-TTS 上，无水印对应 0.2057/0.0401、3.37/2.94、0.0097/0.0018，HPT 为 0.3506/0.0472、3.46/2.96、0.0099/0.0024。可懂度基本保持，但 FAD 并非完全无损。

### 5.3 TTS 与 MusicGen 的外推结果

CosyVoice3 的平均 $-\log p$ 从 Base 的 1.564 提升到 HPT 的 13.927；Spark-TTS 从 9.237 提升到 17.806，说明社区结构可扩展到“自回归核心 + flow matching 精炼”的 TTS 系统，而不仅是纯 codec 语言模型。

MusicGen 无攻击时，Base、WMAR、HPT 的 $-\log p$ 分别为 9.05、9.06、28.46。HPT 经低通、噪声、MP3、DAC、SpeechTokenizer、FaCodec 后为 25.55、24.92、27.86、19.27、13.75、14.92；但高通后仅 0.72，低于 1% FPR 对应的 2，构成明确失败案例。质量代价也更突出：无水印、Base、WMAR、HPT 的 FAD-VGGish 分别为 0.247、0.330、1.193、1.256；HPT 的 FAD-CLAP 为 0.082，虽优于 WMAR 的 0.132，却仍差于无水印的 0.039。因而不能把 Moshi 上的低失真结论无条件推广到音乐生成。

## 6. 学习与应用

HPT 最适合以下条件：能够修改生成 logits，能够调用固定 codec，且希望避免微调 codec 或训练新的嵌入/检测网络。工程落地时应为每个 codec、每个 RVQ 通道单独采集往返样本和聚类，并保存版本化的 cluster 映射；codec 升级后应重新评估混淆图，不能默认旧映射继续有效。检测端可把多通道证据合并，论文理论上将各通道信号相加、方差相加，以利用 RVQ 的多个码本。

若产品场景常见剪辑、变速或未知起点，应额外加入同步处理。论文附录建议在检测时搜索零填充偏移、估计或枚举速度，或与同步水印组合。对于短音频、未知 tokenizer、强高通音乐处理和自适应攻击，现有证据不足，不宜只依赖 HPT 作最终归因。

## 7. 总结

用 codec 的真实混淆社区替代精确 token，修复生成式音频统计水印的重编码错配。

## 8. 图表精读与证据链

论文的证据链较完整：首先用 codec 往返混淆图说明错误具有局部结构；随后用理论式证明检测信号按 $r^{h+1}$ 衰减，并说明 cluster 保持率为何能替换精确匹配率；再通过 Moshi、MusicGen 和两种 TTS 展示跨模型有效性；攻击表将频谱处理、传统压缩、神经 codec 与时间轴攻击分开；质量表同时给出感知、语音质量和 ASR 指标；附录再验证理论 z-score 与实测值的相关性。

仍有三处证据缺口。第一，聚类数据规模与来源已给出，但缺少跨语种、跨领域采集对社区稳定性的系统分析。第二，攻击主要是单一 no-box 后处理，没有自适应删除与伪造。第三，MusicGen 的高通失败和 FAD 退化表明“鲁棒且低失真”依赖模型与模态，论文标题中的简单和免梯度成立，但鲁棒性不能理解为对所有音频变换都成立。

## 9. 复现难度与适合人群

复现难度中等。官方仓库已公开 Moshi/Mimi 与 MusicGen/EnCodec 的实现、预计算 cluster 映射、评测脚本，并补充 CosyVoice3、Spark-TTS 扩展；环境以 Python 3.11 为主。最小复现可先使用发布的映射，在 Moshi 上比较 Base KGW 与 HPT 的无攻击及 EnCodec 重编码 $-\log p$；完整复现则需重新采集 encode—decode—reencode 对、构造每通道混淆矩阵并运行 Leiden，再按表中参数复现全部攻击和质量评测。

适合研究自回归音频生成、神经音频 codec、统计水印和内容溯源的读者。若目标是波形后处理水印、不可控制 logits 的闭源生成服务，或要求密码学级抗伪造保证，则该方法不直接适用。

## 10. 简短全面总结

HPT 针对生成式音频水印最关键的 retokenization mismatch，不再要求 codec 精确恢复原 token，而是通过黑盒往返样本建立 token 混淆图，用 Leiden 把频繁互换的 token 合并为社区，并在社区层完成 KGW 的绿名单划分、上下文哈希和检测。理论上，这把检测信号中的精确匹配率 $r^{h+1}$ 替换为更高的 cluster 匹配率 $r_{cl}^{h+1}$；实验上，Moshi conversational prompts 的无攻击平均 $-\log p$ 达 42.47，显著高于 Base 和 WMAR，并在传统压缩与多种神经 codec 后保持较强证据。方法无需梯度或训练，在线仅增加查表，但每个 codec 需要离线重建社区，时间轴错位仍是主要弱点；MusicGen 的高通失败和 FAD 退化说明其鲁棒性与低失真并非跨模态无条件成立。

## 11. 论文写作逻辑分析

本文采用清晰的“失败机制—结构观察—最小改造—理论解释—跨模型验证”路径。它先把已有 token 水印失效归因到可量化的重编码错配，再从混淆矩阵中发现社区结构，因而 Leiden 聚类不是孤立技巧，而是直接对应问题机制。方法部分只改动 KGW 的词表单位与哈希上下文，突出简单、免梯度和低在线开销；理论部分用 $r^{h+1}$ 把上下文连锁失配压缩成一个直观公式；实验则依次回答检测是否增强、攻击后是否保持、质量是否可接受、能否外推到 TTS 和音乐。

值得借鉴的是作者没有只报告成功项：时间轴攻击、MusicGen 高通和质量退化均被保留，使适用边界可见。仍可改进之处是把安全性主张收得更窄，并增加自适应攻击、跨域聚类稳定性和社区数选择的系统研究；目前分辨率与阈值主要靠网格搜索，理论给出了约束，却尚未给出可直接指导部署的最优选择准则。

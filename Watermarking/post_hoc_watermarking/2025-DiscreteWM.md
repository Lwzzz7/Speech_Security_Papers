# DiscreteWM：在离散中间表示中嵌入语音水印

## 0. 翻译摘要原文

语音水印通过嵌入人耳难以感知但算法可检测的信号，可主动缓解即时声音克隆的潜在危害。已有方法通常在连续空间写入消息，而在稳健的离散潜空间中嵌入有望显著提升鲁棒性。DiscreteWM 使用向量量化自编码器把语音映射到离散潜空间，再通过改变离散 ID 的模运算关系写入水印；为保持不可感知性，另训练操纵器选择适合替换的候选 token。实验显示该方法在鲁棒性与音质上取得较好平衡。帧级方案既可用于声音克隆检测，也可用于信息隐藏，并能在一秒语音中编码 1–150 bit。

## 1. 方法动机

连续扰动水印在波形或谱上学习细小模式，容易被噪声、滤波和重合成抹除；固定长度模型还需滑窗定位，延迟较高。离散声学 token 本身经过量化与重建，若消息被绑定到 token ID 的奇偶性，解码器只需恢复稳定的类别关系，而非精确重建微小连续值。

困难在于“任意换成同奇偶 token”会破坏语义和静音。论文因此把问题拆为：先训练可重建且奇偶可恢复的离散空间，再用上下文概率操纵器从目标奇/偶集合中选择最自然的替代 token。

## 2. 威胁模型解读

- 发布者或生成服务对语音做后处理水印；验证端可做帧级定位、消息恢复或统计存在性检测。
- 信道攻击包括高斯噪声、幅度缩放、重采样、MP3、中值/低通和回声；论文也以 NaturalSpeech 2、Mega-TTS 2 生成语音验证 AI 内容标记流程。
- 论文主要假定攻击者不知道或不使用 VQ codebook、操纵器和恢复器进行白盒优化；没有测试针对 token 奇偶规则的自适应攻击。若规则公开，攻击者可尝试再次量化并翻转/随机化 parity。
- 信息隐藏时要恢复具体 bit；AI 生成检测时只统计多少帧呈现“由 VQ-VAE 重建并加水印”的特征，两者安全目标不同。
- 局部帧水印提高未知长度处理效率，但对非常短语音，Z 检验样本量不足；阈值依赖训练/测试估计的帧级 FPR。

## 3. 方法设计与复现级解读

### 3.1 全局流程与两阶段训练

第一阶段训练 VQ-VAE，把幅度谱映射到离散 token，并联合训练帧级 localizer、token parity restorer 和攻击模拟器，使重建/攻击后仍可判断哪些帧被重建、对应 ID 是奇是偶。第二阶段冻结或复用离散表示，训练并行掩码预测操纵器，根据上下文为待嵌入位置预测 token 分布。推理时只替换部分 token：bit 0 选偶 ID，bit 1 选奇 ID；解码器重建波形，验证端重新提谱并恢复帧位置与 parity。

### 3.2 离散水印规则

波形 $y$ 经 STFT 得幅度谱 $s$，VQ 编码器生成 token $z^{(t)}$ 及 ID $c^{(t)}$。若第 $t$ 帧要写入 0/1，就选择与原 token 语义相近且满足 $c^{(t)}\bmod2=w_t$ 的新 token。水印 token 经解码器和原相位 ISTFT 得水印波形。

这一规则使消息读取变成二分类，但安全性来自模型与载体约束，不是奇偶运算本身。若不限制替换距离，离散化反而会造成突变，因此操纵器是音质关键。

### 3.3 掩码式 VQ-VAE

训练时随机生成帧掩码 $\omega$，只重建掩码区域，未掩码谱直接从原输入旁路：

$$\bar s=\omega\,G(\omega E(s),(1-\omega)s)+(1-\omega)s.$$

掩码比例 $\gamma\in[0.1,0.5]$。损失由多分辨率 STFT 重建、标准 VQ codebook 损失和多周期/多尺度判别器对抗损失构成，$\lambda_{adv}=10^{-2}$。量化器采用 clustering vector quantizer，提高 codebook 使用率并降低重建误差。

### 3.4 Localizer 与 Restorer

Localizer $D$ 以帧级 BCE 区分原始旁路帧与 VQ 重建帧，取代 WavMark 式逐位置 BFD。Restorer $R$ 从攻击后幅度谱恢复 token ID parity，其交叉熵为

$$L_{res}=\mathbb E_{\tilde s\sim p(\tilde s)}[-\log p(\hat c\bmod2)].$$

攻击模拟器位于水印波形与二者之间，迫使定位和 parity 恢复对常见变换稳定。Localizer 检测的是“重建帧”特征，因此 AI 检测模式不要求固定消息内容。

### 3.5 概率操纵器

操纵器以未掩码 token 为上下文，并行预测被掩码 token：

$$P(\omega z\mid(1-\omega)z;\theta_M).$$

嵌入时若原 ID 已满足 bit 奇偶关系则保留；否则只在所需奇/偶候选集合中依据 softmax 概率采样或选高概率 token。替换非自回归执行，效率高，也利用上下文避开“静音 token 换成语音 token”等严重错误。

### 3.6 两种推理模式

信息隐藏模式把消息顺序分配给选中帧并恢复具体 parity；24 kHz、hop 80、最大 50% 掩码时，一秒理论可在 1–150 bit 间调节。AI 检测模式只在比例 $m$ 的帧上加水印，统计 Localizer 标为水印的帧数 $|f|_w$，以干净帧误报率 $\beta$ 构造

$$Z=\frac{|f|_w-\beta T}{\sqrt{\beta(1-\beta)T}}.$$

例如 $T=200,\alpha=95\%,\beta=10\%,m=10\%$ 时 $Z\approx4.01$，单侧假阳性约 $3\times10^{-5}$。这是基于独立/稳定帧误报近似，实际帧间相关可能使理论 p 值偏乐观。

## 4. 与其他方法对比

| 方法 | 表示/定位 | 优点 | 局限 | DiscreteWM 改进 |
|---|---|---|---|---|
| DeAR/连续 DNN | 波形或频谱连续扰动 | 结构直接 | 固定长度、连续模式脆弱 | token parity |
| WavMark | 可逆连续表示+BFD | 高容量 | 滑窗定位慢、模式位占容量 | 帧级 Localizer |
| SeamlessWM | 生成管线水印 | 适合 AI 检测 | 代码/权重不可得 | 同时支持信息隐藏 |
| DiscreteWM | VQ token 奇偶+操纵器 | 容量可调、定位快 | 依赖 codec 重建，规则可被针对 | 本文方案 |

## 5. 实验表现与优势

### 5.1 信息隐藏的音质、容量与常规攻击

在一秒、32 bps 下，DiscreteWM 的 PESQ/SNR 为 4.45/38.08 dB，八类条件平均 BER 0.28%；WavMark 为 4.31/38.61 dB、2.58%，DeAR 为 3.75/26.31 dB、0.54%。本文比 WavMark 鲁棒、比 DeAR 音质高，体现离散表示的折中，而非每项单独绝对最优。

### 5.2 AI 生成检测与效率

utterance-level 模式中，DiscreteWM 的 PESQ/SNR/平均 BER/RTF 为 4.37/38.01/0.32%/0.0044；WavMark 为 4.24/37.92/1.02%/0.1438，慢约 22.1 倍；复现的 SeamlessWM BER 更低 0.18%，但 PESQ/SNR 仅 3.77/29.62。该实验验证帧级定位既快又保持音质。

### 5.3 容量与模块消融

10、32、50、150 bps 的 PESQ 为 4.47、4.45、4.27、3.92，平均 BER 均约 0.27%–0.31%，高容量主要牺牲音质。去掉 VQ 改为连续表示后 PESQ/SNR/BER 为 4.32/34.90/2.39%，证明离散中间表示贡献最大；不用操纵器随机换 token 时降到 3.96/29.55/0.95%，说明上下文选 token 对音质和恢复都重要。

水印比例 3% 时 $Z=4.07$、理论 FPR $2.3\times10^{-5}$；比例至少 10% 且阈值 4 时，报告 utterance TPR/FPR 为 1.0/0.0。该结果应与测试集校准和帧相关性一起解释，不能把理论 p 值视为所有分布下固定误报率。

## 6. 学习与应用

复现优先检查 codebook ID 排列是否带来奇偶语义偏差，以及操纵器是否真正限制到相近 token。部署中可随机置换 codebook ID 或引入密钥化模分组，降低公开 parity 规则被直接攻击的风险。框架还可迁移到神经音频 codec token、TTS 声学 token 和音乐离散表示，但需重新训练 Localizer/Restorer 适配对应解码器与信道。

## 7. 总结

用离散令牌奇偶关系承载水印。

## 9. 复现难度与适合人群

难度高，需训练 VQ-VAE、两类判别器、Localizer、Restorer 和 masked-token 操纵器。最小版本可先在 LibriTTS 子集训练小 codebook，只复现 32 bps、无攻击及 GN/MP3。适合离散语音表示、神经 codec 水印与 AI 内容检测研究者。

## 10. 简短全面总结

DiscreteWM 把水印从连续波形扰动转移到 VQ-VAE 的离散声学 token：每帧 ID 的偶数/奇数分别表示 0/1，概率操纵器依据上下文从目标奇偶集合中选择最自然替代 token；Localizer 找到重建帧，Restorer 在常规攻击后恢复 parity。帧级结构无需昂贵滑窗，并支持一秒 1–150 bit 的动态容量；存在性检测还可用 Z-test 汇总局部命中。LibriTTS 实验中 32 bps 获得 PESQ 4.45、平均 BER 0.28%，速度显著快于 WavMark，离散表示与操纵器消融均有明显贡献。主要边界是依赖神经重建、论文训练细节不完整，且公开奇偶规则下的自适应移除尚未评估。

## 11. 论文写作逻辑分析

论文从连续水印和滑窗定位两个缺口引出“离散、帧级”解法，再列出重建接近、替换接近、攻击后可恢复三项 prerequisite，方法模块与三项条件对应清楚。实验按信息隐藏、AI 检测、容量/表示/操纵器消融展开，论证紧凑。值得借鉴的是用统一离散机制支持两个应用；不足是安全威胁模型不显式，奇偶规则一旦公开的攻击面没有进入实验。

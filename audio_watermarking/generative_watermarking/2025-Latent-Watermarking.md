# 音频生成模型的潜空间水印

## 0. 摘要翻译

本文面向开源音频生成模型：后处理水印可被删掉，改 decoder 的水印也可通过替换声码器绕过。作者先训练对 EnCodec 极强鲁棒的 AudioSeal 式零比特水印器，再在数据加载时给全部训练音频加水印；MusicGen 学习这些音频的离散潜 token 后，其输出会继承水印，即使更换 decoder 仍可检测。对无水印数据微调可削弱水印，但足以移除时生成分布已接近从其他数据重训。

## 1. 方法动机

开放模型权重后，调用者可以注释掉后处理嵌入器；若只微调 EnCodec decoder，也能换成另一 vocoder。真正昂贵且承载数据分布的是潜 token 语言模型，因此水印应传播到 LM 生成的离散表示，而非依赖最后一层波形处理。核心 insight 是“训练数据中的稳健水印可以被生成模型记入输出分布”。

## 2. 威胁模型解读

提供者拥有专有训练数据并计划公开 MusicGen 权重。攻击者能移除后处理代码、替换 EnCodec decoder，或用另一无水印数据集微调 LM；目标是让输出不再被检测，同时保留原模型能力。防御者持有时间定位检测器。实验未覆盖完全重训、针对检测器的白盒对抗损失、训练数据混合比例攻击、剪枝/蒸馏或水印伪造。该威胁模型比纯信号处理更贴近开源模型，但建立在所有训练音频都可预先水印且从头训练的强条件上。

## 3. 方法设计与理解

### 3.1 对 tokenizer 鲁棒的水印器

作者以 AudioSeal 的生成器 G 和逐时间检测器 D 为基础，删除消息编码器，只做零比特存在/定位。G 产生加性残差，感知损失约束不可感知；训练中随机补零、用同 batch 无水印片段替换、或丢弃残差，D 用逐采样标签学习定位。为让水印进入潜 token，50% batch 额外经过将来用于 MusicGen 的 EnCodec 压缩再检测。

### 3.2 数据水印到潜模型

训练 MusicGen 时，每条音频在加载阶段在线加水印，10 秒约需 10 ms；随后 EnCodec 以 32 kHz、50 Hz 帧率、4 个 2048 大小 RVQ 码本编码，总码率约 2.2 kbps。MusicGen 的 delay pattern 将第 k 个残差流延迟 k 步，使 transformer 并行预测四路 token，损失仍是逐码本下一 token 交叉熵。模型结构、目标和训练流程都不因水印修改，变化只来自训练 token 的数据分布。

### 3.3 推理与检测

推理按普通 MusicGen 用文本或音频 prompt、top-k=250、temperature=1.0 采样潜 token，再用任意兼容 decoder 输出波形。部署不再运行嵌入器，只运行标准生成器；验证端以 D 输出逐时间分数，可对整段平均做检测，也可标出局部水印区间。

### 3.4 复现配置

| 项目 | 设置 |
|---|---|
| 水印训练 | 内部 1.5k 首音乐，32 kHz，1秒片段，400k steps，batch 64 |
| tokenizer 增强 | 50% batch 经目标 EnCodec |
| MusicGen 数据 | 20k 小时授权音乐（非公开） |
| 模型 | 300M / 1.5B，200 epochs，batch 128 |
| 算力 | 小模型 32 GPU，中模型 64 GPU |
| 检测评估 | 10k 正样本 + 10k 负样本，15秒 |
| 主要缺口 | 内部数据不可得，具体 optimizer 沿用上游默认 |

## 4. 与其他方法对比

AudioSeal 后处理在信号攻击下更强，但开放权重后可直接绕过；decoder 微调式水印能被替换 vocoder 删除；本文把水印提前到训练数据和潜 LM，decoder 不再是信任边界。代价是必须从头训练 MusicGen，使用 20k 小时全量水印数据和数十张 GPU，且检测鲁棒略低于直接后处理。

## 5. 实验表现

### 5.1 生成质量

40 个 MusicCaps prompt、每条 15 秒、20 名听众评分。300M 原模型/本文 OVL 为 83.67±1.85/84.13±2.21，REL 82.42±1.37/82.46±1.44；1.5B 为 85.92±1.46/84.91±1.53，REL 83.71±1.79/82.64±1.41，差异处于置信区间内，支持数据水印未明显损伤主观质量。

### 5.2 检测、定位与音频编辑

两种模型 AUC 均 0.999，最佳阈值准确率 0.993/0.994，TPR 0.986/0.988，FPR 约 2×10^-3/3×10^-3；局部 IoU 0.81/0.91。白噪声、低通、高通、重采样、AAC、粉噪、回声后 in-model AUC 为 0.88–0.999，整体低于直接 AudioSeal，尤其 AAC 准确率 0.81，但重采样反而达 0.99。

### 5.3 模型级攻击

换用离散 HiFi-GAN 后 AUC 0.999、准确率 0.990；Multi-band diffusion decoder 仍有 AUC 0.991、准确率 0.954，证明水印主要位于潜 token。用无水印数据微调 20 epoch：小学习率下 TPR@指定低 FPR 从 0.983 降至 0.765、FAD 从 2.067 升至 2.908；更大学习率把 TPR 降至约 0.018–0.079，但 FAD 3.34–3.74，接近从新数据 scratch 的 3.839。证据支持“移除需要放弃相当部分原分布”，但不等于无法移除。

## 6. 学习与应用

适合准备公开权重且可控制从头预训练数据管线的生成模型。复现关键是让水印先穿过目标 tokenizer 仍可检测，再验证它能从数据传播到 LM；若只在波形上表现好而 tokenization 后消失，训练再久也不会进入潜模型。

## 7. 总结

以水印训练数据保护开放式潜音频模型。

## 9. 复现难度与适合人群

难度高，瓶颈不是算法而是 20k 小时授权音乐、32/64 GPU 和从头训练 MusicGen。最小复现可用小规模 AudioCraft 模型，先训练 EnCodec-robust AudioSeal，再比较全水印/无水印训练集。适合研究开放模型溯源、数据水印传播和模型级攻击的读者。

## 10. 简短全面总结

本文通过“先给训练音频加水印，再照常训练 MusicGen”把 AudioSeal 的检测信号传播进离散潜 token。水印器在训练时以 50% 概率经过目标 EnCodec，从而保证语言模型能在 token 层学习该特征。300M/1.5B 模型的主观质量差异处于置信区间内，检测 AUC 约 0.999；替换 HiFi-GAN 或扩散 decoder 后仍保持高检测。无水印数据微调能移除水印，但同时使 FAD 接近从其他数据重训。主要代价是必须从头训练，且依赖大规模私有数据与算力。

## 11. 论文写作逻辑分析

论文从“开放模型可删除后处理代码”进一步推到“音频 decoder 也容易替换”，自然引出更深一层的潜 LM。实验严格按威胁链组织：质量、普通检测、decoder 替换、模型净化。最值得借鉴的是没有掩盖 post-hoc 在信号鲁棒上的优势，而是明确以不同部署威胁解释选择；不足是“微调不划算”依赖特定数据与 FAD，尚不足以概括所有参数高效净化攻击。

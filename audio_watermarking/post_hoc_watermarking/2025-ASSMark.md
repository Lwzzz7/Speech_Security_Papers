# ASSMark：以对抗式鲁棒水印同时防御语音合成与追踪版权

## 0. 翻译摘要原文

ASSMark 提出一种语音双防御机制：在一次水印嵌入中同时保留版权追踪信息，并把水印扰动塑造成能够干扰未经授权语音合成的对抗扰动。它不重新设计水印骨干，而是对 AudioSeal、Timbre 或 WavMark 等鲁棒水印编码器—解码器进行协同微调；冻结若干代理语音合成编码器，迫使水印语音的说话人表示远离受保护说话人并接近一个替代锚说话人。论文报告其在未知黑盒合成模型上达到 90% 以上的防御率，同时比“先水印、再对抗扰动”等两阶段方案保持更好的水印恢复和语音质量。

## 1. 方法动机

普通音频水印能在克隆发生后证明素材来源，却不会阻止攻击者用受保护语音训练或提示零样本语音合成；普通对抗样本可干扰克隆，却没有可恢复的版权标识，而且往往严重损害音质。把二者顺序叠加也会互相破坏：后加的对抗扰动会抹掉水印，后加水印又可能削弱防御。

ASSMark 的目标是让同一细微扰动同时承担“可解码的信息载体”和“使合成器说话人表征偏移的对抗载体”。其关键不在新网络，而在可插拔微调目标与多代理迁移训练。

## 2. 威胁模型解读

防御者拥有原始语音及待嵌入 bit，可访问一个鲁棒水印骨干，并在训练时白盒访问三种代理合成模型的说话人编码器。攻击者获得发布后的水印语音，尝试通过零样本语音转换/TTS 克隆原说话人；也可能先施加滤波、增益、duck、噪声、重采样、回声和平滑，再合成或破坏水印。

成功防御需要合成结果不能通过说话人验证，同时发布语音仍清晰、bit 仍可提取。黑盒测试覆盖 TorToiSe 与商业平台 Uberduck，但不是完全无关的任意生成范式。论文没有评估攻击者针对 ASSMark 进行自适应去扰动、对抗训练、声纹净化、多样本聚合或物理重录，也未讨论水印密钥和伪造。

## 3. 方法设计与理解

### 3.1 四个组成部分

水印编码器 `E(x,w)` 产生水印语音；Noise Layer 在线模拟失真；水印解码器从失真语音恢复 bit；冻结的 Speech Synthesis Encoder 提取说话人表示。预训练阶段沿用骨干的保真和恢复目标，微调阶段在此基础上加入防御与感知约束，因此 ASSMark 本质上是一种训练机制，而非固定架构。

### 3.2 从鲁棒水印到对抗水印

编码器损失约束水印语音和原语音的 L2 距离，解码器损失约束恢复水印与原 bit 的距离。防御损失包含两个相反方向：使失真水印语音的说话人嵌入靠近锚说话人 `y`，同时远离原说话人 `x`。作者把远离项权重设为 1。仅做“远离原说话人”会让合成音接近噪声却仍保留受害者身份，因此锚点项为偏移提供明确方向。

锚说话人从 24 名说话人的语音库中选择：计算其与受害者声纹的距离，选最不相似者。该策略增强攻击效果，但也意味着部署时需要锚库和说话人嵌入计算；论文没有充分讨论一个固定或公开锚点是否会形成可检测、可逆的模式。

### 3.3 多代理 top-k 迁移

现实中不能逐一针对所有合成器训练。ASSMark 把迁移视为鲁棒优化：同时计算多个代理编码器的防御损失，选择最难产生有效扰动的 k 个模型求平均。实验用 AdaptVC、SV2TTS、YourTTS 三个代理，`k=3`，其编码器分别覆盖 AdaIN-VAE、GE2E-LSTM 和 H/ASP-ResNet。不同架构共同施压，意在学习可跨系统迁移的声学扰动，而非单一模型漏洞。

### 3.4 感知约束与联合目标

作者沿用 Antifake 的等响度曲线频率惩罚：在人耳不敏感频带允许更大扰动，在敏感频带压低扰动。总损失由编码保真、解码恢复、防御和感知四项组成，实验设置 `λ_defense=10`、`λ_percept=3`，其余超参数遵循各水印骨干原实现。联合微调的意义是让编码器主动寻找“既能被解码器识别、又能欺骗合成器”的共同子空间，避免两阶段扰动争夺同一微小失真预算。

### 3.5 数据与实现

TIMIT 中 4000 条语音用于训练、1000 条用于验证；测试时向 139 条语音写入水印并用于合成。AudioSeal 和 Timbre 使用 16 bit，WavMark 使用 32 bit；AudioSeal/WavMark 采用官方预训练模型，Timbre 重新训练。论文没有给出统一微调轮数、batch size、学习率、硬件、随机种子和各 Noise Layer 的训练强度，因此“可插拔”可由结果验证，但完整训练成本不透明。

## 4. 与其他方法对比

AudioSeal、Timbre、WavMark只负责追踪；Antifake 只负责主动防克隆；ASSMark 把两者的目标写入同一次微调。与两阶段组合相比，它最大的结构优势是防御扰动和水印不再互相视为噪声。与专门重做水印网络的方法相比，它可复用不同骨干，但最终能力受骨干原有鲁棒性上限约束。

公平性需要谨慎：不同骨干 bit 数不同，MOS 较低且不同 ASSMark 版本差异明显；Antifake 的目标并非保留水印，所以 BER 维度天然不适用。论文对合成平台的覆盖有代表性，但数据集与测试说话人规模仍小。

## 5. 实验表现

### 5.1 主动防御：是否能让克隆语音认证失败

指标 AERR 表示合成样本未能绕过 SpeechBrain 说话人验证的比例，越高越好。三种原始水印骨干的白盒平均 AERR 只有 0.117–0.177，黑盒 Uberduck 为 0；随机噪声也仅约 0.2–0.32。微调后，ASSMark+AudioSeal 在三个白盒模型平均 AERR 0.997，在 TorToiSe 和 Uberduck 上为 0.960、0.900；ASSMark+Timbre 分别为 0.950、1.000、1.000；ASSMark+WavMark 为 0.853、0.960、0.833。结果证明对抗目标可以迁移，但“黑盒均超过 90%”并不适用于表中每个骨干—平台组合。

### 5.2 音质：防御成功是否以不可用语音为代价

Antifake 的 STOI 仅 0.285、MOS 1.509，虽有约 0.99 的白盒 AERR，却显著破坏可懂度。ASSMark 三种版本的 STOI 为 0.456–0.858、MOS 为 2.046–2.698；其中 AudioSeal 版本 AERR 最强但 STOI 0.858、MOS 2.138，WavMark 版本 MOS 2.698 较高但白盒 AERR 较低。由此可见 ASSMark 改善了“成功但难听”的问题，却远未恢复到原语音 MOS 3.633 或普通水印约 3.2–3.6，防御—音质权衡仍明显。

### 5.3 水印与防御鲁棒性：编辑后两种功能能否同时保留

测试攻击强度高于训练，覆盖带通、高通、低通、增益、duck、高斯噪声、32 kHz 重采样、0.5 s 回声、平滑和粉红噪声。平均 BER：ASSMark+AudioSeal 0.039，原 AudioSeal 0.155；ASSMark+Timbre 0.024，原 Timbre 0.021；ASSMark+WavMark 0.163，原 WavMark 0.202。说明微调没有系统性破坏恢复，个别骨干甚至因冗余扰动而改善，但效果取决于原骨干。

同样攻击后的白盒平均 AERR 分别为 0.972、0.949、0.884。低通和噪声是相对困难条件，但多数仍在 0.8 以上。这支持 Noise Layer 对双功能鲁棒性的作用，不过没有“移除 Noise Layer”的消融，因果归因不完整。

### 5.4 两阶段比较：联合训练是否真正必要

“Antifake 后加水印”和“先水印后 Antifake”的 STOI 均低于 0.3、MOS 约 1.46–1.52。后者还显著破坏恢复：AudioSeal BER 0.291、Timbre 0.184、WavMark 0.457。ASSMark 对应版本的无失真 BER 为 0，且保持较高 AERR 与更好音质。这是论文最直接、最有说服力的实验：它验证了联合目标，而不仅是证明对抗扰动本身有效。

## 6. 学习与应用

ASSMark 提供一种通用思路：让水印扰动同时成为下游模型的保护性对抗扰动。可用于公开说话人素材、配音数据集和可追踪语音 API 输出。实际部署必须重新评估多语种、长语音、多人语音、真实压缩/重录，以及攻击者通过降噪、声码器重合成或针对性微调消除对抗性的情况。

## 7. 总结

ASSMark 用冻结的多种语音合成编码器指导鲁棒水印微调，使水印语音的声纹远离原说话人并靠近替代锚点，同时保留 bit 恢复。它在多种白盒和两个黑盒合成平台上显著提高 AERR，并比两阶段方案保留更多音质和水印信息。主要局限是发布语音 MOS 仍明显下降，实验规模较小，训练细节不全，且没有自适应攻击评估。

## 9. 复现难度与适合人群

复现难度中高：需要三种水印骨干、三类代理合成器、说话人验证器、锚说话人库和音频攻击流水线。论文训练细节缺失会增加调参成本。适合研究语音克隆防御、对抗样本、水印多任务训练和数据版权保护的读者。

## 10. 简短全面总结

这篇论文不是发明新水印网络，而是把鲁棒水印变成双用途扰动：解码器读出版权 bit，合成器却因声纹偏移而难以克隆原说话人。多代理训练带来一定黑盒迁移，联合优化显著优于两阶段叠加；但音质代价、自适应攻击和复现细节仍是落地前必须补齐的部分。

## 11. 论文写作逻辑分析

文章从“被动追责”和“主动防御”各自缺一条腿切入，随后把 ASSMark 定位为对现有水印的通用微调器；实验先验证防御与音质，再验证编辑鲁棒性，最后用两阶段失败反证联合训练的必要性。写作很紧凑，表 V 是最关键的闭环。需要警惕的是摘要式表述比逐项结果更乐观，且缺少组件消融，使 top-k、锚点选择和感知损失的独立贡献仍主要依赖方法直觉。

# AudioMarkNet：让水印随说话人适配迁移到深伪语音

> Wei Zong, Yang-Wai Chow, Willy Susilo, Joonsang Baek, Seyit Camtepe. **AudioMarkNet: Audio Watermarking for Deepfake Speech Detection**. USENIX Security 2025.

## 0. 摘要翻译

深度生成模型的进步使伪造图像和音频越来越难与真实内容区分。现有深伪检测大多被动寻找生成模型留下的伪影，但生成器持续升级会削弱这些伪影，而且检测结果通常缺少可解释依据。AudioMarkNet 提出一种主动防御：在公开发布的原始语音中预先嵌入身份水印；若攻击者用这些语音对文本转语音模型进行说话人适配，模型也会学习水印，并把它带入新合成的克隆语音。检测器因此可以依据预定义水印识别深伪，而不依赖某个生成器的偶然伪影。实验覆盖 SV2TTS、YourTTS、PlayHT 和 Speechify，表明水印可以迁移至开源和商业 TTS，并能抵抗多种非自适应及自适应攻击。

## 1. 方法动机

反应式检测器把任务定义成“真实语音与当前深伪模型的伪影分类”，模型一换，判别边界便可能失效。已有生成端水印又通常要求服务商控制生成模型，无法约束攻击者自行训练或微调的模型。AudioMarkNet 改变了水印进入系统的位置：不标记生成结果，也不修改攻击者模型，而是先标记攻击者可能收集的训练语音。

其核心观察是，说话人适配要反复拟合目标说话人的频谱特征。如果同一身份水印出现在该说话人的每个训练片段和每个局部帧中，水印就会成为适配数据的稳定相关模式，被 TTS 与音色一起学习。这样，深伪检测从“猜测音频是否由模型生成”转化为“检查合成语音是否继承注册身份水印”。

## 2. 威胁模型解读

- 防御者在语音公开前嵌入与说话人绑定的 16 bit 水印，并维护注册水印集合 $W$；检测端持有解码器和注册表。
- 攻击者从公开数据集、社交媒体或音视频网站收集受害者语音，并用其对 TTS 做 speaker adaptation，再合成语义任意但音色相似的语音。
- 关键假设是攻击者**必须使用受保护语音微调 TTS**。如果预训练模型已经能零样本克隆受害者，或者攻击者只使用未加水印的历史语音，生成语音没有理由包含该水印。
- 安全目标包括三项：水印不明显损害语音可懂度；适配后水印能迁移到新文本对应的合成语音；未加水印语音不能被解码成任何注册水印。
- 论文测试黑盒信号处理、已知嵌入频带的带阻攻击、配对去水印自编码器、提前停止微调以及混入其他水印，但没有给出直接利用完整编码器/解码器梯度优化的白盒移除攻击。
- 论文没有评估扬声器—麦克风重录，且当前实现面向预先注册的闭集说话人，不等同于可实时保护任意新用户的通用水印器。

## 3. 方法设计与理解

### 3.1 从训练语音到克隆语音的完整链路

原始语音按 1 秒切段，每段都嵌入同一个说话人水印；攻击者下载这些语音并微调 TTS；验证时，待测语音同样被切成 1 秒段，每段独立解码。只要某段输出与注册表中的水印完全匹配，该段就被判定为伪造并可进一步归因到对应身份。

重复嵌入有两层作用：一是让不同长度和流式语音都能逐秒检测；二是让 TTS 在适配过程中高频遇到水印。它本质上接近一种目标明确的数据投毒，但“投毒”目标不是破坏模型功能，而是让模型保留可验证的频域标记。

### 3.2 编码器：只学习扰动，不重建载体

编码器先对原始波形做 STFT，同时由预训练 wav2vec 2.0 从原始波形提取内容特征。16 bit 消息经线性层投影到相同维度后与语音特征相加，再通过 LSTM 和多层全连接网络产生帧级频谱扰动 $\delta$。扰动只写入 100–1000 Hz，随后与原 STFT 相加并经 ISTFT 得到水印波形。

只预测扰动而不从头重建语音，减轻了编码器同时学习内容重建和水印隐藏的负担。选择低频带有两个理由：人声能量主要集中在该区域，水印更容易和说话人特征纠缠；邻近的强语音成分还能通过听觉掩蔽遮盖小扰动。训练中也没有放置某个 TTS 模拟器，目的在于避免水印只适配一种合成架构。

### 3.3 解码器与无水印拒识

解码器输入 1 秒语音的对数幅度谱，由多层卷积、两个全连接层组成，第一个全连接层后加 dropout。卷积层采用 instance normalization，而非依赖 batch 内样本相对统计的 batch normalization，因为不同语音的响度差异并不对应稳定语义。

论文不只训练“水印语音恢复消息”，还要求干净语音统一输出一个不属于注册集合的保留码 $w_0$。这一设计为开放世界拒识提供了显式负类，避免任何自然语音都被强行映射到某个真实用户。

### 3.4 损失函数

水印扰动以平方 $\ell_2$ 范数约束：

$$
\ell_\delta=\|\delta\|_2^2.
$$

二进制消息先通过 $h(w)=(2w-1)\tau$ 映射到目标 logits，论文令 $\tau=5$，再分别约束水印语音和干净语音：

$$
\ell_{wm}=\|g(x')-h(w)\|_2^2,
\qquad
\ell_{org}=\|g(x)-h(w_0)\|_2^2.
$$

总损失为

$$
\ell=\alpha\ell_\delta+\beta\ell_{wm}+\gamma\ell_{org},
$$

实验中 $\alpha=\beta=\gamma=1$。训练时偶尔在水印和原始语音上加入高斯噪声再送入解码器，迫使解码器放弃极脆弱的细微特征，同时反向促使编码器产生更稳定的水印。

### 3.5 关键配置

| 项目 | 设置 |
|---|---|
| 防护对象 | YourTTS 的 VCTK 测试集 11 位说话人，7 女 4 男 |
| 水印 | 16 bit，可表示 65,536 个身份；每人唯一 |
| 嵌入范围 | 100–1000 Hz；每个 1 秒片段均重复嵌入 |
| 编码器 | 预训练 wav2vec 2.0 + LSTM + FC，仅预测频谱扰动 |
| 解码器 | 对数 STFT 幅度谱 + 卷积层 + 2 个 FC；instance norm |
| TTS | SV2TTS、YourTTS、PlayHT、Speechify |
| 合成文本 | LibriSpeech test-clean 中 100 个句子；商业系统另用 14 句 |
| 开源模型适配 | 最多 100 iterations，每 10 次评估 |
| 训练资源 | Ubuntu、64 GB RAM、NVIDIA A100；200 epochs 约 138 分钟 |

## 4. 与其他方法对比

AudioMarkNet 与 WavMark、AudioSeal 等发布后水印的主要区别不是网络模块，而是验证对象：后者判断“这段音频是否被嵌入过”，AudioMarkNet 判断“未知文本的合成语音是否从受保护适配数据继承了水印”。相对 Timbre Watermarking，它牺牲更复杂、更隐蔽的水印模式，追求能被 TTS 稳定学习且经信号变换仍可识别的低频结构。相对 AASIST 等被动深伪检测，它需要防御者事先处理训练来源，却不依赖未知 TTS 的生成伪影。

因此不能把它与普通后处理水印只按 BER 横向排序。它最独特的指标应是“水印跨 TTS 训练过程的迁移速度、覆盖说话人数和检测延迟”。

## 5. 实验表现

### 5.1 原始水印语音：能否在低误报下逐秒检测

论文先检验水印本身。VCTK 受保护语音的 PESQ 为 $3.52\pm0.43$，SNR 为 $33.68\pm5.54$ dB；逐秒多类与二类 TPR/BTPR 均为 $99.9\pm0.2\%$，UFL/BUFL 均为 $1.0\pm0.0$ 秒，EER/BEER 均为 0。干净语音 FPR 为 $0.01\pm0.02\%$，约等价于平均 2.5 小时出现 1 秒误判。

这里 UFL/BUFL 不是传统 BER。UFL 表示连续多长的伪造语音仍未被正确归因，BUFL 只要求判出“属于任一注册水印”。数值越小，攻击音频在触发告警前能传播的内容越少。

### 5.2 水印是否真的迁移到不同 TTS

SV2TTS 与 YourTTS 的微调曲线显示，随着适配迭代增加，UFL 和 EER 下降、TPR 上升；未适配的基础模型 FPR 分别仅为 $0.05\pm0.10\%$ 和 0，说明模型不是天然输出注册水印。商业服务更困难：PlayHT 的 UFL/TPR/EER 为 $23.3\pm19.3$ 秒、$2.4\pm2.6\%$、45.5%，Speechify 为 $44.6\pm24.3$ 秒、$0.8\pm1.9\%$、81.8%。若只做二类存在性检测，PlayHT 的 BUFL/BEER 改善到 $17.7\pm16.5$ 秒和 18.2%，但 Speechify 仍为 $33.2\pm25.2$ 秒和 54.6%。

这组结果应谨慎解读：论文证明了部分商业黑盒服务确实会继承水印，却没有证明所有说话人、所有服务都能快速而稳定归因；开放模型上的强结果明显不能直接外推到商业系统。

### 5.3 常规信号处理攻击

时间域测试包括白噪声、重采样、中值滤波、样本抑制、幅度缩放、量化和回声；频域测试包括 MP3、Opus、低通、高通和降噪。开源 TTS 总体保持较短检测延迟，量化是较强的时间域攻击，高通与降噪是较强的频域攻击。作者报告即使在高通和降噪下，水印语音及 SV2TTS/YourTTS 合成语音平均仍可在约 7 秒内检测；Speechify 加降噪后多类水印可能被破坏，但二类检测仍保留部分能力。

### 5.4 自适应攻击

**带阻滤波。** 攻击者知道水印位于 100–1000 Hz，随机移除 100、200 或 400 Hz 宽的频带。带宽增大通常使水印语音和开源 TTS 的 UFL 上升；例如水印语音 UFL 从 100 Hz 时的 $1.8\pm1.9$ 秒升至 400 Hz 时的 $17.1\pm138.1$ 秒，YourTTS 仍为 $5.5\pm45.1$ 秒。结果说明分散在整个低频区比单一窄带稳，但大方差也反映部分样本会长时间漏检。

**去噪自编码器。** 攻击者用另一组 11 位说话人的干净/水印配对数据训练逆变换。它能把训练集 SNR 从 33.86 提高到 47.37 dB，并令训练集 EER 从 0 升到 95%，但对未见受害者泛化较差：受害者语音去噪后 UFL 仅从 1.0 升至 2.1 秒，YourTTS 合成语音从 1.6 升至 2.6 秒。该实验支持的是“这个去噪器跨说话人泛化不足”，并非一般意义上无法学习水印移除器。

**提前停止适配。** 攻击者每轮用 RawNet3 检查克隆语音，一旦能骗过说话人验证便停止。Obama 实验中，YourTTS 约在 16 次迭代后可靠越过验证阈值，此时 UFL 也已降到 5 秒以内；使用 4.9–73.5 秒适配语音的实验中，只要克隆成功骗过验证，水印同样能在 5 秒内被发现。边界仍然是目标若与 TTS 预训练分布足够接近、无需适配即可克隆，水印就不会迁移。

### 5.5 历史干净语音、双重水印与基线

混合数据实验更接近真实部署：水印语音占 80% 时，11/11 个说话人成功迁移，BUFL 为 $1.7\pm1.2$ 秒；占 60%、40%、20% 时分别只覆盖 9/11、6/11、2/11，说话人，20% 时 BUFL 上升到 $29.0\pm115.1$ 秒。可见“只要存在少量受保护语音就有效”不是稳定保证，覆盖率高度依赖水印样本比例和说话人。

攻击者再嵌入另一水印时，原始水印在 YourTTS 100 次适配后仍取得 $1.8\pm1.3$ 秒 UFL、$1.00\pm2.09\%$ EER。与 Timbre Watermarking 的比较中，白噪声加 16→8→16 kHz 重采样使后者即使适配 1000 次解码率仍低于 60%，而 AudioMarkNet 在 40 次时 EER 为 $0.64\pm1.43\%$。这说明易被 TTS 学习的冗余低频水印更抗该组合攻击，但比较只覆盖一个特定攻击和实现。

## 6. 学习与应用

最值得借鉴的是把“水印能否穿过目标学习过程”单独作为训练数据保护问题，而不是把所有水印都当作同一信道上的鲁棒编码。复现时应优先画出水印迁移随适配迭代、受保护数据比例和说话人变化的曲线，并同时报告 UFL 分布而非只看均值。工程上可把它用于固定说话人数据集、配音素材库和名人公开语料的预发布保护；实时通话、新用户即时注册与零样本克隆仍需新的条件化编码器和信道设计。

## 7. 总结

AudioMarkNet 用训练数据水印把深伪检测变成可归因的水印继承检测。

## 9. 复现难度与适合人群

难度高。除编码器和解码器外，还要训练多个说话人的水印模型、反复微调两套 TTS、运行说话人验证及多类攻击；官方 artifacts 约 41.5 GB，但已公开模型、数据处理和结果脚本。适合研究语音深伪主动防御、训练数据保护、模型指纹迁移和可解释来源归因的读者。

## 10. 简短全面总结

AudioMarkNet 在公开语音的每个 1 秒片段、每个局部帧中重复写入 16 bit 身份水印。编码器以 wav2vec 2.0、LSTM 和全连接层生成 100–1000 Hz 的小频谱扰动，解码器逐秒恢复水印，并把干净语音显式映射到未注册保留码。它不在训练时模拟某个 TTS，而依靠说话人适配对频谱特征的拟合，让水印随音色进入未知文本的合成语音。开源 TTS、部分商业服务、信号处理和三类自适应攻击支持该思路，但效果依赖适配、水印语音占比和目标说话人；零样本克隆、实时开放集保护、物理重录及强白盒移除仍未解决。

## 11. 论文写作逻辑分析

论文先指出被动深伪检测的伪影依赖，再把防御位置前移到公开训练语音；随后以“可感知性、迁移性、无误报”三条形式化要求约束方法。结构设计逐项对应要求：扰动范数保持音质，低频逐帧重复促进迁移，保留码承担拒识。实验也按同一逻辑展开：先证明原语音可检测，再证明跨 TTS 迁移，随后逐层增加常规、自适应和数据混合攻击，最后主动讨论闭集和物理信道限制。最值得学习的是它把适用前提写得相对清楚；需要警惕的是标题中的“deepfake speech detection”范围比实际威胁模型更宽，方法真正解决的是“使用受保护语音进行说话人适配后产生的克隆语音检测”。

# WAKE：支持密钥控制与多次嵌入的音频水印

## 0. 翻译摘要原文

神经音频水印仍面临三项问题：无法阻止未授权解码；多次嵌入后早期水印被覆盖；只能处理固定长度消息。WAKE 是首个密钥可控音频水印框架，以指定密钥嵌入并只允许对应密钥恢复消息，错误密钥无法正确解码；它还能在多次嵌入后分别恢复不同水印，并通过多水印组合支持变长消息。实验表明，WAKE 在水印音质和检测准确率上优于现有模型。

## 1. 方法动机

公开解码器使任何人都能读取水印内容，缺少访问控制；同一音频在创作、平台分发、二次授权中可能被不同主体反复标记，普通模型的后一次嵌入会覆盖前一次；固定 bit 长度又要求每种容量重训。

WAKE 以可逆网络为核心，让 8 bit key 控制哪些 invertible blocks 真正注入消息。不同密钥激活不同路径，使多个水印可以共存在同一时频表示中；解码时正确 key 选择对应逆路径，错误 key 应接近随机。Predict Module 进一步从水印音频估计前向过程中丢失的冗余，而非随机采样。

## 2. 威胁模型解读

- 多个授权实体可依次在同一音频中写入消息，每次持有独立 key；验证者只应恢复自己 key 对应的水印。
- 未授权者可以拿到音频，论文假定没有正确 key；错误 key 解码目标为约 50% BER。
- 信道覆盖上下采样、随机/粉红噪声、低/高/带通、增益、衰减和 shush，但没有 MP3、神经 codec、重录、变速/裁剪等更强攻击。
- 安全实验只证明“随机错误 key 下不能恢复”，没有密钥搜索复杂度、已知明文、chosen-message、模型白盒或 key collision 分析。8 bit key 仅 256 种，从密码学角度无法抵抗穷举。
- 多次嵌入主要评估两次；更多次数只指向 demo，正文没有系统容量—干扰曲线。所谓变长水印通过多次 32 bit 嵌入实现，不是单次模型原生接收任意长度序列。

## 3. 方法设计与复现级解读

### 3.1 全局流程

一秒波形与 32 bit 消息分别经 STFT/消息映射进入 8-block INN。8 bit key 的第 $i$ 位控制第 $i$ 个 block 是否把消息特征加到音频分支；输出音频谱经 ISTFT 得水印音频，消息分支输出作为冗余但不随音频传输。解码时水印谱进入 Predict Module 估计音频相关冗余，结合 key 逆向运行 INN，再经映射恢复 bit。训练随机选择单水印或双水印流程，并随机选择一种音频编辑。

### 3.2 Key-gated 可逆块

第 $i$ 个块前向为

$$x_f^{i+1}=x_f^i+\phi(wm_f^i)k_i,$$
$$wm_f^{i+1}=wm_f^i\odot\exp(\alpha(\rho(x_f^{i+1})))+\eta(x_f^{i+1}).$$

当 $k_i=0$，该块不从消息支路向音频分支注入；$k_i=1$ 时注入。耦合变换保留可逆性，8 个 key bit 与 8 个 blocks 一一对应。不同 key 不是独立模型参数，而是同一网络的门控组合，因此多水印仍会共享载体空间并相互干扰。

### 3.3 Predict Module

标准 INN 解码需要前向消息分支产生的冗余，实际传输中不可得，WavMark 等用高斯随机量替代。WAKE 认为冗余已与音频交互，应该由水印谱预测；8 个残差块的 Predict Module 从 $x_{wm,f}$ 输出 $wm_{pre}$，再与 key 一起逆推。该模块既提高恢复，也通过反向约束帮助前向嵌入收敛。

### 3.4 感知与边界权重

感知损失组合波形 $L_2$、判别器、多尺度 64-bin Mel 的 $L_1+L_2$。作者发现一秒片段边缘出现高频能量堆积和爆音，因此 BroadWeight 把首尾 3% 样本权重设 10，其余为 1，直接加重边界谱误差。多尺度 FFT 窗覆盖 $2^5$–$2^{11}$，hop 为窗长四分之一。

### 3.5 正确与错误 key 约束

正确 key 用 BCE 逼近原消息；随机错误 key 的 BCE 若低于阈值 $w_{l2}$ 才受 hinge 惩罚：

$$L_a=\operatorname{BCE}(wm,wm_{re})+w_{l1}\max(0,w_{l2}-\operatorname{BCE}(wm,wm_{wrong})).$$

该项只要求错误 key “足够不相似”，不规定具体反码，避免所有错误 key 收敛到同一可预测消息。

### 3.6 单/双水印交替训练

单水印分支生成 $(wm,k)$，嵌入、随机编辑、正确/错误 key 解码。双水印分支先以 $(wm_1,k_1)$ 写入，再以 $(wm_2,k_2)$ 写入，经同一编辑后分别用 $k_1,k_2$ 解码。总损失对两次水印音频都施加感知约束，并对两条恢复施加 $L_a$。这种训练显式暴露覆盖冲突，是 WAKE 能保留首个水印的关键。

## 4. 与其他方法对比

| 方法 | 访问控制 | 多次嵌入 | 容量 | 局限 |
|---|---|---|---|---|
| AudioSeal | 无 key | 后水印覆盖前水印 | 16 bit | 双嵌首条 BER 46.75% |
| WavMark | 无 key | 后水印覆盖前水印 | 32 bit | 双嵌首条 BER 48.37% |
| WAKE | block-gated key | 可分别恢复 | $32n$ bit | key 仅 8 bit、正文主要双嵌 |

## 5. 实验表现与优势

### 5.1 单水印与 key 控制

单次嵌入时 WAKE 的 SNR/PESQ/正确 key BER 为 41.237 dB/4.396/0.123%，优于 WavMark 38.594/4.256/1.091% 和 AudioSeal 24.686/4.316/1.910%。随机错误 key BER 为 50.09%，说明输出近似随机，但并不能证明抗穷举。

### 5.2 双水印可恢复性

双嵌后 WAKE 的首/次水印 BER 为 1.252%/2.709%，SNR/PESQ 为 38.884/4.318。AudioSeal 首/次为 46.75%/3.79%，WavMark 为 48.37%/1.91%，两基线几乎只保留最后一次。WAKE 错误第三 key 下 BER 为 42.46%/41.44%，未完全达到 50%，提示多水印干扰会泄露一定统计结构。

### 5.3 音频编辑

单水印 WAKE 在九类编辑下 BER 0.127%–1.765%，随机噪声和低通最强；双水印首条为 1.23%–3.26%，第二条为 2.68%–6.65%。shush 使 AudioSeal 接近 49.8%，但 WAKE 仍低。实验覆盖面偏传统，无法推断对重录、变速和神经重合成的能力。

### 5.4 感知与 Predict Module 消融

仅 $L_2$ 时单水印 SNR/PESQ 为 35.113/4.023；加入多尺度 Mel 提升到 39.344/4.231；再加 BroadWeight 达 41.103/4.388，但 BER 从 0.08% 略升到 0.12%。频谱图显示前两者边缘有高频爆音，BroadWeight 消除这一听感缺陷，说明客观平均指标会漏掉局部伪影。

移除 Predict Module 后单水印 BER 从 0.13% 升至 5.10%，双水印首/次从 1.25/2.72% 升至 8.92/13.24%，音质也下降约 4 dB SNR。该消融直接验证音频相关冗余预测优于随机采样。

## 6. 学习与应用

复现优先完成 single/double 交替采样和错误 key hinge，避免只在单水印上训练后直接测试叠加。工程部署必须扩大 key 空间、引入密码学 KDF/认证，并评估 key collision 和穷举；INN key 只能视为条件控制，不应直接宣传为保密加密。多方流转追踪是最自然应用：创作者、平台、分销商各写入独立标记。

## 7. 总结

用密钥门控可逆块保留多重水印。

## 9. 复现难度与适合人群

难度中高，数据量大且需要可逆网络、两种训练分支、判别器与多尺度谱损失。最小版本可在 LibriSpeech 训练 8-block、32 bit，先复现无攻击单/双嵌。适合多方版权链、可逆网络和条件化水印研究者；不适合作为现成密码系统。

## 10. 简短全面总结

WAKE 针对公开解码、多次水印覆盖和固定容量问题，用 8 bit key 逐块门控 8 个可逆网络单元，在一秒音频中写入 32 bit 消息；解码时 Predict Module 从水印谱估计与载体相关的冗余，再用正确 key 逆向恢复。训练随机交替单次和双次嵌入，并用错误 key hinge、波形/多尺度 Mel/对抗损失及首尾 BroadWeight 共同优化。单水印 BER 0.123%、错误 key 约 50%，双水印仍可分别以 1.25%/2.71% BER 恢复，明显避免基线覆盖问题。主要边界是 8 bit key 可穷举、正文只系统验证两次嵌入，且攻击集未覆盖重录、变速和生成式编辑。

## 11. 论文写作逻辑分析

论文以三个具体工程痛点直接对应 key、multi-embedding、variable length 三项贡献，叙事紧凑。INN 解决复用、Predict Module 解决逆过程冗余、双分支训练解决覆盖，模块并非堆叠。实验通过首/次/错误 key 三维 BER 很好地闭环主张，并用边缘爆音说明仅看 SNR/PESQ 的不足。安全写作的主要缺陷是把 8 bit 条件 key 的随机错误结果上升为“安全”，缺少密码学和自适应威胁模型。
